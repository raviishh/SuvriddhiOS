Lists in Python

Lists are ordered, mutable collections used to store multiple values in a single variable. They are one of the most fundamental and frequently used data structures in Python.

1. What Is a List?

A list is a sequence of elements enclosed in square brackets. Lists can contain elements of different types and preserve insertion order. 

numbers = [1, 2, 3, 4]
names = ["Alice", "Bob", "Charlie"]
mixed = [1, "hello", 3.14, True]
Key Properties
• Ordered
• Mutable (can be changed)
• Allow duplicate values
• Can store mixed data types

2. Accessing Elements

Lists use zero-based indexing.

fruits = ["apple", "banana", "cherry"]

print(fruits[0])   # apple
print(fruits[-1])  # cherry (negative indexing)

You can also use slicing to extract sublists.

print(fruits[0:2])   # ['apple', 'banana']
print(fruits[:2])    # same result
print(fruits[1:])    # ['banana', 'cherry']

3. Modifying Lists

Because lists are mutable, their contents can be changed.

numbers = [1, 2, 3]

numbers[0] = 10
numbers.append(4)
numbers.insert(1, 99)
numbers.remove(3)

print(numbers)

Common list methods:

  • append(x) – Add item to end
  • insert(i, x) – Insert at position
  • remove(x) – Remove first occurrence
  • pop() – Remove and return last item
  • sort() – Sort list in-place
  • reverse() – Reverse list in-place

4. Iterating Over Lists

numbers = [1, 2, 3, 4]

for num in numbers:
    print(num)

If you need both index and value:

for index, value in enumerate(numbers):
    print(index, value)

5. List Operations

a = [1, 2]
b = [3, 4]

print(a + b)      # Concatenation
print(a * 3)      # Repetition
print(2 in a)     # Membership test

Lists support comparison operations and can be nested.

matrix = [[1, 2], [3, 4]]
print(matrix[0][1])  # 2

6. Copying Lists

Assigning one list to another variable does not create a copy. It creates a reference. 

a = [1, 2, 3]
b = a

b.append(4)
print(a)  # [1, 2, 3, 4]

To create a shallow copy:

b = a.copy()
# or
b = a[:]

7. Common Mistakes

  • Forgetting lists are zero-indexed
  • Confusing append() with extend()
  • Accidentally modifying a list through shared references
  • Using lists when a set or dictionary would be more appropriate

8. Why Lists Matter

Lists provide flexible, dynamic storage for ordered data. They serve as the foundation for many algorithms and higher-level data structures.

Mastering lists is essential before progressing to comprehensions, stacks, queues, and more advanced data abstractions.
