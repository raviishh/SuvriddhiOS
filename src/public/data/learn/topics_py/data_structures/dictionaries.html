Dictionaries in Python

Dictionaries are unordered (in concept), key–value data structures optimized for fast lookups. They map unique keys to corresponding values and are implemented using hash tables.

1. What Is a Dictionary?

A dictionary stores data as key–value pairs inside curly braces. Each key must be unique and hashable. 

student = {
    "name": "Alice",
    "age": 16,
    "grade": "A"
}
Key Properties
• Key–value structure
• Keys must be immutable (hashable)
• Values can be any type
• Fast average-case lookup (O(1))

2. Creating Dictionaries

a = {"x": 1, "y": 2}

b = dict(x=10, y=20)

pairs = [("a", 1), ("b", 2)]
c = dict(pairs)

An empty dictionary: 

empty = {}

3. Accessing and Modifying Values

student = {"name": "Alice", "age": 16}

print(student["name"])

student["age"] = 17
student["city"] = "Bangalore"

print(student)

Safer access using get(): 

print(student.get("grade"))          # None
print(student.get("grade", "N/A"))   # Default value

4. Dictionary Methods

d = {"a": 1, "b": 2}

print(d.keys())
print(d.values())
print(d.items())

d.pop("a")
d.update({"c": 3})

Important methods:

  • keys() – View of keys
  • values() – View of values
  • items() – View of key–value pairs
  • pop(key) – Remove key and return value
  • update(dict) – Merge dictionaries
  • clear() – Remove all entries

5. Iterating Over Dictionaries

d = {"a": 1, "b": 2}

for key in d:
    print(key, d[key])

for key, value in d.items():
    print(key, value)

Iteration order preserves insertion order (Python 3.7+).

6. Dictionary Performance and Design

Dictionaries use hash tables internally. Average-case lookup, insertion, and deletion are O(1).

Design Insight
Use dictionaries when you need fast lookups by identifier, not when order-based indexing is required.

7. Nested Dictionaries

students = {
    "Alice": {"age": 16, "grade": "A"},
    "Bob": {"age": 17, "grade": "B"}
}

print(students["Alice"]["grade"])

Nested dictionaries model structured or hierarchical data.

8. Common Mistakes

  • Using mutable types (like lists) as keys
  • Accessing non-existent keys without get()
  • Confusing dictionaries with lists
  • Overusing nested dictionaries when classes would be clearer
Quick Check
What happens here? d = {"a": 1} d["b"] = 2 del d["a"] print(d)

9. Why Dictionaries Matter

Dictionaries are foundational for building scalable systems. They power configuration systems, JSON structures, caching, and APIs.

Mastering dictionaries enables efficient data modeling and high-performance lookup operations.

Practice Strategy
Build a small key–value database such as a contact manager or inventory tracker using dictionaries.
