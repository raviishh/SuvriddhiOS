Nested Data Structures in Python

Nested data structures are combinations of lists, tuples, sets, and dictionaries stored within one another. They allow representation of hierarchical, multi-dimensional, or structured data.

1. What Are Nested Data Structures?

A nested data structure contains one data structure inside another. 

matrix = [
    [1, 2, 3],
    [4, 5, 6]
]

student = {
    "name": "Alice",
    "scores": [85, 90, 88]
}
Common Patterns
• List of lists (matrices)
• Dictionary of lists
• List of dictionaries
• Dictionary of dictionaries

2. Accessing Nested Elements

Access requires multiple indexing operations. 

matrix = [[1, 2], [3, 4]]
print(matrix[1][0])   # 3

students = {
    "Alice": {"math": 90, "science": 85}
}

print(students["Alice"]["math"])  # 90

Each level requires a separate index or key lookup.

3. Modifying Nested Structures

matrix = [[1, 2], [3, 4]]
matrix[0][1] = 99

students = {
    "Alice": {"math": 90}
}

students["Alice"]["math"] = 95
students["Alice"]["english"] = 88

Modifications depend on whether the inner structure is mutable.

4. Iterating Through Nested Structures

Iteration often requires nested loops. 

matrix = [[1, 2], [3, 4]]

for row in matrix:
    for value in row:
        print(value)

Iterating through nested dictionaries: 

students = {
    "Alice": {"math": 90, "science": 85},
    "Bob": {"math": 78, "science": 88}
}

for name, subjects in students.items():
    for subject, score in subjects.items():
        print(name, subject, score)

5. Combining Comprehensions

Nested comprehensions can generate complex structures. 

matrix = [[i * j for j in range(3)] for i in range(3)]

squares = {
    i: {j: i * j for j in range(3)}
    for i in range(3)
}

Keep comprehensions readable and avoid excessive nesting.

6. Deep vs Shallow Copy

Shallow copying duplicates only the outer structure. Inner objects remain shared. 

import copy

matrix = [[1, 2], [3, 4]]
shallow = matrix.copy()
deep = copy.deepcopy(matrix)
Important Distinction
Use deep copies when modifying nested data independently.

7. Real-World Applications

  • Representing JSON data
  • Modeling databases
  • Storing configuration files
  • Building adjacency lists in graphs
Design Insight
If nesting becomes too deep, consider defining classes for better structure.

8. Common Mistakes

  • Incorrect indexing depth
  • Modifying shared references unintentionally
  • Overcomplicating structure instead of simplifying design
  • Forgetting to handle missing keys in nested dictionaries
Quick Check
What is printed? data = [[1, 2], [3, 4]] copy_data = data copy_data[0][0] = 99 print(data)

9. Why Nested Data Structures Matter

Nested structures enable modeling of complex, real-world systems. They form the backbone of APIs, configuration systems, and structured data processing.

Mastery of nested data structures is essential for writing scalable and maintainable applications.

Practice Strategy
Parse a JSON-like dictionary and extract specific nested values. Then refactor the structure for clarity and efficiency.
