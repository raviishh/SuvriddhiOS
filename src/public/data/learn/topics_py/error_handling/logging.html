The Logging Module

Print statements are temporary. Logging is systematic. In real applications, you don’t just fix bugs — you monitor systems while they run.

1. Why Not Just Use print()?

Beginners debug using print(). That works — for small scripts.

But in larger programs:

  • You need different levels of importance
  • You may want logs saved to a file
  • You might need timestamps
  • You may want to disable debug output later
Core Idea
Logging gives structure, control, and scalability.

2. Basic Logging Setup

Python provides a built-in logging module.

import logging

logging.basicConfig(level=logging.INFO)

logging.info("Program started.")
logging.warning("This is a warning.")
logging.error("Something went wrong.")

This replaces print statements with categorized messages.

3. Logging Levels

Logging messages have severity levels.

  • DEBUG → Detailed diagnostic info
  • INFO → General program events
  • WARNING → Something unexpected happened
  • ERROR → A serious problem occurred
  • CRITICAL → Program may not continue

Example:

import logging

logging.basicConfig(level=logging.DEBUG)

logging.debug("Variable x = 10")
logging.info("User logged in")
logging.warning("Low disk space")
logging.error("File not found")
logging.critical("System failure")

If the logging level is set to INFO, DEBUG messages won’t appear. This filtering is powerful.

4. Logging to a File

In real applications, logs are saved for later analysis.

import logging

logging.basicConfig(
    filename="app.log",
    level=logging.INFO,
    format="%(asctime)s - %(levelname)s - %(message)s"
)

logging.info("Application started.")

Now logs are written to app.log instead of the console.

The format string adds:

  • Timestamp
  • Severity level
  • Message

5. Logging Exceptions Properly

Instead of printing errors manually, log them.

import logging

logging.basicConfig(level=logging.ERROR)

try:
    result = 10 / 0
except ZeroDivisionError:
    logging.exception("Division by zero occurred")

logging.exception() automatically includes the traceback. This is professional error reporting.

6. When to Log vs When to Raise

This distinction matters.

  • Raise → Stop execution because a rule was violated
  • Log → Record information about what happened

Sometimes you do both.

if balance < 0:
    logging.error("Negative balance detected.")
    raise ValueError("Balance cannot be negative.")

Logging documents the event. Raising enforces the rule.

7. Avoid Common Logging Mistakes

  • Logging everything at ERROR level
  • Using print and logging inconsistently
  • Exposing sensitive data in logs
  • Ignoring logs entirely

Logs are diagnostic tools. Use them thoughtfully.

8. Mini Exercise

Create a program that:

  • Logs when the program starts
  • Asks the user for a number
  • Logs DEBUG information about the input
  • Handles division by zero
  • Logs errors properly
  • Saves logs to a file

Think about which logging level each message belongs to.
