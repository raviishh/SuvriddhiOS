Understanding Package Structure in Python

Learn how to organize multiple modules inside folders. Packages help structure large projects in a clean and professional way.

1. What is a package?

A package is simply a folder that contains Python modules.

If a module is one file, a package is a folder of related files.

Simple Analogy
If modules are notebooks, then a package is a school bag that holds many notebooks for one subject.

2. Creating your first package

Let’s create this folder structure: 

myproject/
│
├── main.py
└── tools/
    ├── __init__.py
    ├── math_tools.py
    └── string_tools.py

The folder tools is a package.

The file __init__.py tells Python that this folder should be treated as a package.

3. Writing modules inside a package

Inside math_tools.py: 

# tools/math_tools.py

def multiply(a, b):
    return a * b

Now in main.py: 

# main.py

from tools.math_tools import multiply

print(multiply(4, 5))
Output:
20

4. What is __init__.py?

The file __init__.py runs when the package is imported.

It can:

  • Initialize the package
  • Import specific functions for easier access
  • Stay empty (which is common)

# tools/__init__.py

from .math_tools import multiply

Now you can write: 

from tools import multiply

Instead of: 

from tools.math_tools import multiply

5. Absolute vs Relative Imports

There are two main ways to import inside packages:

Absolute Import (recommended)

from tools.math_tools import multiply

Relative Import

from .math_tools import multiply

The dot . means “current package”.

Best Practice
Use absolute imports in most cases. They are clearer and easier to understand.

6. Why package structure matters

Packages help:

  • Organize large projects
  • Avoid name conflicts
  • Make code reusable
  • Prepare projects for distribution

Most professional Python projects use packages.

7. Key idea to remember

A package is a folder with Python modules inside it.

Structure makes projects easier to manage as they grow.

Friendly tip
If your project has more than 3–4 modules, consider organizing them into packages.
