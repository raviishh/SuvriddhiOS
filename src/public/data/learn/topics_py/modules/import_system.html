What is the Python Import System?

A simple guide to understanding how Python shares code between files. Think of this as learning how to borrow tools instead of building everything from scratch.

1. What is a module?

A module is simply a Python file. That’s it. If you create a file called tools.py, that file is a module.

Modules help us organize code. Instead of writing everything in one big file, we split code into smaller pieces.

Simple Analogy
Imagine cooking. You don’t grow the vegetables every time. You go to the market and get what you need. Modules are like that market — you import what you need.

Python already gives us many built-in modules like:

  • math (for mathematics)
  • random (for random numbers)
  • os (for working with the system)

2. Basic import

To use a module, we write the import keyword. 

import math

print(math.sqrt(16))

Explanation:

  • import math loads the math module.
  • math.sqrt(16) calls the square root function.

Output:
4.0

3. Import specific parts

Sometimes we only want one thing from a module. 

from math import sqrt

print(sqrt(25))

Now we can use sqrt directly without writing math.sqrt.

Important
Using from module import * is not recommended for beginners. It can cause name conflicts and confusion.

4. Using an alias

Sometimes module names are long. We can rename them using as. 

import math as m

print(m.pi)

This is common in real projects. For example:

  • import numpy as np
  • import pandas as pd

5. Creating your own module

Let’s create a file called greetings.py. 

# greetings.py

def say_hello(name):
    return f"Hello, {name}!"

Now in another file: 

import greetings

print(greetings.say_hello("Anna"))
Output:
Hello, Anna!

Python finds greetings.py in the same folder and loads it as a module.

6. What happens when you import?

When Python imports a module:

  • It runs the file one time.
  • It stores the module in memory.
  • If imported again, it does not run it again.

This makes programs faster and prevents repeating code.

7. Why imports matter

Imports allow:

  • Code organization
  • Code reuse
  • Cleaner projects
  • Team collaboration

Friendly tip
If your file becomes very long, it is time to split it into modules. Small files are easier to understand and debug.
