Variables in Python

A clear and friendly guide to understanding variables in Python — how they store data, how they behave, and why they matter. Think of variables as labeled jars where you store ingredients for your program.

1. What is a variable?

A variable is a name that refers to a value. In Python, when you create a variable, you are telling the computer: “Store this value and let me refer to it using this name.”

Analogy
Imagine you have jars in a kitchen. You label one sugar and put sugar inside. The label is the variable name, and what’s inside the jar is the value.

Unlike some languages, Python does not require you to declare a type explicitly. The type is determined automatically based on the value you assign.

2. Creating variables

In Python, you create a variable using the assignment operator =. 

age = 16
name = "Aarav"
height = 5.8
is_student = True

print(age)
print(name)
print(height)
print(is_student)

Explanation:

  • age = 16 stores an integer.
  • name = "Aarav" stores a string (text).
  • height = 5.8 stores a float (decimal).
  • is_student = True stores a boolean value.

Try it: Save as variables.py and run python variables.py.
Output:
16
Aarav
5.8
True

3. Variable types — what can they store?

Python supports many data types. The most common beginner types are: 

x = 10            # int
y = 3.14          # float
z = "Hello"       # str
flag = False      # bool

print(type(x))
print(type(y))
print(type(z))
print(type(flag))

Notes:

  • int → whole numbers
  • float → decimal numbers
  • str → text
  • bool → True or False

Important Idea
Python is dynamically typed. This means you don’t tell Python the type — it figures it out automatically.

4. Changing values

Variables can change. You can reassign them at any time. 

score = 10
print(score)

score = 20
print(score)

The variable score first stores 10, then stores 20. The old value is replaced.

Think Carefully
In Python, variables are names that reference objects. When you reassign, the name now points to a different value.

5. Variable naming rules

Python has simple naming rules:

  • Names can contain letters, numbers, and underscores.
  • Names cannot start with a number.
  • Names are case-sensitive (age and Age are different).

user_name = "Riya"   # valid
score2 = 100         # valid
2score = 50          # invalid (cannot start with number)

Best practice: use descriptive names like total_price instead of vague names like tp.

6. Simple operations with variables

Variables can be used in calculations and string operations. 

a = 5
b = 3
sum_result = a + b

first = "Hello"
second = "World"
message = first + " " + second

print(sum_result)
print(message)

Here:

  • Numbers can be added, subtracted, multiplied, etc.
  • Strings can be combined using + (concatenation).

7. Practice ideas

To strengthen your understanding, try:

  • Create variables for your name, age, and favorite subject.
  • Calculate your age next year.
  • Create two strings and combine them into a sentence.

Friendly tip
Keep experimenting. Change values, print types, mix numbers and strings. The fastest way to understand variables is to play with them.
