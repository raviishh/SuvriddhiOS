Packaging and Distribution

A simple introduction to packaging in Python — how to organize your code and share it with others. Think of this as turning your code into a toolbox that other people can use.

1. What is a Python Package?

A Python package is a collection of Python files grouped together. It helps organize code into modules.

A module is just a Python file. A package is a folder that contains Python files.

Analogy
One tool is useful. A toolbox with many tools is more powerful. A package is like that toolbox.

2. Installing Packages with pip

Python uses a tool called pip to install packages. 

pip install numpy

This downloads and installs the package.

To see installed packages: 

pip list

3. Virtual Environments

A virtual environment is a separate workspace for your project. It keeps project packages isolated.

This prevents version conflicts between projects. 

python -m venv myenv

Activate it: 

# Windows
myenv\Scripts\activate

# Mac/Linux
source myenv/bin/activate

After activation, installed packages stay inside this environment.

4. requirements.txt

A requirements.txt file lists the packages your project needs. 

pip freeze > requirements.txt

This saves all installed packages into the file.

Another person can install them using: 

pip install -r requirements.txt

5. Basic Project Structure

A simple project folder may look like this: 

my_project/
│
├── main.py
├── utils.py
├── requirements.txt
└── README.md

  • main.py — main program
  • utils.py — helper functions
  • requirements.txt — needed packages
  • README.md — project explanation

Good structure makes your project easier to understand.

6. Sharing Your Project

You can share your project by:

  • Sending the folder
  • Uploading to GitHub
  • Publishing as a package

Clear structure and requirements make sharing easy.

Friendly Tip
Always include a README file. Explain what the project does and how to run it.

7. Why Packaging Matters

Packaging helps you:

  • Organize code clearly
  • Avoid dependency problems
  • Share projects professionally

Even small projects should follow good structure. Good habits make you a better developer.
