Recursion in Python

Recursion is a technique where a function calls itself to solve a problem. Instead of repeating code with loops, recursion breaks a problem into smaller versions of the same problem.

1. What Is Recursion?

A recursive function solves a problem by:

  • Calling itself
  • Reducing the problem size each time
  • Stopping when it reaches a base case
Core Components
Every recursive function must have:
  • A base case (stopping condition)
  • A recursive case (self-call)

2. Basic Example: Factorial

def factorial(n):
    if n == 0:
        return 1
    return n * factorial(n - 1)

print(factorial(5))

Breakdown:

  • Base case: factorial(0) = 1
  • Recursive case: n × factorial(n − 1)

Each call reduces the problem size until the base case is reached.

3. How the Call Stack Works

Each recursive call is placed on the call stack. Once the base case is reached, calls resolve in reverse order. 

factorial(3)
= 3 * factorial(2)
= 3 * (2 * factorial(1))
= 3 * (2 * (1 * factorial(0)))
= 3 * 2 * 1 * 1

The stack grows during calls and shrinks during returns.

4. Another Example: Fibonacci

def fibonacci(n):
    if n <= 1:
        return n
    return fibonacci(n - 1) + fibonacci(n - 2)

This demonstrates multiple recursive calls per function call.

Performance Warning
Naive Fibonacci recursion is inefficient due to repeated calculations.

5. When to Use Recursion

Recursion is especially useful for:

  • Tree structures
  • Divide-and-conquer algorithms
  • Mathematical definitions
  • Backtracking problems

Examples include binary trees, file systems, and quicksort.

6. Recursion vs Iteration

# Iterative factorial
def factorial_iter(n):
    result = 1
    for i in range(1, n + 1):
        result *= i
    return result

Iteration often uses less memory. Recursion often improves conceptual clarity.

Design Decision
Choose recursion when it models the problem naturally. Choose iteration when performance and memory matter.

7. Common Mistakes

  • Forgetting the base case (infinite recursion)
  • Not reducing the problem size
  • Exceeding Python’s recursion depth limit
  • Using recursion where iteration is simpler
Quick Check
What happens? def test(n): return test(n - 1) test(5)

8. Why Recursion Matters

Recursion encourages thinking in terms of self-similarity and decomposition. It is foundational in computer science and algorithm design.

Mastering recursion strengthens problem-solving and algorithmic reasoning.

Practice Strategy
Rewrite a simple loop-based problem (like summing numbers) using recursion. Compare readability and efficiency.
