Parameters & Arguments in Python

Parameters and arguments define how functions receive data. They are the mechanism that transforms a static block of code into a flexible, reusable abstraction.

1. The Core Distinction

Although often used interchangeably in casual conversation, parameters and arguments are technically different.

  • Parameter — a variable listed in the function definition
  • Argument — the actual value passed when calling the function
def greet(name):   # 'name' is a parameter
    print("Hello,", name)

greet("Alice")     # "Alice" is an argument
Precision Matters
Parameters belong to the definition. Arguments belong to the call.

2. Multiple Parameters

Functions can accept multiple inputs separated by commas. 

def add(a, b):
    return a + b

result = add(3, 5)

The order of arguments matters unless specified otherwise.

3. Positional vs Keyword Arguments

Positional Arguments

Matched based on order. 

def describe(name, age):
    print(name, "is", age, "years old")

describe("Sam", 16)

Keyword Arguments

Matched by name. 

describe(age=16, name="Sam")
Design Insight
Keyword arguments improve readability and reduce errors in long parameter lists.

4. Default Parameters

Parameters can have default values. 

def greet(name="Guest"):
    print("Hello,", name)

greet()
greet("Alice")

If no argument is provided, the default is used.

Best Practice
Default values should be immutable (e.g., numbers, strings, tuples) to avoid unintended side effects.

5. Variable-Length Arguments

Python allows flexible argument counts using special syntax.

*args (Positional)

def total(*numbers):
    return sum(numbers)

print(total(1, 2, 3, 4))

**kwargs (Keyword)

def display_info(**info):
    print(info)

display_info(name="Alex", age=17)

*args collects extra positional arguments as a tuple. **kwargs collects extra keyword arguments as a dictionary.

6. Argument Unpacking

You can unpack sequences or dictionaries into arguments. 

values = (3, 5)
print(add(*values))

data = {"name": "Sam", "age": 16}
describe(**data)

This spreads elements into individual arguments.

7. Common Mistakes

  • Confusing parameters with arguments
  • Mixing positional and keyword arguments incorrectly
  • Using mutable default arguments (e.g., lists)
  • Passing too many or too few arguments
Quick Check
What happens? def test(a, b=2): return a + b print(test(3))

8. Why Parameters & Arguments Matter

Parameters define a function’s interface. Arguments supply the data that powers its behavior.

Together, they enable abstraction — allowing one function to operate on many different inputs without modification.

Practice Strategy
Refactor a function to replace multiple hardcoded values with parameters. Evaluate how flexibility improves.
