Async Programming — Switching Tasks Smartly

In this lesson, you will learn what asynchronous programming means. It is a way to handle many waiting tasks without using threads or processes.

1. The problem: waiting wastes time

Imagine your program needs to:

  • Download data from the internet
  • Wait for a server response
  • Wait for a file to load

These tasks take time. During that time, the program often just waits.

Simple idea
If a task is just waiting, the CPU is mostly free. We should let it do something else.

2. What is asynchronous programming?

Asynchronous programming (async programming) is a way to:

  • Start a task
  • Pause it while it waits
  • Let another task run
  • Come back later when ready

The program switches between tasks while they wait.

Analogy
Imagine a chef cooking pasta and soup. While the pasta boils, the chef prepares vegetables. The chef does not just stand and wait.

3. Blocking vs Non-Blocking

A blocking task stops the program until it finishes. 

import time

print("Start")
time.sleep(3)
print("End")

The program waits 3 seconds. Nothing else runs.

A non-blocking task allows the program to do something else while waiting.

Async programming uses non-blocking tasks.

4. The event loop

Async programs use something called an event loop.

The event loop:

  • Starts tasks
  • Checks which tasks are waiting
  • Switches to tasks that are ready
Simple Analogy
Think of the event loop as a manager. It decides which worker should work next.

5. A first async example

import asyncio

async def say_hello():
    print("Hello")
    await asyncio.sleep(2)
    print("World")

asyncio.run(say_hello())

New things here:

  • async def creates an async function.
  • await pauses the function while waiting.
  • asyncio.run() starts the event loop.

await asyncio.sleep(2) waits 2 seconds without blocking the whole program.

6. Why async is useful

Async programming is very useful for:

  • Web servers
  • Chat applications
  • Downloading many files
  • Calling many APIs

It is powerful when many tasks spend time waiting.

Important
Async does NOT use multiple CPU cores. It is about smart waiting, not heavy computing.

Tip
