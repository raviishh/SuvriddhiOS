What Are Threads?

A simple guide to threads in Python — how to run more than one task at the same time. Think of this as learning how to give your computer more than one job without making it panic.

1. Why threads exist

Normally, Python runs code one line at a time. It finishes one task before starting the next.

But what if one task is slow? For example, waiting 5 seconds for data from the internet. During those 5 seconds, the program does nothing.

Analogy
Imagine one worker in a shop. If the worker waits for a delivery, no other work gets done. But if there are two workers, one can wait while the other keeps working.

Threads allow your program to do multiple tasks at the same time — especially useful when tasks spend time waiting.

2. A blocking example

Let’s look at a simple program that waits. 

import time

def slow_task():
    print("Starting slow task...")
    time.sleep(3)
    print("Slow task finished.")

slow_task()
print("Program finished.")

Output:

Starting slow task... (3 second pause) Slow task finished. Program finished.

Notice: the program waits 3 seconds before doing anything else.

3. Creating a thread

Python has a module called threading. It lets us create a new thread. 

import threading
import time

def slow_task():
    print("Starting slow task...")
    time.sleep(3)
    print("Slow task finished.")

thread = threading.Thread(target=slow_task)
thread.start()

print("Program finished.")

What changed?

  • threading.Thread creates a new thread.
  • target=slow_task tells it what function to run.
  • thread.start() starts the thread.

Now the slow task runs in a separate thread.

Important
The main program does NOT wait automatically. It continues running.

4. Waiting for a thread — join()

Sometimes we want the main program to wait until the thread finishes. For that, we use join(). 

import threading
import time

def slow_task():
    print("Starting slow task...")
    time.sleep(3)
    print("Slow task finished.")

thread = threading.Thread(target=slow_task)
thread.start()

thread.join()

print("Program finished.")

join() tells the main program: "Wait here until the thread is done."

5. Two threads running together

Let’s run two tasks at the same time. 

import threading
import time

def task_one():
    for i in range(3):
        print("Task One:", i)
        time.sleep(1)

def task_two():
    for i in range(3):
        print("Task Two:", i)
        time.sleep(1)

t1 = threading.Thread(target=task_one)
t2 = threading.Thread(target=task_two)

t1.start()
t2.start()

t1.join()
t2.join()

print("All tasks finished.")

Output might look like:

Task One: 0 Task Two: 0 Task One: 1 Task Two: 1 Task One: 2 Task Two: 2 All tasks finished.

The order can change. Threads run independently.

6. When should you use threads?

Threads are useful when your program:

  • Waits for internet responses
  • Reads or writes files
  • Waits for user input

These are called I/O tasks (input/output tasks).

Remember
Threads are good when the program spends time waiting.
