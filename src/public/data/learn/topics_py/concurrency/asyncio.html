asyncio — The Engine Behind Async Programs

In this lesson, you will learn how asyncio works. It is the built-in Python library that manages async tasks and the event loop.

1. What is asyncio?

asyncio is a Python module for asynchronous programming.

It provides:

  • The event loop
  • Tools to create tasks
  • Ways to run many async functions together

In short:
asyncio manages async tasks.

Simple idea
If async functions are workers, asyncio is the manager that gives them turns.

2. The Event Loop (Deeper Look)

The event loop:

  • Starts tasks
  • Pauses tasks that are waiting
  • Resumes tasks when ready
  • Repeats this very quickly

This switching happens many times per second.

Important:
Async programs usually run on a single thread.

3. Creating tasks

Sometimes we want to schedule async functions manually.

import asyncio

async def task():
    print("Task started")
    await asyncio.sleep(2)
    print("Task finished")

async def main():
    t = asyncio.create_task(task())
    await t

asyncio.run(main())

asyncio.create_task() schedules a coroutine to run soon.

The event loop decides when it runs.

4. Running many tasks together

We can create multiple tasks and run them at the same time.

import asyncio

async def download(name, delay):
    print(f"Starting {name}")
    await asyncio.sleep(delay)
    print(f"Finished {name}")

async def main():
    task1 = asyncio.create_task(download("File A", 3))
    task2 = asyncio.create_task(download("File B", 1))
    task3 = asyncio.create_task(download("File C", 2))

    await task1
    await task2
    await task3

asyncio.run(main())

Even though they start together:

  • File B finishes first (1 second)
  • File C finishes second (2 seconds)
  • File A finishes last (3 seconds)

They overlap while waiting.

5. gather() vs create_task()

Using gather:

await asyncio.gather(
    download("A", 2),
    download("B", 1)
)

gather() runs multiple coroutines and waits for all of them.

Using create_task:

task = asyncio.create_task(download("A", 2))
await task

create_task() gives you more control. You can start tasks early and await them later.

6. When should you use asyncio?

Use asyncio when:

  • You have many network requests
  • You build web servers
  • You handle many users at once
  • Your program waits often

Do NOT use asyncio for:

  • Heavy CPU calculations
Clear Rule
Asyncio is for high-level waiting tasks. Multiprocessing is for heavy CPU tasks.

7. Final Summary of This Section

  • Threads → good for waiting tasks.
  • Multiprocessing → good for heavy CPU work.
  • GIL → limits CPU threads.
  • Async / asyncio → smart switching between waiting tasks.
Big Picture
Concurrency is about handling many tasks. Parallelism is about using many CPU cores. Choose the right tool.
