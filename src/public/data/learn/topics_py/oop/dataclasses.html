Dataclasses

Dataclasses are a shorter and cleaner way to create classes that mainly store data. They reduce repeated code.

1. The Problem with Normal Classes

Sometimes we create classes that only store data. 

class Student:
    def __init__(self, name, marks):
        self.name = name
        self.marks = marks

student1 = Student("Arjun", 85)
print(student1.name)

This works fine. But if we create many classes like this, we keep writing the same pattern:

  • Constructor
  • Attribute assignments
  • Sometimes a __str__ method

This becomes repetitive.

2. Introducing Dataclasses

Python provides a shortcut called dataclass.

We must first import it. 

from dataclasses import dataclass

@dataclass
class Student:
    name: str
    marks: int

student1 = Student("Arjun", 85)
print(student1.name)

Explanation:

  • @dataclass tells Python to generate special methods automatically.
  • We only define variable names and types.
  • Python automatically creates __init__ for us.

3. Automatic __str__ and More

Dataclasses also automatically create:

  • __init__
  • __repr__
  • Comparison methods (like ==)
student1 = Student("Arjun", 85)
print(student1)

The output will show the data clearly without writing extra code.

4. Default Values

We can also give default values. 

from dataclasses import dataclass

@dataclass
class Book:
    title: str
    author: str = "Unknown"

book1 = Book("Python Basics")
print(book1)

If we do not provide the author, it becomes "Unknown".

5. When to Use Dataclasses

Use dataclasses when:

  • Your class mainly stores data
  • You do not need complex logic inside the constructor
  • You want cleaner code
Important
If your class has complex behavior or many custom methods, a normal class may be better. Dataclasses are for simple data models.

6. Small Exercise

Create a dataclass called Car.

  • Add attributes brand (string).
  • Add attribute year (integer).
  • Add default value color = "Black".
  • Create two objects and print them.

7. What You Learned

You now understand:

  • Dataclasses reduce repeated code.
  • @dataclass generates methods automatically.
  • They are best for simple data containers.
Friendly Tip
If you find yourself writing many simple classes with only attributes, consider using dataclasses. They keep your code short and clean.
