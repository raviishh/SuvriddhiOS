Encapsulation

Encapsulation means protecting data inside a class. It controls how data is accessed and changed.

1. Why Protect Data?

Sometimes we do not want users of a class to change data directly.

For example, a bank account balance should not be changed freely.

Analogy
Think of a safe. You cannot touch the money inside directly. You must use the correct method to open it.

2. Public vs Private Variables

In Python:

  • Normal variables are public.
  • Variables starting with _ are “protected” (by convention).
  • Variables starting with __ are private (name-mangled).

Let us see a normal class first. 

class BankAccount:
    def __init__(self, balance):
        self.balance = balance

account = BankAccount(1000)
account.balance = -500   # This should not be allowed
print(account.balance)

Here, we changed the balance to a negative number. This is unsafe.

3. Making a Variable Private

We can use double underscore __ to make a variable private. 

class BankAccount:
    def __init__(self, balance):
        self.__balance = balance

account = BankAccount(1000)
print(account.__balance)

This will cause an error. Python hides the variable.

The variable still exists, but it cannot be accessed directly.

4. Using Methods to Control Access

Instead of direct access, we use methods. 

class BankAccount:
    def __init__(self, balance):
        self.__balance = balance

    def deposit(self, amount):
        if amount > 0:
            self.__balance += amount

    def get_balance(self):
        return self.__balance

account = BankAccount(1000)
account.deposit(500)

print(account.get_balance())

Explanation:

  • __balance is private.
  • deposit() safely changes the balance.
  • get_balance() allows reading the balance.

Now the data is protected.

5. Adding Safety Rules

We can add rules inside methods. 

def withdraw(self, amount):
    if 0 < amount <= self.__balance:
        self.__balance -= amount
    else:
        print("Invalid withdrawal")

This prevents wrong operations.

Important Idea
Encapsulation does not hide data completely. It controls how data is changed.

6. Small Exercise

Create a class called Temperature.

  • Add a private variable __celsius.
  • Add a method set_temperature(self, value).
  • Do not allow temperatures below -273.
  • Add a method to return the temperature.

7. What You Learned

You now understand:

  • Encapsulation protects data inside a class.
  • Private variables use __.
  • Methods control how data is accessed and changed.
  • This prevents unsafe operations.

Friendly Tip
Good class design means controlling data carefully. Do not allow users to change important values directly. Always use methods to protect your logic.
