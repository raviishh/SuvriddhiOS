Inheritance

Inheritance allows one class to reuse code from another class. It helps us avoid repeating the same code again and again.

1. What is Inheritance?

Inheritance means one class can take properties and methods from another class.

The original class is called the parent class. The new class is called the child class.

Analogy
A child inherits traits from parents. For example, eye color or height. In programming, a child class inherits data and behavior.

2. Basic Example

class Animal:
    def speak(self):
        print("This animal makes a sound")

class Dog(Animal):
    pass

dog1 = Dog()
dog1.speak()

Explanation:

  • Animal is the parent class.
  • Dog(Animal) means Dog inherits from Animal.
  • dog1 can use the speak() method.

Even though Dog does not define speak(), it can still use it.

3. Adding More to the Child Class

A child class can also have its own methods. 

class Animal:
    def speak(self):
        print("This animal makes a sound")

class Dog(Animal):
    def fetch(self):
        print("The dog is fetching the ball")

dog1 = Dog()
dog1.speak()
dog1.fetch()

Now Dog has:

  • The inherited speak() method
  • Its own fetch() method

4. Overriding a Parent Method

A child class can change a method from the parent class. This is called overriding. 

class Animal:
    def speak(self):
        print("This animal makes a sound")

class Dog(Animal):
    def speak(self):
        print("Woof!")

dog1 = Dog()
dog1.speak()

Now Dog has its own version of speak(). The parent version is replaced.

5. Using super()

Sometimes we want to use both the parent method and add new behavior. 

class Animal:
    def speak(self):
        print("This animal makes a sound")

class Dog(Animal):
    def speak(self):
        super().speak()
        print("Woof!")

dog1 = Dog()
dog1.speak()

Explanation:

  • super() calls the parent method.
  • Then the child adds extra behavior.

6. Why Use Inheritance?

Inheritance:

  • Reduces repeated code
  • Makes programs easier to expand
  • Creates logical relationships between classes

Simple Rule
Use inheritance when there is an “is-a” relationship. A Dog is an Animal. A Car is not an Animal.

7. Small Exercise

Create a parent class called Vehicle.

  • Add a method start() that prints "Vehicle started".
  • Create a child class Car.
  • Override start() to print "Car started".
  • Create an object and test it.

8. What You Learned

You now understand:

  • A child class inherits from a parent class.
  • The child can add new methods.
  • The child can override parent methods.
  • super() calls the parent version.

Friendly Tip
Do not use inheritance just to avoid typing. Use it when there is a real logical relationship. Good design matters.
