Classes & Objects

A simple introduction to one of the most important ideas in Python. Classes help us model real-world things inside a program.

1. What is a Class?

A class is a blueprint. It describes what something is and what it can do.

Analogy
Think about a cookie cutter. The cutter is the class. The cookies made from it are objects.

The class does not create anything by itself. It only defines the structure. 

class Dog:
    pass

Explanation:

  • class Dog: creates a new class named Dog.
  • pass means “do nothing for now.”

2. What is an Object?

An object is a real thing created from a class.

If a class is the blueprint, an object is the actual building. 

class Dog:
    pass

dog1 = Dog()
dog2 = Dog()

print(dog1)
print(dog2)

Explanation:

  • Dog() creates a new object.
  • dog1 and dog2 are two different objects.
  • Each object lives in a different place in memory.

Even though they come from the same class, they are separate objects.

3. Giving Objects Data (Attributes)

Objects can store information. This information is called an attribute. 

class Dog:
    pass

dog1 = Dog()
dog1.name = "Buddy"
dog1.age = 3

print(dog1.name)
print(dog1.age)

Explanation:

  • dog1.name creates an attribute called name.
  • dog1.age stores the dog's age.
  • Attributes belong to that specific object.

If we create another dog, it will not automatically have these values. 

dog2 = Dog()
print(dog2.name)

This will cause an error because dog2 does not have a name yet.

4. Why Use Classes?

Classes help organize code.

Instead of using many separate variables like: 

dog1_name = "Buddy"
dog1_age = 3

We group related data together inside an object.

Real-World Thinking
In real life, a dog has a name, age, color, and more. Using classes lets us model real-world objects clearly.

5. Small Exercise

Create a class called Car.

Then:

  • Print them.

6. What You Learned

You now understand:

  • A class is a blueprint.
  • An object is created from a class.
  • Objects can store data using attributes.
