Constructors (__init__)

In this lesson, you will learn how objects are prepared when they are created. The special method __init__ helps us set up data automatically.

1. What is a Constructor?

A constructor is a special method that runs when an object is created.

In Python, the constructor is named __init__.

Simple Idea
When you create something, you often fill in details. The constructor is where we fill in those details.

2. Basic Example

Without a constructor, we must manually add attributes. 

class Book:
    pass

book1 = Book()
book1.title = "Python Basics"
book1.author = "Aisha"

print(book1.title)

This works, but it is not clean.

Now let us use a constructor. 

class Book:
    def __init__(self, title, author):
        self.title = title
        self.author = author

book1 = Book("Python Basics", "Aisha")

print(book1.title)
print(book1.author)

Explanation:

  • __init__ runs automatically when we create the object.
  • self.title = title stores the value inside the object.
  • We pass values inside Book(...).

3. Why Use __init__?

It makes sure every object starts with correct data.

For example: 

book2 = Book("Data Science", "Rahul")

Every book must have a title and an author. The constructor forces this rule.

Important
If you do not provide the required values, Python will give an error. This protects your program.

4. Default Values

We can give default values to parameters. 

class Book:
    def __init__(self, title, author="Unknown"):
        self.title = title
        self.author = author

book1 = Book("Python Basics")
print(book1.author)

If no author is given, it becomes "Unknown".

This makes objects more flexible.

5. Constructor + Methods Together

Constructors work together with methods. 

class Student:
    def __init__(self, name, marks):
        self.name = name
        self.marks = marks

    def show_details(self):
        print("Name:", self.name)
        print("Marks:", self.marks)

student1 = Student("Arjun", 85)
student1.show_details()

The constructor sets up the data. The method uses that data.

  • Create two car objects with different values.

7. What You Learned

You now understand:

  • __init__ is the constructor method.
  • It runs automatically when an object is created.
  • It sets up the object's initial data.
  • It can take parameters and default values.

Friendly Tip
Most classes use a constructor. If your objects need starting data, always use __init__. It keeps your code clean and organized.
