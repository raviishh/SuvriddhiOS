Polymorphism

Polymorphism means “many forms.” In Python, it allows different objects to use the same method name in different ways.

1. Same Method Name, Different Behavior

Different classes can define the same method. But each class can give it different behavior.

Analogy
Different people can “speak.” A baby speaks differently from a teacher. The action is the same. The behavior is different. 
class Dog:
    def speak(self):
        print("Woof!")

class Cat:
    def speak(self):
        print("Meow!")

dog = Dog()
cat = Cat()

dog.speak()
cat.speak()

Both objects use speak(), but the output is different.

2. Polymorphism with a Function

Polymorphism becomes powerful when we treat different objects the same way. 

def make_sound(animal):
    animal.speak()

dog = Dog()
cat = Cat()

make_sound(dog)
make_sound(cat)

Explanation:

  • The function does not care if it receives a Dog or a Cat.
  • It only expects the object to have a speak() method.

This is called duck typing.

Duck Typing
“If it looks like a duck and acts like a duck, it is a duck.” If an object has the right method, Python allows it.

3. Polymorphism with Inheritance

Polymorphism often works together with inheritance. 

class Animal:
    def speak(self):
        print("This animal makes a sound")

class Dog(Animal):
    def speak(self):
        print("Woof!")

class Cat(Animal):
    def speak(self):
        print("Meow!")

animals = [Dog(), Cat()]

for animal in animals:
    animal.speak()

Each object responds differently to the same method call.

4. Built-in Polymorphism

Polymorphism also exists in Python’s built-in functions. 

print(len("Hello"))
print(len([1, 2, 3, 4]))

Explanation:

  • len() works on strings.
  • It also works on lists.
  • The function behaves correctly for different types.

One function. Many forms.

5. Why Polymorphism is Useful

Polymorphism:

  • Makes code flexible
  • Reduces special conditions
  • Allows easier expansion

Instead of writing: 

if type(animal) == Dog:
    ...
elif type(animal) == Cat:
    ...

We just call: 

animal.speak()

Cleaner code. Less checking.

6. Small Exercise

Create three classes:

  • Car with method move() that prints "Car is driving".
  • Boat with method move() that prints "Boat is sailing".
  • Plane with method move() that prints "Plane is flying".

Create a list of these objects. Use a loop to call move() on each one.

7. What You Learned

You now understand:

  • Polymorphism means one method name can behave differently.
  • Objects can be treated the same if they share method names.
  • Duck typing allows flexible design.
  • Polymorphism reduces complex condition checks.

Friendly Tip
If your code has many type checks, you might not be using polymorphism well. Let objects handle their own behavior.
