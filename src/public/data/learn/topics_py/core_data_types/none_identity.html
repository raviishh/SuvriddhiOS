NoneType & Identity (is vs ==)

Sometimes the most important value in a program is the absence of a value. That’s where None comes in. And once you understand None, you must also understand the critical difference between identity and equality.

1. What is None?

None represents the absence of a value. It is its own type: NoneType. 

x = None

print(type(x))   # <class 'NoneType'>

Important:

  • None is not 0
  • It is not an empty string
  • It is not False
It literally means “nothing here.”

Analogy
Think of None as an empty parking spot. The spot exists. But there is no car in it.

2. When does None appear?

Functions return None by default if they don’t explicitly return something. 

def greet():
    print("Hello")

result = greet()
print(result)  # None

Many built-in methods also return None because they modify objects instead of producing new values.

3. Checking for None (the correct way)

To check for None, use is, not ==. 

value = None

if value is None:
    print("No value found")

Why? Because None is a single unique object in Python.

Rule
Always use is None Never use == None

4. Equality (==) — comparing values

The == operator checks if two objects have the same value. 

a = [1, 2, 3]
b = [1, 2, 3]

print(a == b)   # True

Even though they are separate lists, their contents are equal.

5. Identity (is) — comparing memory location

The is operator checks whether two variables refer to the same object in memory. 

a = [1, 2, 3]
b = [1, 2, 3]
c = a

print(a is b)   # False
print(a is c)   # True

a and b have equal values, but they are different objects. a and c point to the same object.

6. Why this distinction matters

Confusing is and == leads to subtle bugs. 

x = 1000
y = 1000

print(x == y)  # True
print(x is y)  # Sometimes False

Small integers may appear identical due to internal optimization, but this is an implementation detail — not something you should rely on.

Mental Model
== asks: “Do these look the same?” is asks: “Are these literally the same object?”

7. Truthiness of None

None evaluates to False in Boolean contexts. 

value = None

if not value:
    print("This runs")

But be careful: Checking if not value does NOT specifically check for None. It also catches 0, "", empty lists, etc.

8. Common beginner mistakes

  • Using == None instead of is None
  • Using is to compare strings or numbers
  • Confusing equality with identity
  • Overusing if not value when specifically checking for None
Quick Check
What does this print? a = [] b = [] print(a == b) print(a is b)

9. Where this fits in Core Data Types

None represents absence. == checks value equality. is checks identity.

This distinction becomes crucial when working with mutable objects — which is exactly what we explore next in Mutable vs Immutable Deep Dive.

Practice Strategy
Create variables. Compare them with == and is. Use id() to inspect memory identity. Build intuition by experimenting.
