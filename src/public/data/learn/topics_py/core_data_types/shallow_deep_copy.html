Shallow vs Deep Copy

Copying data sounds simple — until it isn’t. With mutable objects, copying can either duplicate the container only… or duplicate everything inside it. That difference is the line between predictable code and subtle bugs.

1. The core problem

Consider this list:

original = [[1, 2], [3, 4]]
copy = original

This is not a copy. Both variables point to the same object in memory. 

copy[0][0] = 99
print(original)  # [[99, 2], [3, 4]]

Changing one changes the other. This is aliasing — not copying.

2. Shallow copy — copying the outer container

A shallow copy creates a new outer object, but the inner objects are still shared. 

import copy

original = [[1, 2], [3, 4]]
shallow = copy.copy(original)

shallow[0][0] = 99

print(original)  # [[99, 2], [3, 4]]

The outer list is new, but the inner lists are still the same objects.

You can also create shallow copies using:

shallow = original[:]
shallow = list(original)
Mental Model
Shallow copy = new box, same contents inside.

3. Deep copy — copying everything recursively

A deep copy duplicates the object and all nested objects inside it. 

import copy

original = [[1, 2], [3, 4]]
deep = copy.deepcopy(original)

deep[0][0] = 99

print(original)  # [[1, 2], [3, 4]]

Now the structures are completely independent.

Mental Model
Deep copy = new box, new contents, new everything.

4. Visualizing the difference

Original structure:

original → [ A , B ]
             ↓   ↓
            [1,2] [3,4]

Shallow copy:

original → [ A , B ]
shallow  → [ A , B ]   (same inner lists)

Deep copy:

original → [ A , B ]
deep     → [ C , D ]   (completely new inner lists)

That’s the entire difference.

5. When do you need deep copy?

  • Working with nested lists or dictionaries
  • Creating backups of data structures
  • Avoiding side effects in functions

If your data structure contains only immutable values (like ints or strings), shallow copy is usually enough.

6. Performance considerations

Deep copying is slower and uses more memory because it duplicates everything.

Don’t use deep copy automatically. Use it when necessary.

7. Common beginner mistakes

  • Thinking new_list = old_list creates a copy
  • Using shallow copy for nested structures
  • Overusing deep copy without understanding performance impact
  • Not understanding how identity relates to copying
Quick Check
What happens here? a = [[1], [2]] b = a[:] b[0].append(99) print(a)

8. Where this fits in Core Data Types

Copying connects directly to mutability and identity. If you don’t understand how objects are stored in memory, shallow vs deep copy will confuse you.
