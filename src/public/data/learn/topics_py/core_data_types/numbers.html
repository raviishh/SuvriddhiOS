Numbers (int, float, complex)

Numbers are the backbone of programming. Whether you're counting users, calculating profit, or simulating physics — it all starts with numeric data. In Python, numbers come in three core types: int, float, and complex.

1. Integers — whole numbers

An int represents whole numbers. No decimal point. Just clean, solid values. 

x = 10
y = -3
z = 0

print(type(x))  # <class 'int'>

Integers can be positive, negative, or zero. Python supports arbitrarily large integers — meaning they don’t “overflow” the way they might in lower-level languages.

Analogy
Think of int like counting books on a shelf. You can have 5 books. Not 5.7 books. Whole objects → whole numbers.

2. Floats — numbers with decimals

A float stores decimal numbers (technically, floating-point numbers). 

price = 19.99
temperature = -4.5

print(type(price))  # <class 'float'>

Floats are used when precision matters — measurements, money, scientific calculations.

Important Reality
Floats are not perfectly precise. Due to how computers store decimal numbers in binary, some results may look slightly “off.” 
print(0.1 + 0.2)  
# 0.30000000000000004

That tiny error isn't a bug — it's a representation issue. Later lessons will show how to handle precision properly.

3. Complex numbers — real + imaginary

Python also supports complex numbers. These have a real part and an imaginary part. 

z = 2 + 3j

print(type(z))       # <class 'complex'>
print(z.real)        # 2.0
print(z.imag)        # 3.0

The j represents the imaginary unit (√-1). Complex numbers are heavily used in advanced math, physics, and engineering.

When would you use this?
Signal processing, electrical engineering, simulations, or advanced mathematics. If you're just building web apps — probably not daily. But it's powerful that Python includes it natively.

4. Basic arithmetic operations

Python supports standard mathematical operators: 

a = 10
b = 3

print(a + b)   # Addition → 13
print(a - b)   # Subtraction → 7
print(a * b)   # Multiplication → 30
print(a / b)   # Division → 3.333...
print(a // b)  # Floor division → 3
print(a % b)   # Modulus → 1
print(a ** b)  # Exponent → 1000

Key distinction:

  • / always returns a float
  • // discards the decimal (floor division)
  • % gives the remainder

5. Mixing number types

Python automatically promotes types when needed. 

result = 5 + 2.0
print(result)        # 7.0
print(type(result))  # float

When an int interacts with a float, the result becomes a float. This prevents accidental data loss.

6. Useful built-in numeric functions

Python includes built-in functions that work with numbers: 

print(abs(-10))     # 10
print(round(3.1415, 2))  # 3.14
print(pow(2, 3))    # 8

These utilities help you avoid reinventing the wheel.

7. Common beginner mistakes

  • Expecting perfect float precision
  • Confusing / and //
  • Forgetting that division always produces a float
  • Using strings instead of numbers in calculations
Quick Check
What is the type of 5 / 2? What about 5 // 2? Run it. Don’t guess.
