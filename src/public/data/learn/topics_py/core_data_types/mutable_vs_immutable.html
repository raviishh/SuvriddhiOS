Mutable vs Immutable Deep Dive

This is one of the most important concepts in Python. If you truly understand mutability, you understand how Python stores and manages data in memory.

1. What does “mutable” mean?

A mutable object can be changed after it is created. 

numbers = [1, 2, 3]
numbers.append(4)

print(numbers)  # [1, 2, 3, 4]

The list changed — same object, new content.

2. What does “immutable” mean?

An immutable object cannot be changed after it is created. Any “change” creates a new object. 

x = 10
print(id(x))

x = x + 5
print(id(x))  # Different memory location

The original integer wasn’t modified. A new integer object was created.

Core Idea
Immutable → cannot change the object Mutable → can change the object

3. Common immutable types

  • int
  • float
  • bool
  • str
  • tuple
  • NoneType

4. Common mutable types

  • list
  • dict
  • set
  • Most custom objects

5. Why mutability matters (the aliasing problem)

Consider this:

a = [1, 2, 3]
b = a

b.append(4)
print(a)  # [1, 2, 3, 4]

Both variables refer to the same object. Modifying one modifies the other.

With immutable types:

x = 10
y = x

y = y + 5
print(x)  # 10

Because integers are immutable, reassignment creates a new object.

6. Strings — immutable but tricky

Strings look mutable when you “modify” them — but they are not.

s = "hello"
s = s.upper()

print(s)  # HELLO

upper() returns a new string. The original string is untouched.

7. Function side effects

Mutable objects can be modified inside functions.

def add_item(lst):
    lst.append(99)

my_list = [1, 2]
add_item(my_list)

print(my_list)  # [1, 2, 99]

This is powerful — and dangerous if unintended.

Professional Insight
Many bugs in real systems come from unintended mutations.

8. The tuple twist

Tuples are immutable — but they can contain mutable objects.

t = ([1, 2], 3)
t[0].append(99)

print(t)  # ([1, 2, 99], 3)

The tuple itself didn’t change — but the list inside it did.

9. Common beginner mistakes

  • Thinking assignment creates a copy
  • Forgetting that lists modify in-place
  • Not understanding why strings behave differently from lists
  • Unexpected function side effects

10. Why this concept is foundational

Mutability affects copying, function design, debugging, performance, and memory usage.

If you understand mutability deeply, you can reason about complex Python programs with confidence.
