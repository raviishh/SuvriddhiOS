Booleans & Truthiness

Booleans power decision-making. Every conditional, every loop guard, every validation check ultimately depends on one question: is this True or False?

1. What is a Boolean?

A Boolean is a data type with only two possible values: True and False. 

is_logged_in = True
is_admin = False

print(type(is_logged_in))  # <class 'bool'>

Note: True and False must be capitalized. They are special keywords.

Analogy
Think of a Boolean as a light switch. It’s either ON or OFF. No in-between.

2. Comparison operators

Booleans often come from comparisons. 

x = 10
y = 5

print(x > y)    # True
print(x < y)    # False
print(x == 10)  # True
print(x != y)   # True
print(x >= 10)  # True

Common comparison operators:

  • == equal to
  • != not equal to
  • >, <
  • >=, <=

Common Mistake
= assigns a value. == compares values. Mixing them up causes logic errors.

3. Logical operators

You can combine Boolean expressions using logical operators. 

age = 20
has_id = True

print(age >= 18 and has_id)  # True
print(age < 18 or has_id)    # True
print(not has_id)            # False

Logical operators:

  • and → both conditions must be True
  • or → at least one must be True
  • not → reverses the result

4. Booleans in conditionals

Booleans are most commonly used inside if statements. 

temperature = 30

if temperature > 25:
    print("It's warm!")
else:
    print("It's cool!")

The if block runs only if the condition evaluates to True.

5. Truthiness — not everything is strictly True or False

In Python, many values can be evaluated as True or False even if they are not Boolean types. This is called truthiness. 

print(bool(0))        # False
print(bool(1))        # True
print(bool(""))       # False
print(bool("Hello"))  # True
print(bool([]))       # False
print(bool([1, 2]))   # True

Generally:

  • Zero values → False
  • Empty collections → False
  • Everything else → True

Why this matters
You don’t always need to write if len(my_list) > 0. You can simply write if my_list:

6. Short-circuit evaluation

Python evaluates logical expressions from left to right and stops as soon as the result is determined. 

print(False and print("Hello"))  # "Hello" never runs
print(True or print("World"))    # "World" never runs

This is called short-circuiting. It’s efficient — and sometimes surprising.

7. Common beginner mistakes

  • Using = instead of ==
  • Forgetting operator precedence in complex conditions
  • Overcomplicating logic instead of simplifying conditions
  • Not understanding truthiness of empty values
Quick Check
What does this print? bool("False") Run it. Think about why.
