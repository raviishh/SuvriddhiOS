Strings Deep Dive

Strings are how programs work with text. Names, messages, passwords, file paths, user input — all of it lives inside strings. If numbers are for calculation, strings are for communication.

1. What is a string?

A string is a sequence of characters wrapped in quotes. 

name = "Alice"
message = 'Hello, world!'
empty = ""

print(type(name))  # <class 'str'>

You can use double quotes (") or single quotes ('). Python treats them the same.

Analogy
Think of a string as a train made of characters. Each letter is a carriage, connected in order.

2. Indexing — accessing characters

Each character in a string has a position called an index. Indexing starts at 0. 

word = "Python"

print(word[0])  # P
print(word[1])  # y
print(word[-1]) # n (last character)

Negative indexing counts from the end.

3. Slicing — extracting parts

Slicing lets you extract a portion of a string. 

text = "Programming"

print(text[0:6])   # Progra
print(text[3:])    # gramming
print(text[:4])    # Prog
print(text[::2])   # Pormig

Format: string[start:end:step] The end index is not included.

4. Strings are immutable

Strings cannot be changed after creation. 

name = "Alice"
# name[0] = "M"   ❌ This will cause an error

Instead, you create a new string: 

name = "Alice"
new_name = "M" + name[1:]
print(new_name)  # Mlice
Why this matters
Immutability makes strings safer and more predictable. We’ll revisit this concept in the Mutability lesson.

5. String methods — built-in tools

Strings come with many helpful methods. 

text = "  Hello World  "

print(text.lower())      # hello world
print(text.upper())      # HELLO WORLD
print(text.strip())      # removes spaces
print(text.replace("World", "Python"))
print(len(text))         # length of string

Methods don’t change the original string — they return a new one.

6. Concatenation & repetition

You can combine strings using +. 

first = "Hello"
second = "World"

print(first + " " + second)  # Hello World
print("Ha" * 3)              # HaHaHa

Repetition uses the * operator.

7. f-Strings — modern formatting

The cleanest way to insert variables into strings is using f-strings. 

name = "Alice"
age = 20

print(f"My name is {name} and I am {age} years old.")

Inside {}, you can even write expressions: 

print(f"Next year I will be {age + 1}.")

f-strings are readable, powerful, and the recommended approach.

8. Escape characters

Some characters need special handling using a backslash \. 

print("Line 1\nLine 2")
print("He said \"Hello\"")
print("C:\\Users\\Alice")

Common escapes:

  • \n → new line
  • \t → tab
  • \" → quote
  • \\ → backslash

9. Common beginner mistakes

  • Trying to modify a string directly
  • Forgetting indexing starts at 0
  • Mixing numbers and strings without conversion
  • Using + instead of f-strings for complex formatting
Quick Check
What does "Python"[::-1] return? Run it. Understand it.

10. Where this fits in Core Data Types

Strings are immutable sequences of characters. They are foundational for user interaction, file handling, web development, and almost every real-world program.
