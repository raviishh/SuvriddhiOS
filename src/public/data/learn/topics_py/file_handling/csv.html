Working with CSV Files in Python

CSV files power spreadsheets, datasets, exports, analytics pipelines, and dashboards. If text files are notebooks, CSV files are structured tables. This lesson teaches you how to read and write them properly.

1. What is a CSV file?

CSV stands for Comma-Separated Values. It stores tabular data in plain text. Each line represents a row. Commas separate columns. 

name,age,score
Alice,17,92
Bob,18,85
Charlie,17,88

Spreadsheet programs like Excel and Google Sheets export and import CSV files easily. Python handles them using the built-in csv module.

Key Insight
CSV is simple but strict. Commas, quotes, and line breaks must be handled correctly. Never parse CSV manually using split(",") in production code.

2. Reading a CSV file

Use the csv.reader object to iterate through rows. 

import csv

with open("students.csv", "r", newline="") as file:
    reader = csv.reader(file)
    for row in reader:
        print(row)

Each row is returned as a list of strings.

Why newline=""? It prevents blank-line issues on certain systems. This is recommended best practice in Python’s documentation.

3. Skipping headers

Many CSV files include a header row. 

import csv

with open("students.csv", "r", newline="") as file:
    reader = csv.reader(file)
    header = next(reader)  # Skip header
    
    for row in reader:
        print(row)

next(reader) advances the iterator once.

4. Using DictReader (Cleaner approach)

Instead of lists, csv.DictReader returns each row as a dictionary. Column names become keys. 

import csv

with open("students.csv", "r", newline="") as file:
    reader = csv.DictReader(file)
    
    for row in reader:
        print(row["name"], row["score"])

This is safer and more readable for structured datasets.

Professional Practice
Use DictReader whenever your CSV includes headers. It reduces indexing mistakes.

5. Writing to a CSV file

Basic writer

import csv

with open("output.csv", "w", newline="") as file:
    writer = csv.writer(file)
    
    writer.writerow(["name", "age", "score"])
    writer.writerow(["Alice", 17, 92])
    writer.writerow(["Bob", 18, 85])

Each writerow() call writes one row.

Using DictWriter

import csv

fieldnames = ["name", "age", "score"]

with open("output.csv", "w", newline="") as file:
    writer = csv.DictWriter(file, fieldnames=fieldnames)
    
    writer.writeheader()
    writer.writerow({"name": "Alice", "age": 17, "score": 92})
    writer.writerow({"name": "Bob", "age": 18, "score": 85})

writeheader() automatically writes column names.

6. Common pitfalls

  • Forgetting newline="" → extra blank rows
  • Assuming numeric types — CSV reads everything as strings
  • Manually splitting strings instead of using the csv module
  • Ignoring encoding (UTF-8 is usually safe)
# Converting string values to integers
age = int(row["age"])
score = float(row["score"])

CSV remains a universal exchange format — lightweight, readable, and tool-compatible. It is foundational in data engineering and analytics workflows.
