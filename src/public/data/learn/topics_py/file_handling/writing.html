Writing Text Files in Python

Reading is passive. Writing is power. In this lesson, you’ll learn how Python stores new data on disk — safely, intentionally, and without accidentally deleting everything.

1. Why writing matters

Writing files is how programs save results, logs, user input, reports, and generated data. If reading is opening the cabinet, writing is placing new documents inside.

Reality Check
Writing can overwrite existing data. One careless mode choice can wipe a file. Professionals treat file writing carefully.

2. File modes — understand before you type

When opening a file for writing, the mode determines behavior:

  • "w" → Write (creates file or overwrites existing file)
  • "a" → Append (adds to end of file)
  • "x" → Create (fails if file already exists)
Most Common Mistake
Using "w" on an important file. It deletes existing content immediately.

3. Basic writing with "w"

with open("output.txt", "w") as file:
    file.write("Hello, world!\n")
    file.write("This is a new file.\n")

Explanation:

  • If output.txt does not exist, Python creates it.
  • If it exists, its previous contents are erased.
  • \n creates a new line.

4. Appending instead of overwriting

To preserve existing data and add new content, use append mode. 

with open("output.txt", "a") as file:
    file.write("Adding another line.\n")

This adds content at the end without deleting previous lines.

Use Case
Logging systems use append mode so old log entries remain intact.

5. Writing multiple lines efficiently

Instead of calling write() repeatedly, you can write multiple lines at once. 

lines = ["First line\n", "Second line\n", "Third line\n"]

with open("output.txt", "w") as file:
    file.writelines(lines)

Important:

  • writelines() does NOT add newlines automatically.
  • You must include \n yourself.

6. Writing formatted data

Often you’ll write computed values to files. 

name = "Alice"
score = 92

with open("report.txt", "w") as file:
    file.write(f"Student: {name}\n")
    file.write(f"Score: {score}\n")

Using f-strings keeps your file output clean and readable.

7. Safe writing with error handling

Files may fail to write due to permission issues or disk errors. Defensive programming prevents crashes. 

try:
    with open("output.txt", "w") as file:
        file.write("Safe writing example.\n")
except PermissionError:
    print("You do not have permission to write to this file.")
except OSError:
    print("A system-related error occurred.")
