Reading Text Files in Python

Learn how to read data from text files in Python — clearly, safely, and efficiently. If variables are boxes in memory, files are storage rooms on disk. This lesson teaches you how to open the door and look inside.

1. Why read from files?

Programs rarely run in isolation. Real software loads configuration files, reads logs, processes datasets, and consumes user-generated content. File input is how your program interacts with persistent data — information that survives after the program exits.

Analogy
Think of RAM as your desk — temporary workspace. A file is the filing cabinet. When the computer shuts down, your desk is cleared. The cabinet remains.

Reading files correctly is foundational for data science, backend systems, automation scripts, and even embedded software.

2. Opening a file — the open() function

Python provides the built-in function open() to access files. To read a text file, we use mode "r". 

file = open("example.txt", "r")
content = file.read()
print(content)
file.close()

Explanation:

  • open("example.txt", "r") opens the file in read mode.
  • read() loads the entire file as one string.
  • close() releases the file resource.

Important
Always close files. Leaving them open wastes system resources and can cause bugs.

3. The better way — context managers (with)

Manually closing files is error-prone. If an exception occurs, close() might never run. Python solves this with context managers. 

with open("example.txt", "r") as file:
    content = file.read()
    print(content)

Once the block ends, Python automatically closes the file.

Best Practice
Use with for all file operations. In professional codebases, manual open/close is discouraged.

4. Reading strategies — choosing the right method

Python offers multiple ways to read file contents. Each serves a different use case.

1. Read entire file

with open("example.txt", "r") as file:
    data = file.read()

Best for small files.

2. Read line by line (memory efficient)

with open("example.txt", "r") as file:
    for line in file:
        print(line.strip())

Ideal for large files.

3. Read all lines into a list

with open("example.txt", "r") as file:
    lines = file.readlines()

Returns a list where each element is one line.

5. Understanding file paths

Python looks for files relative to the current working directory unless given an absolute path. 

# Relative path
open("data/info.txt")

# Absolute path (example)
open("/home/user/data/info.txt")

If you see FileNotFoundError, verify:

  • The file exists
  • The path is correct
  • You have read permissions

6. Handling errors safely

Production-grade code anticipates failure. Files may not exist, permissions may be restricted, or encoding issues may occur. 

try:
    with open("example.txt", "r") as file:
        print(file.read())
except FileNotFoundError:
    print("File not found.")
except PermissionError:
    print("Permission denied.")

This prevents your program from crashing unexpectedly.
