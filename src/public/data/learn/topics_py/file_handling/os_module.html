File Paths and the os Module

Files don’t float in space. They live in directories. If file handling is persistence, paths are navigation. This lesson teaches you how Python locates files and how to manage the filesystem safely.

1. Understanding file paths

A file path tells Python exactly where a file is located. There are two main types:

  • Relative path — based on current working directory
  • Absolute path — full path from the root of the system
# Relative
open("data.txt")

# Absolute (Linux/macOS example)
open("/home/user/projects/data.txt")

# Absolute (Windows example)
open("C:\\Users\\User\\projects\\data.txt")
Reality Check
Most “FileNotFoundError” issues happen because the working directory is not what you think it is.

2. The current working directory

Python executes relative paths based on the current working directory. You can check it using os.getcwd(). 

import os

print(os.getcwd())

To change it: 

os.chdir("new_folder")

Changing directories mid-program should be done carefully — it affects all relative paths afterward.

3. Building paths safely

Never manually combine paths with string concatenation. Different operating systems use different separators. 

import os

folder = "data"
filename = "info.txt"

path = os.path.join(folder, filename)
print(path)

os.path.join() automatically inserts the correct separator.

Best Practice
Avoid: folder + "/" + filename
Prefer: os.path.join()

4. Checking file and directory existence

import os

if os.path.exists("data.txt"):
    print("File exists")

if os.path.isfile("data.txt"):
    print("It is a file")

if os.path.isdir("data_folder"):
    print("It is a directory")

This prevents runtime errors and makes your code more robust.

5. Creating and removing directories

import os

# Create directory
os.mkdir("new_folder")

# Create nested directories
os.makedirs("parent/child/grandchild")

# Remove empty directory
os.rmdir("new_folder")

Important:

  • mkdir() creates one level
  • makedirs() creates nested paths
  • Directories must be empty before removal

6. Listing directory contents

import os

files = os.listdir(".")

for file in files:
    print(file)

"." represents the current directory.

7. A note on pathlib

Modern Python increasingly favors pathlib for object-oriented path handling. 

from pathlib import Path

path = Path("data") / "info.txt"

if path.exists():
    print("File exists")

pathlib is often cleaner and more expressive than os.path.

8. Why this matters

You now understand:

  • Relative vs absolute paths
  • How to inspect and change working directories
  • How to build cross-platform paths safely
  • How to create, check, and manage directories

Filesystem awareness is not optional. Scripts break in production when developers hard-code paths incorrectly.
