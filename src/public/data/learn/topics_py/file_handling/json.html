JSON Parsing & Writing in Python

JSON is the language of APIs, web apps, configs, and modern data exchange. If CSV is a spreadsheet, JSON is a structured data tree. This lesson shows you how to read it, manipulate it, and write it back safely.

1. What is JSON?

JSON stands for JavaScript Object Notation. It stores structured data using key–value pairs and arrays. 

{
  "name": "Alice",
  "age": 17,
  "subjects": ["Math", "Physics", "Economics"],
  "active": true
}

JSON maps naturally to Python data types:

  • Object → dict
  • Array → list
  • String → str
  • Number → int / float
  • true / false → True / False
  • null → None
Key Insight
JSON is text. Python objects are in memory. Parsing converts text → Python objects. Writing converts Python objects → JSON text.

2. Reading (Parsing) JSON from a file

Use the built-in json module. 

import json

with open("data.json", "r") as file:
    data = json.load(file)

print(data)
print(data["name"])

json.load() reads the file and converts JSON into a Python object.

3. Parsing JSON from a string

If JSON is received from an API or stored as a string: 

import json

json_string = '{"name": "Alice", "age": 17}'
data = json.loads(json_string)

print(data["age"])

loads() parses a JSON string. Think: load → file, loads → string.

4. Writing JSON to a file

To save Python data as JSON: 

import json

data = {
    "name": "Alice",
    "age": 17,
    "subjects": ["Math", "Physics"]
}

with open("output.json", "w") as file:
    json.dump(data, file)

json.dump() writes Python data as JSON text.

5. Pretty printing JSON

By default, JSON is written in compact format. For readability, use indentation. 

with open("output.json", "w") as file:
    json.dump(data, file, indent=4)

The indent parameter formats the output cleanly.

Professional Practice
Use indentation for config files and human-readable storage. Use compact format for performance-sensitive data transfer.

6. Modifying and rewriting JSON

A common workflow:

  1. Load JSON
  2. Modify Python object
  3. Write back to file
import json

with open("data.json", "r") as file:
    data = json.load(file)

data["age"] = 18
data["subjects"].append("Chemistry")

with open("data.json", "w") as file:
    json.dump(data, file, indent=4)

7. Common pitfalls

  • Forgetting that JSON keys must be strings
  • Trying to serialize unsupported Python objects (e.g., sets)
  • Confusing load vs loads
  • Overwriting files without backup
# This will raise TypeError
data = {"numbers": {1, 2, 3}}

JSON supports lists but not sets.

8. Why JSON is critical

You now understand:

  • How JSON maps to Python data types
  • How to parse from files and strings
  • How to write structured data safely
  • How to modify and persist changes

JSON is the backbone of REST APIs, web services, configuration systems, and modern data pipelines.
