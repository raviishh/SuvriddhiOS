Loop Control in Python

Loop control statements modify how loops behave during execution. They allow you to exit early, skip iterations, or intentionally do nothing.

1. The break Statement

break immediately terminates the loop. Control moves to the first statement after the loop. 

for i in range(10):
    if i == 5:
        break
    print(i)

Output: 0 1 2 3 4 The loop stops when i == 5.

Execution Insight
break overrides the loop condition entirely.

2. The continue Statement

continue skips the rest of the current iteration and moves to the next cycle. 

for i in range(5):
    if i == 2:
        continue
    print(i)

Output: 0 1 3 4 The value 2 is skipped.

Important Distinction
break ends the loop. continue skips only one iteration.

3. The pass Statement

pass does nothing. It is a placeholder used when syntax requires a statement. 

for i in range(5):
    if i == 3:
        pass
    print(i)

This loop behaves normally. pass simply occupies space.

When to Use
Use pass when scaffolding code or defining empty functions/classes.

4. Loop else Behavior

Both for and while loops can include an else block.

The else block executes only if the loop completes normally — meaning no break occurred. 

for i in range(3):
    print(i)
else:
    print("Completed without break")

If a break is triggered, the else block is skipped.

5. Practical Patterns

Loop control statements are commonly used for:

  • Search operations
  • Input validation
  • Early termination for efficiency
  • Filtering logic
numbers = [3, 7, 9, 12]

for num in numbers:
    if num % 2 == 0:
        print("Even number found")
        break
else:
    print("No even numbers found")

6. Common Mistakes

  • Placing continue before updating loop variables (in while loops)
  • Accidentally skipping necessary logic
  • Misunderstanding loop else behavior
  • Overusing break instead of restructuring logic
Quick Check
What happens here? x = 0 while x < 3: x += 1 if x == 2: break else: print("Done") Does "Done" print?

7. Why Loop Control Matters

Loop control statements refine iteration behavior. They give you precision over execution flow.

Mastery of loop control prevents infinite loops, improves efficiency, and enables elegant search and validation logic.

Practice Strategy
Implement a search algorithm twice — once without break, and once with it. Compare clarity and efficiency.
