Pattern Matching in Python

Pattern matching allows you to compare a value against multiple patterns and execute code based on structural matches. Introduced in Python 3.10, it provides a powerful alternative to long if-elif chains.

1. The match Statement

Pattern matching uses the match keyword followed by one or more case blocks. 

status = 404

match status:
    case 200:
        print("OK")
    case 404:
        print("Not Found")
    case 500:
        print("Server Error")
    case _:
        print("Unknown status")

The underscore (_) acts as a wildcard — similar to else.

Key Distinction
Unlike if, pattern matching compares structure, not just equality.

2. Matching Multiple Values

You can match multiple literals using the pipe (|) operator. 

day = "Saturday"

match day:
    case "Saturday" | "Sunday":
        print("Weekend")
    case _:
        print("Weekday")

3. Matching Sequences

Pattern matching can destructure sequences like lists and tuples. 

point = (3, 0)

match point:
    case (0, 0):
        print("Origin")
    case (x, 0):
        print("On X-axis at", x)
    case (0, y):
        print("On Y-axis at", y)
    case (x, y):
        print("Point at", x, y)

Variables like x and y capture matched values.

4. Using Guards (Conditional Patterns)

Guards add extra conditions using if. 

number = 15

match number:
    case n if n % 2 == 0:
        print("Even")
    case n if n % 2 != 0:
        print("Odd")

Guards refine matches with Boolean logic.

5. Matching Dictionaries

You can match dictionary structures by keys. 

response = {"status": 200, "data": "OK"}

match response:
    case {"status": 200, "data": message}:
        print("Success:", message)
    case {"status": 404}:
        print("Not Found")

Only specified keys are required to match.

6. When to Use Pattern Matching

Pattern matching is useful when:

  • Handling structured data (APIs, JSON, tuples)
  • Replacing long if-elif chains
  • Working with different command types
  • Parsing input formats
Design Consideration
For simple equality checks, if may be clearer. Use match when structure matters.

7. Common Mistakes

  • Forgetting the default case (case _:)
  • Misunderstanding variable capture vs literal match
  • Using it for trivial logic where if is simpler
  • Running on Python versions earlier than 3.10
Quick Check
What prints? value = (0, 5) match value: case (0, y): print("Y-axis", y)

8. Why Pattern Matching Matters

Pattern matching introduces structural thinking into Python control flow. Instead of asking “Is this equal?”, you ask “Does this shape match?”

It encourages clearer branching logic when working with complex data.

Practice Strategy
Refactor a long if-elif chain into a match statement. Compare readability and maintainability.
