Closures

In this lesson, you will learn how a function can remember data, even after another function has finished running. This idea is called a closure.

1. Functions inside functions

Python allows one function to be defined inside another. 

def outer():
    message = "Hello"

    def inner():
        print(message)

    inner()

outer()

Output:

Hello

The inner function can use variables from the outer function.

2. What is a closure?

A closure happens when:

  • An inner function remembers variables from the outer function.
  • The outer function has already finished running.

Simple Analogy
Think of a closure like a backpack. The inner function carries some variables with it, even after leaving the outer function.

3. A real closure example

def outer():
    message = "Hello"

    def inner():
        print(message)

    return inner

my_function = outer()
my_function()

Output:

Hello

Notice:

  • outer() has finished.
  • But inner() still remembers message.

That is a closure.

4. Changing remembered variables

What if we want to change the outer variable? We use the keyword nonlocal. 

def counter():
    count = 0

    def increase():
        nonlocal count
        count += 1
        print(count)

    return increase

my_counter = counter()

my_counter()
my_counter()
my_counter()

Output:

1 2 3

The inner function updates the remembered variable.

5. Closure vs Class

The counter example can also be written using a class. 

class Counter:
    def __init__(self):
        self.count = 0

    def increase(self):
        self.count += 1
        print(self.count)

c = Counter()
c.increase()
c.increase()

Both solutions work.

Difference:

  • Closures are simpler for small tasks.
  • Classes are better for large, complex programs.

6. Why closures matter

Closures are important because:

  • Decorators use closures.
  • They help store state without using global variables.
  • They make code cleaner.

Important Idea
A closure is created when a function remembers variables from its outer scope.

7. Try it yourself

Create a function that:

  • Takes a number as input.
  • Returns a function.
  • The returned function multiplies its input by the remembered number.

Challenge
Try building a simple login system where a function remembers a password.

8. What you learned

Now you understand:

  • Inner functions can access outer variables.
  • Closures remember variables after the outer function ends.
  • nonlocal allows modification of outer variables.
