Static Type Checking (mypy — Concept)

In the last lesson, you added type hints. Now we learn how tools can check those type hints before the program runs. This is called static type checking.

1. What is static type checking?

Python normally checks types while the program is running. This is called dynamic typing.

Static type checking is different. It checks types before the program runs.

Simple Analogy
Think of static type checking like a spelling checker. It finds mistakes before you publish your work.

2. Why do we need it?

Look at this example: 

def add(a: int, b: int) -> int:
    return a + b

result = add(5, "10")

This code has a mistake. We passed a string instead of an integer.

Python will only show an error when it runs. But static type checking can warn us earlier.

3. What is mypy?

mypy is a tool that checks Python type hints.

It reads your code. It checks the type hints. It reports type errors.

Important
mypy does not run your program. It only analyzes the code.

4. Example of a type error

def greet(name: str) -> str:
    return "Hello " + name

greet(123)

If we check this file with mypy, it would say something like:

error: Argument 1 to "greet" has incompatible type "int"; expected "str"

The program has not run yet. But the tool already found the problem.

5. How it helps large projects

In small programs, mistakes are easy to find. In large programs with many files, mistakes are harder.

Static type checking:

  • Finds type mistakes early.
  • Makes teamwork easier.
  • Improves code reliability.

Many professional teams use tools like mypy.

6. Important limitations

Static type checking is helpful. But it is not perfect.

  • It only checks type hints you provide.
  • If you do not add hints, it cannot check much.
  • It does not replace testing.
Key Idea
Type checking reduces errors. It does not guarantee perfect code.

7. Try it yourself

Add type hints to this function: 

def divide(a, b):
    return a / b

Then think: What type errors could happen?

Challenge
Write a function that takes a list of integers. Try passing a list of strings. Imagine what mypy would report.

8. What you learned

Now you understand:

  • What static type checking means.
  • How it is different from dynamic typing.
  • How mypy checks type hints before running code.
  • Why it helps large projects.
