Named Tuples

In this lesson, you will learn how to store related data in a clean and readable way. This is done using named tuples.

1. The problem with normal tuples

A tuple can store multiple values: 

person = ("Alice", 25, "Engineer")

print(person[0])
print(person[1])

This works. But:

  • What does index 0 mean?
  • What does index 1 mean?

It is not very clear.

Problem
Index numbers are hard to remember. Code becomes confusing.

2. What is a named tuple?

A named tuple is like a normal tuple. But each value has a name.

We create it using the collections module. 

from collections import namedtuple

Person = namedtuple("Person", ["name", "age", "job"])

p = Person("Alice", 25, "Engineer")

print(p.name)
print(p.age)

Output:

Alice 25

Now the code is easier to read.

3. Named tuple features

Named tuples:

  • Are immutable (cannot be changed).
  • Allow access by name.
  • Still behave like tuples.

print(p[0])      # still works
print(p.name)    # better way

Trying to change a value will cause an error: 

p.age = 30

This will fail because named tuples are immutable.

4. Converting to dictionary

Named tuples have a useful method: 

print(p._asdict())

Output:

{'name': 'Alice', 'age': 25, 'job': 'Engineer'}

This converts the named tuple into a dictionary.

5. Named tuple vs dictionary vs class

Let us compare:

  • Tuple — simple, but not readable.
  • Dictionary — readable, but slightly heavier.
  • Class — powerful, but more code.
  • Named tuple — simple and readable.

Use named tuples when:

  • You want simple structured data.
  • You do not need to change values.

Important Idea
Named tuples are good for small data structures. For complex logic, use a class.

6. Modern note (typing.NamedTuple)

Python also provides another way using the typing module. 

from typing import NamedTuple

class Person(NamedTuple):
    name: str
    age: int
    job: str

p = Person("Alice", 25, "Engineer")

print(p.name)

This version works well with type hints.
