Environment Variables (Python)

Learn how programs store hidden settings using environment variables. These are small pieces of information given by the operating system to your program.

1. What Are Environment Variables?

Environment variables are special values stored by the operating system. Programs can read them while running.

They usually store:

  • Paths to important folders
  • System settings
  • Configuration values

Analogy
Think of environment variables as sticky notes the operating system gives to your program before it starts.

Some common environment variables:

  • PATH – Where programs are located
  • HOME – Your home folder (Linux/macOS)
  • USERPROFILE – Your home folder (Windows)

2. Reading Environment Variables

We use the os module to access them. 

import os

print(os.environ)

os.environ is like a dictionary that contains all environment variables.

Get One Variable

import os

home = os.getenv("HOME")
print(home)

getenv() safely gets a variable. If it does not exist, it returns None.

3. Creating Environment Variables in Python

import os

os.environ["MY_NAME"] = "Alice"

print(os.getenv("MY_NAME"))

This creates a new environment variable called MY_NAME.

Important
This variable exists only while the program is running. It disappears after the program ends.

4. Why Are Environment Variables Useful?

They help programs behave differently without changing code.

For example:

  • Store an API key
  • Switch between test mode and production mode
  • Change folder locations

Instead of writing: 

API_KEY = "12345"

You can write: 

import os

API_KEY = os.getenv("API_KEY")

This keeps secret values outside your code.

5. Small Practice Project

Write a program that:

  • Reads an environment variable called USER_NAME
  • If it exists, prints: "Hello, USER_NAME"
  • If it does not exist, prints: "Hello, Guest"

import os

name = os.getenv("USER_NAME")

if name:
    print("Hello,", name)
else:
    print("Hello, Guest")
