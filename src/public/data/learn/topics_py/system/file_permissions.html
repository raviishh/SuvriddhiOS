File Permissions (Python)

Learn how the operating system controls who can read, change, or run a file. In this lesson, you will check and change file permissions using Python.

1. What Are File Permissions?

File permissions decide:

  • Who can read a file
  • Who can write (change) a file
  • Who can execute (run) a file

Most systems use three basic permissions:

  • Read (r) – Can open and view the file
  • Write (w) – Can modify the file
  • Execute (x) – Can run the file as a program

Analogy
Think of a file like a notebook. Read means you can look at it. Write means you can edit it. Execute means you can use it to perform an action.

Permissions are mainly important on Linux and macOS. Windows handles them differently.

2. Checking File Permissions

We use the os module. 

import os

info = os.stat("example.txt")
print(info)

os.stat() gives information about a file.

The permission information is stored inside this result. It may look complex, but Python gives us tools to work with it.

3. Changing File Permissions

We use os.chmod() to change permissions. 

import os

os.chmod("example.txt", 0o444)

The number 0o444 means:

  • Read permission only
  • No writing
  • No executing

Another example: 

os.chmod("example.txt", 0o644)

This usually means:

  • Owner can read and write
  • Others can only read

Important
Changing permissions may not work the same way on Windows. These numbers are mainly for Linux and macOS systems.

4. Why Permissions Matter

Permissions protect files from:

  • Accidental changes
  • Unauthorized access
  • Security risks

For example:

  • A system file should not be writable by everyone.
  • A script must have execute permission to run.

Good permission settings make your system safer.

5. Small Practice Challenge

Try writing a program that:

  • Creates a file called test.txt
  • Changes its permission to read-only

import os

# create file
with open("test.txt", "w") as f:
    f.write("Hello")

# change permission
os.chmod("test.txt", 0o444)

print("Permission changed to read-only")
Friendly Tip
If you cannot edit the file after this, that means the permission change worked.
