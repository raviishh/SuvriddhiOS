PEP 8 Deep Dive — Writing Clean Python Code

PEP 8 is the official style guide for Python code. It explains how Python code should look. Clean code is easier to read, understand, and maintain.

1. What is PEP 8?

PEP stands for Python Enhancement Proposal. PEP 8 is the document that defines Python style rules.

It does NOT change how Python works. It changes how your code looks.

Main Idea
Code is read more often than it is written. Style makes code easier for humans.

2. Naming Rules

Variables and functions: use snake_case 

# Good
total_price = 100
def calculate_total():

# Bad
TotalPrice = 100
def CalculateTotal():

Classes: use CapitalizedWords 

# Good
class UserAccount:

# Bad
class user_account:

Constants: use ALL_CAPS

MAX_USERS = 100

3. Spacing Rules

Use spaces around operators:

# Good
total = price + tax

# Bad
total=price+tax

Use blank lines between functions:

def first_function():
    pass


def second_function():
    pass

Line length:

Keep lines shorter than 79–100 characters. Short lines are easier to read.

4. Imports

Imports should:

  • Be at the top of the file
  • Be one per line
  • Be grouped clearly
# Good
import os
import sys

from math import sqrt

# Bad
import os, sys

5. Example: Bad vs Good Code

Bad style:

class user:
 def __init__(self,name):
  self.name=name
 def sayHello(self):
  print("Hello "+self.name)

Good style:

class User:

    def __init__(self, name):
        self.name = name

    def say_hello(self):
        print("Hello " + self.name)

The second example is easier to read. The structure is clear.

6. Tools That Help

You do not need to check style manually. There are tools that help you.

flake8 checks for style problems:

pip install flake8
flake8 your_file.py

black automatically formats your code:

pip install black
black your_file.py
Tip
In real projects, formatting tools are used automatically.
