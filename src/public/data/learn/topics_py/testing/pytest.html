Introduction to pytest

In the previous lesson, we used unittest. Now we learn pytest — a simpler and more modern testing tool for Python.

1. Why pytest?

unittest works well. But sometimes it feels long and complex.

pytest is:

  • Shorter to write
  • Easier to read
  • Very popular in real projects

Simple Idea
If unittest is like filling a long form, pytest is like writing a short message.

2. Installing pytest

pytest is not built into Python. You need to install it. 

pip install pytest

After installing, you can use it in any project.

3. Writing Your First pytest Test

Let’s test the same function as before.

Step 1: Create a file called math_utils.py

def add(a, b):
    return a + b

Step 2: Create a file called test_math_utils.py

from math_utils import add

def test_add_positive_numbers():
    result = add(2, 3)
    assert result == 5

That’s it. No classes. No special methods. Just a normal function.

The line assert result == 5 checks if the result is correct. If it is wrong, the test fails.

4. Running pytest

Open your terminal in the project folder. Run: 

pytest

If the test passes, you will see something like:

1 passed in 0.01s

If the test fails, you will see details like:

E assert 6 == 5

pytest clearly shows what went wrong.

5. Writing Multiple Tests

You can write many test functions in the same file. 

def test_add_negative_numbers():
    assert add(-2, -3) == -5

def test_add_zero():
    assert add(0, 5) == 5

pytest automatically finds all functions that:

  • Start with test_
  • Are inside files that start with test_

Important Rule
Name your test files and test functions correctly. Otherwise, pytest will not find them.

6. unittest vs pytest

unittest style:

class TestAdd(unittest.TestCase):
    def test_add(self):
        self.assertEqual(add(2, 3), 5)

pytest style:

def test_add():
    assert add(2, 3) == 5

pytest is shorter and easier to read. That is why many developers prefer it.

7. Small Exercise

1. Write a function subtract(a, b). 2. Write at least 3 pytest tests for it. 3. Run pytest. 4. Break your function on purpose. 5. Observe how pytest shows the error.

Friendly Tip
pytest makes testing simple. Simple tools help you focus on writing good code.
