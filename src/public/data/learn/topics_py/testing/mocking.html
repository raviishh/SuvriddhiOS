Mocking in Python

Sometimes your code depends on the outside world — files, APIs, time, or databases. In this lesson, you will learn how to mock those parts during testing.

1. The Problem: Outside Dependencies

Look at this example: 

import requests

def get_user_name():
    response = requests.get("https://api.example.com/user")
    data = response.json()
    return data["name"]

This function depends on:

  • The internet
  • An external server
  • The API working correctly

If the internet is slow or the server is down, your test fails. But maybe your code is correct.

Main Idea
Good tests should be fast and stable. They should not depend on the real internet or real databases.

2. What is Mocking?

Mocking means replacing a real object with a fake one during testing.

Instead of calling the real API, we create a fake version that returns what we want.

Simple Analogy
Imagine you are testing a car. Instead of driving on a real road, you test it in a simulator. The simulator is a "mock" version of the road.

3. Using unittest.mock

Python has a built-in module called unittest.mock.

Let’s simplify our example.

File: data_service.py

def get_data():
    return {"name": "Real User"}

def get_user_name():
    data = get_data()
    return data["name"]

Now we want to test get_user_name() without calling the real get_data().

File: test_data_service.py

import unittest
from unittest.mock import patch
from data_service import get_user_name

class TestUserName(unittest.TestCase):

    @patch("data_service.get_data")
    def test_get_user_name(self, mock_get_data):
        mock_get_data.return_value = {"name": "Fake User"}
        result = get_user_name()
        self.assertEqual(result, "Fake User")

if __name__ == "__main__":
    unittest.main()

Explanation:

  • @patch("data_service.get_data") replaces the real function.
  • mock_get_data.return_value sets fake data.
  • The real function is not called.

4. Why Mocking is Important

Mocking:

  • Makes tests faster
  • Makes tests more reliable
  • Lets you test code in isolation

If you do not mock external systems, your tests may:

  • Fail randomly
  • Be slow
  • Depend on the internet

5. Mocking Another Example (Time)

Sometimes your code depends on the current time.

File: clock.py

import datetime

def get_current_year():
    return datetime.datetime.now().year

We can mock datetime.datetime.now to return a fixed date.

This makes your test predictable.

Friendly Tip
Mock only what you do not control. Test your own logic. Fake the outside world.
