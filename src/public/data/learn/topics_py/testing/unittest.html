Unit Testing in Python (unittest)

Learn how to check if your Python code works correctly. In this lesson, we use Python’s built-in testing tool called unittest. Simple, clear, and step by step.

1. What is Testing?

Testing means checking if your code works the way you expect. When you write a function, you want to make sure it gives the correct answer.

Simple Analogy
Imagine you solve 10 math questions. Before giving your homework to the teacher, you check your answers. Testing is like checking your answers before showing your program to users.

Programs can have bugs. A bug is a mistake in the code. Testing helps us find bugs early.

2. What is a Unit?

A unit is one small part of a program. Usually, a unit is a single function. 

def add(a, b):
    return a + b

Here, add is a unit. We can test it to make sure it works correctly.

3. Writing Your First Test

Python has a built-in module called unittest. We use it to write tests.

Step 1: Create a file called math_utils.py

def add(a, b):
    return a + b

Step 2: Create another file called test_math_utils.py

import unittest
from math_utils import add

class TestAddFunction(unittest.TestCase):

    def test_add_positive_numbers(self):
        result = add(2, 3)
        self.assertEqual(result, 5)

if __name__ == "__main__":
    unittest.main()

Explanation:

  • import unittest loads the testing tool.
  • class TestAddFunction is where we write tests.
  • self.assertEqual() checks if two values are equal.
  • If they are not equal, the test fails.

4. Running the Test

Open your terminal inside the folder. Run: 

python -m unittest

If the test passes, you will see something like:

. ---------------------------------------------------------------------- Ran 1 test in 0.001s OK

If the test fails, you will see something like:

FAIL: test_add_positive_numbers AssertionError: 6 != 5

This means something is wrong in your function.

5. Writing More Tests

Good testing means checking many cases. 

def test_add_negative_numbers(self):
    result = add(-2, -3)
    self.assertEqual(result, -5)

def test_add_zero(self):
    result = add(0, 5)
    self.assertEqual(result, 5)

Why write many tests?

  • To check different inputs
  • To catch hidden bugs
  • To make your code strong

6. Why Unit Testing Is Important

Unit testing:

  • Helps you find bugs early
  • Makes your code safer to change
  • Makes your project look professional

Important Idea
If you change your code later, you can run tests again. If tests still pass, your code is still correct.
