What is C?

A friendly introduction to the C programming language — short, clear, and full of examples. Think of this as learning how to give instructions to the computer like a careful chef following a recipe.

1. Meet C — the language beneath the hood

C is a programming language created in the 1970s. It's simple, powerful, and very close to how computers actually work. Imagine a robot that obeys exact instructions. C teaches you to write those exact instructions so the robot does useful things.

Analogy
Think of this: if high-level languages like Python are fancy LEGO kits (easy to snap together), C is the basic bricks and glue — you can build anything, but you need to be careful with the placement.

Why learn C? Because it helps you understand how computers store data, how memory works, and how programs run. Operating systems, embedded devices, and games often use C for its speed and control.

2. A C program — the first example

The simplest C program has a function called main(). When you run a C program, the computer starts by calling main. 

#include <stdio.h>

int main() {
    printf("Hello, World!\n");
    return 0;
}

Explanation:

  • #include <stdio.h> tells the compiler we want the standard input/output functions (like printf).
  • int main() is the starting point for the program.
  • printf prints text to the screen.
  • return 0; ends the program and tells the system everything is okay.

Try it: Save as hello.c, compile with gcc hello.c -o hello, run ./hello.
Output:
Hello, World!

3. Variables — boxes for information

Variables are like labeled boxes where you store values. Each box has a type that says what it can hold: integers, characters, decimals, etc. 

#include <stdio.h>

int main() {
    int apples = 5;        // integer box
    float price = 12.50;   // decimal box
    char letter = 'A';     // single character

    printf("Apples: %d\n", apples);
    printf("Price: %.2f\n", price);
    printf("Letter: %c\n", letter);

    return 0;
}

Notes:

  • %d prints integers, %.2f prints floating numbers with 2 decimals, and %c prints a character.
  • Try changing the values to see different outputs.

Kid Friendly Analogy
Imagine you have a school locker for books (an int box). If you want to store toys (a different type), you need a different locker built for toys. Types tell the computer how to store the thing.

4. Control flow — making decisions and repeating tasks

Programs often need to make choices and repeat actions. C gives you tools: if, else, and loops like for and while. 

#include <stdio.h>

int main() {
    int n = 5;
    if (n > 0) {
        printf("Positive\n");
    } else {
        printf("Zero or negative\n");
    }

    // loop
    for (int i = 0; i < n; i++) {
        printf("i = %d\n", i);
    }

    return 0;
}

Explanation:

  • if (condition) runs the code if the condition is true.
  • for repeats code a fixed number of times — good for counting.

5. Functions — naming actions

Functions let you group instructions and give them a name. This keeps programs tidy and lets you reuse work. 

#include <stdio.h>

int add(int a, int b) {
    return a + b;
}

int main() {
    int result = add(3, 4);
    printf("3 + 4 = %d\n", result);
    return 0;
}

Why functions? It's like asking someone to "make tea" instead of listing every single step each time — you name the action and use it whenever needed.

Small exercise
Write a function multiply that multiplies two numbers and returns the result. Call it from main.

6. How a program becomes runnable — compilation

C code you write is text. To run it, you compile it: the compiler (like a translator) turns your code into machine instructions the computer understands.

If the compiler finds mistakes (errors), it gives messages so you can fix them. Learning to read these messages is a key skill.
