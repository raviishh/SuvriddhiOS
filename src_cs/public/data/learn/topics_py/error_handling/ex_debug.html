Part 1 — Fix the ZeroDivisionError

The function crashes when passed an empty list because len([]) is 0. Guard against edge cases.

# calculate_average([]) → ZeroDivisionError: division by zero

What to do

  • Fix calculate_average() so it returns 0 when passed an empty list.
  • It should still work correctly for non-empty lists.
  • Print calculate_average([]) and calculate_average([10, 20, 30]).

Watch out for

  • Return 0 (integer), not 0.0 — the checker expects '0' on the first line.
  • Do not change the logic for non-empty lists — only guard the empty case.
Hints (4)
  • Add an early return: if not numbers: return 0
  • Or check len(numbers) == 0.
  • if not numbers is idiomatic Python — an empty list is falsy.
  • The average of [10, 20, 30] is (10+20+30)/3 = 60/3 = 20.0.

Part 2 — Off-by-One in a Loop

range(n) goes from 0 to n-1. range(1, n) goes from 1 to n-1. Use range(1, n+1) to include n.

# This prints 0 through 4 on separate lines — missing 5!
for i in range(5):
    print(i)
Output:
0
1
2
3
4

What to do

  • Fix the code so it prints numbers 1 through 5 inclusive, one per line.

Watch out for

  • range(5) gives 0, 1, 2, 3, 4 — starting at 0, stopping before 5.
  • Do not hardcode the numbers — use range.
Hints (3)
  • range(1, 6) starts at 1 and stops before 6.
  • A common pattern: range(1, n+1) when you want 1 through n inclusive.
  • Remember: range's stop is exclusive — range(1, 6) gives 1, 2, 3, 4, 5.

Stretch — Multi-Bug Hunt stretch

Real bugs are rarely alone. This snippet has three distinct bugs — a logic error, a type error, and a name error. Find and fix all three.

def word_count(text):
    words = text.split(" ")
    counts = {}
    for Word in words:
        counts[word] = counts.get(word, 0) + 1
    return counts

def most_frequent(counts)
    return max(counts, key=counts.get)

result = word_count('the cat sat on the mat the cat')
print(most_frequent(result))
print(lenn(result))

What to do

  • Fix all bugs in the snippet above.
  • word_count should return {'the': 3, 'cat': 2, 'sat': 1, 'on': 1, 'mat': 1}.
  • most_frequent should return 'the'.
  • The final print should output the number of unique words: 5.
  • Print most_frequent(result) then len(result).

Watch out for

  • Fix all three bugs — the program must run to completion without any errors.
  • Do not change the logic, only fix the bugs — the algorithm is correct once the syntax/name errors are resolved.
Hints (4)
  • Bug 1 (NameError): the loop variable is capitalised as Word, but used as word — Python names are case-sensitive.
  • Bug 2 (SyntaxError): the def line for most_frequent is missing a colon at the end.
  • Bug 3 (NameError): lenn is not a function — the built-in is len.
  • After fixing: run mentally — word_count produces the dict, most_frequent returns 'the', len returns 5.
