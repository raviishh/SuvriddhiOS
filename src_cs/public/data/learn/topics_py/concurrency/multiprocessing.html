Multiprocessing — True Parallel Work

Learn how Python can use multiple CPU cores to run heavy tasks at the same time. This is useful when your program needs real computing power.

1. Threads vs Processes

In the last lesson, we learned about threads. Threads run inside the same program and share memory.

Processes are different. A process is a completely separate Python program. It has its own memory and its own space.

Simple idea
A thread is like two workers sharing one desk. A process is like two workers in two different rooms.

Multiprocessing allows Python to use multiple CPU cores. This is important for heavy calculations.

2. CPU-heavy tasks

Some programs do a lot of math or calculations. These are called CPU-bound tasks. 

def heavy_task():
    total = 0
    for i in range(100000000):
        total += i
    print("Done")

This task uses the CPU a lot. Threads will not make this faster in Python. For this, we use multiprocessing.

3. Creating a process

Python has a module called multiprocessing. 

from multiprocessing import Process

def say_hello():
    print("Hello from another process!")

if __name__ == "__main__":
    process = Process(target=say_hello)
    process.start()
    process.join()

    print("Main program finished.")

Important parts:

  • Process(target=...) creates a new process.
  • start() begins the process.
  • join() waits for it to finish.

Very Important
Always use if __name__ == "__main__": when using multiprocessing. Without it, your program may break.

4. Running two processes

Let’s run two heavy tasks at the same time. 

from multiprocessing import Process

def heavy_task(name):
    total = 0
    for i in range(50000000):
        total += i
    print(f"{name} finished")

if __name__ == "__main__":
    p1 = Process(target=heavy_task, args=("Process 1",))
    p2 = Process(target=heavy_task, args=("Process 2",))

    p1.start()
    p2.start()

    p1.join()
    p2.join()

    print("All processes finished.")

If your computer has multiple CPU cores, both processes can run at the same time.

5. Memory difference

Threads share memory. Processes do NOT share memory by default.

This means:

  • Threads can change the same variable.
  • Processes cannot directly change each other's variables.

Why this matters
Processes are safer for heavy work because they do not interfere with each other.

6. When should you use multiprocessing?

Use multiprocessing when your program:

  • Does heavy math
  • Processes large data
  • Runs complex calculations

These are called CPU-bound tasks.

Remember
Threads are good for waiting. Processes are good for computing.
