Virtual Environments (venv) in Python

Learn how to isolate your project’s dependencies. Virtual environments keep projects clean and prevent version conflicts.

1. Why virtual environments exist

Different projects may need different versions of the same library.

Example:

  • Project A needs Django 3
  • Project B needs Django 4

If both are installed globally, they can conflict.

Simple Analogy
Imagine two school projects needing different textbooks. Instead of mixing books in one bag, you keep separate bags. A virtual environment is that separate bag.

2. What is a virtual environment?

A virtual environment is an isolated Python environment inside your project folder.

It has:

  • Its own Python interpreter
  • Its own installed packages

It does not affect other projects.

3. Creating a virtual environment

In your project folder, run: 

python -m venv venv

This creates a folder called venv.

Inside this folder are the isolated Python files.

4. Activating the virtual environment

Before installing packages, you must activate it.

On Windows:

venv\Scripts\activate

On macOS / Linux:

source venv/bin/activate

After activation, your terminal usually shows: 

(venv)

This means you are inside the virtual environment.

5. Installing packages inside venv

Now install a package: 

pip install requests

This installs the package only inside this virtual environment.

Other projects will not see it.

6. Deactivating the environment

To leave the virtual environment, type: 

deactivate

The (venv) label disappears.

7. Why this is important

Virtual environments:

  • Prevent version conflicts
  • Make projects reproducible
  • Keep system Python clean
  • Are required in professional development

Best Practice
Create a virtual environment for every serious project. It avoids many problems later.

8. Key idea to remember

A virtual environment is a separate space for one project.

It keeps your dependencies organized and safe.

Friendly tip
Never install large libraries globally unless absolutely necessary. Always prefer a virtual environment.
