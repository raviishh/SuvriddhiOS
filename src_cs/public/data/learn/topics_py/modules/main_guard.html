Understanding __name__ == "__main__"

Learn how Python knows whether a file is being run directly or imported. This small line of code controls how your program behaves.

1. What is __name__?

Every Python file has a special built-in variable called __name__.

Python automatically sets this variable.

  • If the file is run directly → __name__ becomes "__main__"
  • If the file is imported → __name__ becomes the module name
Simple Analogy
Think of a person who can act as a team leader or a team member. The same person behaves differently depending on the role. A Python file can also act differently depending on how it is used.

2. Running a file directly

Create a file called example.py. 

# example.py

print("File is running")
print("__name__ is:", __name__)

Run it: 

python example.py
Output:
File is running __name__ is: __main__

Because we ran the file directly, Python sets __name__ to "__main__".

3. Importing the file

Now create another file called test.py. 

# test.py

import example

Run test.py.

Output:
File is running __name__ is: example

Now __name__ is "example", not "__main__".

4. The special if statement

We can control code execution using: 

if __name__ == "__main__":
    # code runs only when file is executed directly

Let’s improve example.py. 

# example.py

def greet():
    print("Hello!")

if __name__ == "__main__":
    greet()

Now:

  • If you run example.py → it prints "Hello!"
  • If you import example → it does not print anything automatically

5. Why this is important

This pattern allows a file to:

  • Act as a reusable module
  • Also act as a standalone program

This is very common in real Python projects.

Best Practice
Place testing code or demo code inside the if __name__ == "__main__" block. Keep reusable functions outside it.

6. Key idea to remember

__name__ == "__main__" means:

“Run this code only if this file is executed directly.”

It prevents unwanted code from running when the file is imported.

Friendly tip
If you forget this pattern, your module may run test code every time it is imported. That can cause confusion.
