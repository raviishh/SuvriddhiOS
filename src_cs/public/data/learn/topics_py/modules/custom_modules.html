Creating Custom Modules in Python

Learn how to split your Python code into multiple files. This keeps projects clean, readable, and easier to manage.

1. Why create custom modules?

When programs grow bigger, one file becomes messy and hard to read. Instead of writing everything in one file, we separate code into multiple files.

Each file becomes a module with a clear purpose.

Simple Analogy
Think of a school bag. You don’t throw all books loose inside. You organize them into notebooks for each subject. Modules are like those notebooks.

2. Creating your first custom module

Step 1: Create a new file called calculator.py. 

# calculator.py

def add(a, b):
    return a + b

def subtract(a, b):
    return a - b

Step 2: Create another file in the same folder called main.py. 

# main.py

import calculator

result = calculator.add(5, 3)
print(result)
Output:
8

Python finds calculator.py because it is in the same folder.

3. Importing specific functions

You can import only what you need. 

from calculator import add

print(add(10, 4))

This keeps your code shorter and clearer.

4. How Python finds your module

When you import a module, Python searches:

  • The current folder
  • Installed packages
  • System folders

If Python cannot find the module, you will see: 

ModuleNotFoundError: No module named 'calculator'
Important
The files must be in the same folder, or properly organized inside packages (which we will learn next).

5. The __pycache__ folder

After importing a module, you may see a new folder called __pycache__.

Python creates this automatically. It stores compiled versions of your module to make loading faster.

You do not need to edit or worry about this folder.

6. Good naming practices

Follow these rules when naming modules:

  • Use lowercase letters
  • Use short, clear names
  • Avoid spaces
  • Do not use names of built-in modules (like math.py)

Common Mistake
If you name your file random.py, it may conflict with Python’s built-in random module.

7. Why custom modules matter

Custom modules help you:

  • Organize large programs
  • Reuse code in different projects
  • Work better in teams
  • Fix bugs faster

Friendly tip
If your file grows beyond 200–300 lines, consider splitting it into modules. Clean structure saves time later.
