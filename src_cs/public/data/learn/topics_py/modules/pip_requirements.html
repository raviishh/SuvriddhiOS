pip and Requirements Files

Learn how to install packages and share dependencies with others. pip and requirements files make Python projects easy to reproduce.

1. What is pip?

pip is Python’s package manager.

It allows you to:

  • Install libraries
  • Upgrade libraries
  • Remove libraries

Simple Analogy
If Python is your kitchen, pip is the delivery service that brings ingredients.

2. Installing packages with pip

Install a package: 

pip install requests

Upgrade a package: 

pip install --upgrade requests

Remove a package: 

pip uninstall requests

Always run pip inside your virtual environment.

3. What is a requirements file?

A requirements file is a text file that lists all dependencies and their versions.

The file is usually named: 

requirements.txt

Example content: 

requests==2.31.0
flask==2.3.2

4. Creating a requirements file

If you already installed packages inside your virtual environment, run: 

pip freeze > requirements.txt

This saves all installed packages and their exact versions.

Now you can share this file with others.

5. Installing from requirements.txt

To install everything listed in the file: 

pip install -r requirements.txt

pip reads the file and installs all dependencies automatically.

6. Why requirements files matter

Requirements files:

  • Make projects reproducible
  • Help teams work together
  • Prevent version conflicts
  • Are required in professional projects

Best Practice
Always include a requirements.txt file when sharing a Python project.

7. Key idea to remember

pip installs packages.

requirements.txt records exactly which packages and versions your project needs.

Friendly tip
Never guess which versions you used. Always record them using pip freeze.
