Dependency Management in Python

Learn how to control and manage the external libraries your project depends on. Good dependency management keeps projects stable and reproducible.

1. What is a dependency?

A dependency is an external library your project needs to work.

Example:

  • If you use requests for web requests → your project depends on it.
  • If you use flask to build a website → your project depends on it.

Without installing these libraries, your project will not run.

Simple Analogy
If you build a robot and it needs batteries, the batteries are dependencies. Without them, nothing works.

2. The problem with versions

Libraries change over time. New versions may:

  • Add features
  • Fix bugs
  • Remove old behavior

If your code works with version 1.0 but someone installs version 2.0, the project might break.

Important
Always control which version of a dependency your project uses.

3. Installing a specific version

You can install a specific version using: 

pip install requests==2.31.0

The == locks the version exactly.

You can also allow compatible versions: 

pip install "requests>=2.30,<3.0"

This installs any version between 2.30 and below 3.0.

4. Reproducibility

Reproducibility means:

“If someone else installs this project, it works exactly the same.”

Without controlling versions:

  • Code may behave differently
  • Bugs may appear
  • Projects may fail to start

5. Checking installed dependencies

To see installed packages inside your virtual environment: 

pip list

To see exact versions: 

pip freeze

This shows precise versions, which helps with reproducibility.

6. Why dependency management matters

Good dependency management:

  • Prevents unexpected errors
  • Makes teamwork easier
  • Improves long-term stability
  • Is required in professional software development

Best Practice
Never leave dependency versions uncontrolled in serious projects.

7. Key idea to remember

Dependencies are external tools your project relies on.

Managing versions keeps your project stable and predictable.

Friendly tip
Always combine dependency management with virtual environments for best results.
