CLI Applications with Python (argparse)

In this lesson, you will learn how to turn a normal Python script into a real command-line tool. By the end, you will build a small Task Manager that works directly from the terminal.

1. What is a CLI application?

CLI means Command Line Interface. Instead of clicking buttons, you type commands.

Simple idea
A CLI app is like talking to your computer using short text instructions.

For example:

python greet.py John

The word John is given to the program as input.

Many real tools are CLI programs:

  • Git
  • Docker
  • Python itself

2. Reading arguments (the simple way)

When you run a Python file, you can pass words after it. Python stores them inside a list called sys.argv. 

import sys
print(sys.argv)

If you run:

python test.py hello world

Output:

['test.py', 'hello', 'world']

The first value is always the file name.

Problem
Using sys.argv directly can become messy and hard to manage.

That is why we use a better tool: argparse.

3. Introducing argparse

argparse is a built-in Python module that helps us:

  • Read command arguments clearly
  • Define required inputs
  • Create helpful messages
import argparse

parser = argparse.ArgumentParser()
parser.add_argument("name")

args = parser.parse_args()
print("Hello,", args.name)

Run it like this:

python greet.py Alice

Output:

Hello, Alice

Now your program understands structured input.

4. Optional arguments and flags

Sometimes we want optional inputs. These usually start with --. 

import argparse

parser = argparse.ArgumentParser()
parser.add_argument("name")
parser.add_argument("--age", type=int)

args = parser.parse_args()

print("Name:", args.name)
print("Age:", args.age)

Run:

python app.py John --age 16

If you skip --age, it will be None.

Helpful feature
Try running the program with -h. You will see an automatic help message.

5. Building a real project — Task Manager CLI

Now we build something useful. A small task manager that can:

  • Add a task
  • List tasks
import argparse

tasks = []

parser = argparse.ArgumentParser()
parser.add_argument("action", choices=["add", "list"])
parser.add_argument("task", nargs="?", help="Task description")

args = parser.parse_args()

if args.action == "add":
    if args.task:
        tasks.append(args.task)
        print("Task added:", args.task)
    else:
        print("Please provide a task description.")

elif args.action == "list":
    if not tasks:
        print("No tasks yet.")
    else:
        for i, task in enumerate(tasks, start=1):
            print(f"{i}. {task}")

Run:

python tasks.py add "Study math"
python tasks.py list

This works — but there is a limitation.

Important
Tasks disappear when the program stops. In the next lesson, we will learn how to save data properly.

6. Why CLI apps matter

Many professional tools are CLI-based because:

  • They are fast
  • They are powerful
  • They are easy to automate

Learning CLI design teaches you:

  • How programs receive input
  • How to structure commands
  • How to build tools instead of simple scripts
