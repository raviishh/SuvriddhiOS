Add argparse to an Existing Project

Turn a hardcoded script into a flexible CLI tool using the argparse module.

Part 1 — Positional and Optional Arguments

argparse.ArgumentParser() builds a parser. add_argument() defines positional args (no dashes) and optional flags (with -- prefix).

parser = argparse.ArgumentParser()
parser.add_argument("name")
parser.add_argument("--shout", action="store_true")
args = parser.parse_args(["Alice", "--shout"])
print(args.name, args.shout)
Output:
Alice True

What to do

  • Build a parser with a required positional argument 'filename' and an optional flag '--verbose' (action='store_true').
  • Also add an optional argument '--count' with type=int and default=1.
  • Parse the simulated arguments ['data.txt', '--verbose', '--count', '5'].
  • Print args.filename, args.verbose, and args.count on separate lines.

Watch out for

  • Always pass a list to parse_args() in this simulated environment — normally argparse reads sys.argv automatically, but explicit lists make testing deterministic.
  • Without type=int, args.count would be the string '5', not the integer 5 — print(args.count) would look the same but int-specific operations would fail.
Hints (4)
  • parser.add_argument('filename') with no dashes makes it required and positional.
  • parser.add_argument('--count', type=int, default=1) ensures args.count is an int, not a string.
  • parser.parse_args(['data.txt', '--verbose', '--count', '5']) simulates command-line input without needing sys.argv.
  • action='store_true' means the flag's presence sets the attribute to True; its absence leaves it False.

Part 2 — Subcommands

add_subparsers() lets a single CLI tool support multiple subcommands (like 'git commit' vs 'git push'), each with its own arguments.

parser = argparse.ArgumentParser()
sub = parser.add_subparsers(dest="command")
add_p = sub.add_parser("add")
add_p.add_argument("x", type=int)
add_p.add_argument("y", type=int)
args = parser.parse_args(["add", "2", "3"])
print(args.command, args.x + args.y)
Output:
add 5

What to do

  • Build a parser with subcommands 'greet' and 'add'.
  • 'greet' takes one positional argument 'name'.
  • 'add' takes two positional integer arguments 'x' and 'y'.
  • Parse ['greet', 'World'] and print f'Hello, {args.name}!'.
  • Parse ['add', '10', '20'] (using a freshly created parser or re-parsing) and print the sum.

Watch out for

  • Each subcommand's arguments are isolated — 'name' only exists on args when the 'greet' subcommand was used.
  • Print exactly 'Hello, World!' with the exclamation mark, matching the f-string format given.
Hints (4)
  • sub = parser.add_subparsers(dest='command') then sub.add_parser('greet').add_argument('name').
  • args.command tells you which subcommand was used, useful for dispatching to different functions.
  • You can reuse the same parser object for both parse_args() calls — subparsers don't retain state between calls.
  • For the 'add' subcommand, remember both x and y need type=int so you can sum them directly.

Stretch — Validation and Custom Error Messages stretch

argparse's type= parameter can be any callable, including a custom validation function that raises argparse.ArgumentTypeError with a helpful message.

def positive_int(value):
    ivalue = int(value)
    if ivalue <= 0:
        raise argparse.ArgumentTypeError(f"{value} is not a positive integer")
    return ivalue

What to do

  • Write the positive_int(value) validator shown above.
  • Build a parser with one argument '--workers' using type=positive_int and default=1.
  • Parse ['--workers', '4'] and print args.workers.
  • Then attempt to parse ['--workers', '-2'] wrapped in a try/except catching SystemExit (argparse calls sys.exit() on validation errors), printing 'Rejected invalid value' if it was caught.

Watch out for

  • argparse's default behaviour on a validation failure is to print usage info to stderr and call sys.exit(2) — this is normal and expected; your job is just to catch the SystemExit so the exercise script itself doesn't crash.
  • Do not catch generic Exception here — SystemExit does not inherit from Exception, so you need 'except SystemExit' specifically.
Hints (4)
  • argparse.parse_args() calls sys.exit(2) internally when a type validator raises ArgumentTypeError — catch SystemExit to prevent the whole program from exiting during this exercise.
  • You'll likely want to suppress the default error message argparse prints to stderr; redirecting or ignoring it is fine for this exercise — only the two required print lines are checked.
  • Build a fresh parser (or reuse the same one) for both parse attempts since parsers are stateless between calls.
  • positive_int must both convert to int AND validate positivity — do both steps in the same function.
