Consume a Public API and Store Results

Simulate calling a JSON API, parsing the response, and persisting selected fields — without needing live network access.

Part 1 — Parsing a JSON API Response

requests.get(url).json() would normally give you a dict parsed from the response body. Here we start from that already-parsed dict to focus on extraction logic.

response = {"status": "ok", "data": {"temp_c": 21.5, "humidity": 60}}
if response["status"] == "ok":
    print(response["data"]["temp_c"])
Output:
21.5

What to do

  • Given api_response = {'status': 'ok', 'results': [{'id': 1, 'name': 'Widget', 'price': 9.99}, {'id': 2, 'name': 'Gadget', 'price': 19.99}, {'id': 3, 'name': 'Gizmo', 'price': 14.5}]}, extract the 'results' list.
  • Print the total value of all items: sum of price, printed rounded to 2 decimal places.
  • Print the name of the most expensive item.

Watch out for

  • Always check the 'status' field (or equivalent) before trusting a real API's data — this simulated example assumes status is already 'ok'.
  • Round only for display — don't round intermediate values before further calculations, or you can accumulate small errors.
Hints (4)
  • results = api_response['results']
  • total = round(sum(item['price'] for item in results), 2)
  • max(results, key=lambda item: item['price'])['name'] gives the most expensive item's name.
  • 9.99 + 19.99 + 14.5 = 44.48

Part 2 — Handling Errors and Missing Fields

Real APIs sometimes return error statuses or omit expected fields. Defensive code checks before accessing, using .get() with defaults rather than direct indexing.

response = {"status": "error", "message": "Not found"}
if response.get("status") != "ok":
    print(f"API error: {response.get('message', 'unknown error')}")
Output:
API error: Not found

What to do

  • Write fetch_price(item) that returns item['price'] if 'price' is a key in item, otherwise returns None (use .get() rather than raising).
  • Given items = [{'name': 'A', 'price': 5}, {'name': 'B'}, {'name': 'C', 'price': 12}], compute the sum of prices, treating missing prices as 0.
  • Print the total.
  • Print a list of names for items missing a price, in original order.

Watch out for

  • Using 'or 0' works here because a legitimate price is never falsy in this dataset, but be cautious with 'or' when 0 itself could be a valid, meaningful value elsewhere.
  • Check for None explicitly with 'is None' when identifying missing items, not with a falsy check, for the same reason.
Hints (4)
  • def fetch_price(item): return item.get('price')
  • total = sum(fetch_price(item) or 0 for item in items) — treating a None (missing price) as 0.
  • missing = [item['name'] for item in items if fetch_price(item) is None]
  • 5 + 0 + 12 = 17

Stretch — Caching Simulated Requests to Avoid Repeat Calls stretch

Real API clients often cache responses to avoid redundant network calls, especially for rate-limited APIs. A simple in-memory dict keyed by request parameters works well for short-lived scripts.

cache = {}
def get_cached(key, compute_fn):
    if key not in cache:
        cache[key] = compute_fn()
    return cache[key]

What to do

  • Simulate an expensive API call with call_count = [0] (a mutable list used as a counter) and a function simulate_api_call(city) that increments call_count[0] and returns a fixed dict like {'city': city, 'temp': 20 + len(city)}.
  • Write get_weather(city) that uses a module-level cache dict keyed by city name, calling simulate_api_call only on a cache miss.
  • Call get_weather('Paris') three times and get_weather('Tokyo') twice.
  • Print call_count[0] — it should be 2 (one real call per unique city, not per call).
  • Print the temp for 'Paris': 25.

Watch out for

  • Watch out for the setdefault() trap mentioned in the hints — it's a common source of bugs when the default value itself has side effects (like an API call).
  • call_count must be a mutable container (like a one-element list) if you want to increment it inside a nested function without needing a 'nonlocal' or 'global' declaration.
Hints (4)
  • cache.setdefault(city, simulate_api_call(city)) has a subtle bug: it evaluates simulate_api_call(city) every time regardless of whether the key exists. Use an explicit if city not in cache: cache[city] = simulate_api_call(city) instead.
  • call_count[0] increments only inside simulate_api_call — if your cache works correctly, it increments exactly once per unique city.
  • 'Paris' has 5 letters, so temp = 20 + 5 = 25.
  • Calling get_weather('Paris') a second and third time should hit the cache and NOT call simulate_api_call again.
