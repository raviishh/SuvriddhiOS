Return Values in Python

Return values allow functions to send data back to the caller. They are the foundation of composability — enabling functions to build on one another rather than just produce output.

1. What return Does

The return statement ends a function and sends a value back to the calling code. 

def add(a, b):
    return a + b

result = add(3, 5)
print(result)

When add(3, 5) runs, the value 8 is returned and stored in result.

2. Return vs Print

This distinction is critical.

def square(x):
    print(x * x)

The function above displays a value, but does not return it.

value = square(4)
print(value)

value becomes None.

Design Rule
Printing is for users. Returning is for programs.

3. Returning Multiple Values

Python allows multiple values to be returned as a tuple.

def divide(a, b):
    quotient = a // b
    remainder = a % b
    return quotient, remainder

q, r = divide(10, 3)

Multiple values are automatically packaged into a tuple.

4. Early Return

A function stops executing immediately when it hits return. 

def check_number(n):
    if n < 0:
        return "Negative"
    return "Non-negative"

Early returns simplify branching logic and reduce nested conditions.

5. Implicit Return of None

If no return statement is provided, Python automatically returns None. 

def example():
    x = 10

print(example())

Output:

None

Every function returns something — even if you do not specify it.

6. Returning Complex Data

Functions can return any object: lists, dictionaries, classes, or other functions. 

def create_profile(name, age):
    return {
        "name": name,
        "age": age
    }

This allows structured data to flow between parts of your program.

7. Common Mistakes

  • Using print instead of return
  • Placing code after a return (it will never run)
  • Forgetting that a function without return returns None
  • Returning inconsistent types across branches
Quick Check
What prints? def test(x): if x > 5: return x print("Small") print(test(3))

8. Why Return Values Matter

Return values make functions composable. Instead of isolated blocks of logic, you create a chain of reusable transformations.

Mastering return values is essential for building scalable, modular, and testable software.

Practice Strategy
Refactor a function that prints output into one that returns data. Then reuse that returned value in another function.
