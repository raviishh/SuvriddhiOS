Function Refactoring Drills

Take flat, repetitive code and refactor it into clean, reusable functions.

Part 1 — Extract a Function

A function groups reusable logic behind a name. Use def, give it parameters, and return the result.

def square(n):
    return n ** 2

print(square(5))  # 25
Output:
25

What to do

  • Write a function square(n) that returns n squared.
  • Call it for 2, 3, and 4 and print each result on its own line.

Watch out for

  • Do not print inside the function — return the value and let the caller print.
  • square(n) should work for any n, not just 2, 3, and 4.
Hints (4)
  • def square(n): return n ** 2
  • Call print(square(2)) etc.
  • The function must return the value, not print it — the caller decides what to do with the result.
  • ** is the exponentiation operator in Python: 2 ** 2 = 4.

Part 2 — Default Parameters

Parameters can have default values, making them optional at call time.

def greet(name, greeting="Hello"):
    return f"{greeting}, {name}!"

print(greet("Alice"))          # Hello, Alice!
print(greet("Bob", "Hi"))     # Hi, Bob!
Output:
Hello, Alice!
Hi, Bob!

What to do

  • Write a function power(base, exponent=2) that returns base ** exponent.
  • Call power(3) and print the result.
  • Call power(2, 10) and print the result.

Watch out for

  • Default parameters are evaluated once at function definition time — don't use mutable objects (like lists) as defaults.
  • power(3) means power(3, 2) — the default silently fills in the missing argument.
Hints (4)
  • def power(base, exponent=2): return base ** exponent
  • power(3) uses the default exponent of 2.
  • Default parameters must come after non-default parameters in the signature.
  • power(2, 10) passes 2 as base and 10 as exponent — 2**10 = 1024.

Stretch — Memoisation with a Closure stretch

A closure captures variables from its enclosing scope. This lets you build a memoised function: results are cached in a dict so repeated calls skip recomputation.

def make_memoised(fn):
    cache = {}
    def wrapper(n):
        if n not in cache:
            cache[n] = fn(n)
        return cache[n]
    return wrapper

What to do

  • Write make_memoised(fn) as shown above.
  • Apply it to a slow_square(n) function that prints 'computing [n]' and returns n ** 2.
  • Call memoised_square = make_memoised(slow_square).
  • Call memoised_square(4) twice and memoised_square(5) once.
  • Expected: 'computing 4' appears once (not twice), 'computing 5' appears once, and all three calls return the correct result.

Watch out for

  • slow_square should print 'computing N' then return n ** 2 — print inside the function, not in the caller.
  • The closure shares one cache dict across all calls to the same memoised_square — don't recreate it on every call.
Hints (4)
  • The cache dict lives inside make_memoised and is captured by wrapper — it persists between calls.
  • The first call to memoised_square(4) misses the cache and calls slow_square(4).
  • The second call to memoised_square(4) hits the cache and returns 16 without calling slow_square again.
  • Print the return value of each call to memoised_square.
