Scope in Python (LEGB Rule)

Scope determines where a variable can be accessed. Python follows the LEGB rule — a systematic order for resolving variable names.

1. What Is Scope?

Scope defines the visibility and lifetime of variables. Not all variables exist everywhere in your program.

Core Principle
A variable can only be accessed within the scope in which it is defined.

2. The LEGB Rule

When Python encounters a variable name, it searches in this order:

  • L — Local
  • E — Enclosing
  • G — Global
  • B — Built-in

Python stops searching once it finds a match.

3. Local Scope (L)

Variables defined inside a function exist only within that function. 

def example():
    x = 10
    print(x)

example()
print(x)  # Error

The variable x is local to example.

4. Enclosing Scope (E)

Enclosing scope appears in nested functions.

def outer():
    x = "Enclosing"

    def inner():
        print(x)

    inner()

outer()

The inner function can access variables from the enclosing function.

Note
Use nonlocal to modify enclosing variables.

5. Global Scope (G)

Variables defined outside functions exist in global scope.

x = 50

def show():
    print(x)

show()

Global variables can be accessed inside functions, but modifying them requires the global keyword. 

x = 10

def change():
    global x
    x = 20

change()
print(x)

6. Built-in Scope (B)

Python includes built-in names like print, len, and sum. 

print(len([1, 2, 3]))

These names exist automatically unless you override them.

7. Variable Shadowing

A variable in a smaller scope can override one with the same name in a larger scope. 

x = 5

def example():
    x = 10
    print(x)

example()
print(x)

The two x variables are different.

8. Common Mistakes

  • Trying to access local variables outside their function
  • Modifying global variables without global
  • Confusing nonlocal and global
  • Overriding built-in names (e.g., naming a variable list)
Quick Check
What prints? x = 5 def test(): x = 10 test() print(x)

9. Why the LEGB Rule Matters

Understanding scope prevents subtle bugs and naming conflicts. It clarifies how data flows through your program.

Mastering LEGB allows you to write predictable, modular, and well-structured code.

Practice Strategy
Write nested functions and experiment with local, nonlocal, and global. Observe how variable resolution changes.
