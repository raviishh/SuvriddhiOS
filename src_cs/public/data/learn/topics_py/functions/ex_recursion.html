Recursive Challenges

Implement factorial, Fibonacci, and a list-flattener using recursion.

Part 1 — Factorial

A recursive function calls itself with a smaller input. Every recursive function needs a base case to stop.

def countdown(n):
    if n == 0:        # base case
        print("Go!")
    else:
        print(n)
        countdown(n - 1)

countdown(3)
Output:
3
2
1
Go!

What to do

  • Write factorial(n) recursively.
  • factorial(0) and factorial(1) should both return 1.
  • Print factorial(5) and factorial(10).

Watch out for

  • Do not use a loop — the solution must be recursive.
  • factorial(0) must return 1 by mathematical convention — do not return 0.
Hints (4)
  • Base case: if n <= 1: return 1
  • Recursive case: return n * factorial(n - 1)
  • Trace it mentally: factorial(3) = 3 * factorial(2) = 3 * 2 * factorial(1) = 3 * 2 * 1 = 6.
  • Python's default recursion limit is 1000 — fine for n=10, but factorial(1000) would hit it.

Part 2 — Fibonacci

Fibonacci: fib(0)=0, fib(1)=1, fib(n)=fib(n-1)+fib(n-2). This is naturally recursive but slow for large n — fine for small values here.

# fib(6) = 0,1,1,2,3,5,8  → 8

What to do

  • Write fib(n) recursively.
  • Print fib(0) through fib(7), one per line.

Watch out for

  • fib(0) is 0 and fib(1) is 1 — both must be explicit base cases, not just one.
  • Do not confuse the index with the value: fib(7) = 13, not 7.
Hints (4)
  • Base cases: if n == 0: return 0; if n == 1: return 1
  • Recursive case: return fib(n-1) + fib(n-2)
  • fib(n) makes exponentially many calls — fib(30) takes noticeably longer. This is intentional; the next stretch goal addresses it.
  • Use a for loop calling fib(i) for i in range(8) to print all 8 values.

Part 3 — Flatten a Nested List

Recursion shines when the depth of nesting is unknown. Check if each item is itself a list, and if so, recurse into it.

nested = [1, [2, [3, 4]], 5]
# Should produce: [1, 2, 3, 4, 5]

What to do

  • Write flatten(lst) that returns a flat list from an arbitrarily nested list.
  • Print flatten([1, [2, [3, 4]], 5]).
  • Print flatten([[1, 2], [3, [4, [5]]]]).

Watch out for

  • Do not use sum([[...], [...]], []) as a shortcut — it is O(n²) and considered unreadable.
  • The function must handle arbitrary depth — do not hard-code a maximum level.
Hints (4)
  • Loop through each item. If isinstance(item, list), extend with flatten(item). Otherwise append item.
  • Start with result = [] and return it at the end.
  • isinstance(item, list) is safer than type(item) == list because it also catches subclasses.
  • extend() adds all items from an iterable; append() adds one item — use extend with the recursive result.

Stretch — Memoised Fibonacci stretch

Naive recursive Fibonacci recomputes the same values exponentially. Storing results in a dict (memoisation) turns it into O(n).

memo = {}
def fib_memo(n):
    if n in memo:
        return memo[n]
    # ... compute and store

What to do

  • Implement fib_fast(n) using a memo dict (either passed in or module-level).
  • Print fib_fast(35) — it should return instantly whereas naive fib(35) would take seconds.
  • Also print the number of unique subproblems solved: len(memo) after the call.
  • Expected: 9227465 and 36.

Watch out for

  • Pre-seed memo = {0: 0, 1: 1} or handle the base cases with if n <= 1 before the cache lookup.
  • If you use a decorator (@functools.lru_cache) that's valid Python, but implement the dict approach manually first so you understand what the decorator is doing.
Hints (4)
  • Check the cache at the top of the function before recursing.
  • Store the result in memo before returning it.
  • A module-level dict persists between calls to fib_fast — that's intentional.
  • len(memo) after fib_fast(35) should be 36 (values for n=0 through n=35).
