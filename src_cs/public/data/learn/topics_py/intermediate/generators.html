Generators & yield

In the last lesson, you built your own iterator. Now we learn something easier and more powerful: generators.

1. The problem with normal functions

A normal function runs once and returns a value. After it returns, it is finished. 

def give_numbers():
    return [1, 2, 3]

numbers = give_numbers()
print(numbers)

This works. But notice:

  • All numbers are created at once.
  • They are stored in memory as a full list.

Think About It
What if we needed one million numbers? Storing them all at once could use a lot of memory.

2. What is a generator?

A generator is a special kind of function. Instead of returning all values at once, it gives them one by one.

Simple Analogy
A normal function gives you the whole cake. A generator gives you one slice at a time.

Generators use a special keyword: yield.

3. Your first generator

def count_to_three():
    yield 1
    yield 2
    yield 3

gen = count_to_three()

print(next(gen))
print(next(gen))
print(next(gen))

Output:

1 2 3

Each time we call next(), the function continues from where it stopped.

When there are no more yield statements, Python raises StopIteration.

4. How yield works

The keyword yield does two things:

  • It sends a value back.
  • It pauses the function.

When next() is called again, the function continues from the line after yield. 

def example():
    print("Start")
    yield 10
    print("Middle")
    yield 20
    print("End")

g = example()

print(next(g))
print(next(g))

Output:

Start 10 Middle 20

Notice that "End" is not printed yet. It will print only if we call next() again.

5. Generators with loops

Generators are very powerful when combined with loops. 

def count_up_to(n):
    for i in range(1, n + 1):
        yield i

for number in count_up_to(5):
    print(number)

Output:

1 2 3 4 5

This generator gives one number at a time. It does not create a full list.

6. Generator expressions

Python also allows a short way to create generators. 

numbers = (x * 2 for x in range(5))

for num in numbers:
    print(num)

This looks like a list comprehension: 

[x * 2 for x in range(5)]

But notice:

  • Square brackets [ ] create a list.
  • Parentheses ( ) create a generator.

7. Why generators matter

Generators:

  • Use less memory.
  • Work well with large data.
  • Follow the iterator protocol automatically.

Every generator is an iterator. But not every iterator is a generator.

Important Idea
Generators are easier than writing your own __iter__() and __next__().
