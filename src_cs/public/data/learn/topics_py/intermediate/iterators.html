Iterators & The Iterator Protocol

In this lesson, you will learn how Python really goes through items one by one. You use for loops all the time — but what is happening behind the scenes?

1. What is an iterator?

An iterator is something that gives values one at a time. It does not give everything at once. It gives the next value only when asked.

Simple Analogy
Think of a candy machine. You press a button → one candy comes out. Press again → another candy. That machine is like an iterator.

Lists, tuples, strings, and many other objects in Python can be used in a for loop. These objects are called iterables.

An iterable is something you can loop over. An iterator is the object that actually gives the next value.

2. Using iter() and next()

Python gives us two important functions:

  • iter() — turns an iterable into an iterator
  • next() — asks for the next value

numbers = [10, 20, 30]

my_iterator = iter(numbers)

print(next(my_iterator))
print(next(my_iterator))
print(next(my_iterator))

Output:

10 20 30

Each time we call next(), we get the next item.

But what happens if we ask again? 

print(next(my_iterator))

Python gives an error:

StopIteration

This means: "There are no more values."

3. How for loops really work

When you write: 

for number in numbers:
    print(number)

Python secretly does something like this: 

my_iterator = iter(numbers)

while True:
    try:
        number = next(my_iterator)
        print(number)
    except StopIteration:
        break

So a for loop is just:

  • Create iterator
  • Keep asking for next value
  • Stop when there are no more values

Important Idea
The for loop is not magic. It uses iter() and next() behind the scenes.

4. The Iterator Protocol

The iterator protocol is a simple rule. If an object:

  • Has a method called __iter__()
  • Has a method called __next__()
Then Python knows how to loop over it.

__iter__() → returns the iterator object __next__() → returns the next value

5. Creating your own iterator

Let us build a simple counter that counts from 1 to a number. 

class Counter:
    def __init__(self, max_number):
        self.max = max_number
        self.current = 1

    def __iter__(self):
        return self

    def __next__(self):
        if self.current > self.max:
            raise StopIteration
        value = self.current
        self.current += 1
        return value


counter = Counter(3)

for number in counter:
    print(number)

Output:

1 2 3

Explanation:

  • __iter__() returns the object itself.
  • __next__() gives the next number.
  • When finished, it raises StopIteration.

7. Why this matters

Many advanced Python features depend on the iterator protocol:

  • Generators
  • File reading
  • Data streaming

If you understand iterators, you understand how Python moves through data. This is a core idea in Python.
