Decorators

In this lesson, you will learn how to modify a function without changing its code. This is called a decorator.

1. Functions are objects

In Python, functions are not special. They are objects. This means:

  • You can store them in variables.
  • You can pass them as arguments.
  • You can return them from other functions.

def greet():
    print("Hello!")

say_hello = greet
say_hello()

Output:

Hello!

Notice: we did not call greet(). We stored the function inside another variable.

2. Functions inside functions

A function can create another function. 

def outer():
    def inner():
        print("Inside inner function")
    return inner

my_function = outer()
my_function()

Output:

Inside inner function

The outer function returns the inner function.

3. Adding behavior to a function

Suppose we want to print something before and after a function runs. 

def decorator_function(original_function):

    def wrapper():
        print("Before the function runs")
        original_function()
        print("After the function runs")

    return wrapper


def greet():
    print("Hello!")

new_greet = decorator_function(greet)
new_greet()

Output:

Before the function runs Hello! After the function runs

The decorator wraps the original function with extra behavior.

Simple Analogy
Think of a decorator like gift wrapping. The gift inside stays the same, but something extra is added outside.

4. The @ syntax

Python gives a cleaner way to use decorators. 

def decorator_function(original_function):

    def wrapper():
        print("Before the function runs")
        original_function()
        print("After the function runs")

    return wrapper


@decorator_function
def greet():
    print("Hello!")

greet()

This is exactly the same as: 

greet = decorator_function(greet)

The @ symbol is just a shortcut.

5. Decorators with arguments

What if the function takes parameters? We must pass them inside the wrapper. 

def decorator_function(original_function):

    def wrapper(name):
        print("Before the function runs")
        original_function(name)
        print("After the function runs")

    return wrapper


@decorator_function
def greet(name):
    print("Hello,", name)

greet("Alice")

For many arguments, we can use: *args and **kwargs. 

def decorator_function(original_function):

    def wrapper(*args, **kwargs):
        print("Before the function runs")
        result = original_function(*args, **kwargs)
        print("After the function runs")
        return result

    return wrapper

6. Why decorators are useful

Decorators are often used for:

  • Logging
  • Measuring time
  • Checking permissions
  • Caching results

Many web frameworks use decorators. For example, defining routes in web applications.

Important Idea
A decorator changes behavior without changing the original function’s code.

8. What you learned

Now you understand:

  • Functions are objects.
  • Functions can return other functions.
  • Decorators wrap functions.
  • The @ symbol is just syntax sugar.
