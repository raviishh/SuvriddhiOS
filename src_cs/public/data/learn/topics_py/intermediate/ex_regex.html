Regex Pattern Challenges

Extract, validate, and transform strings using Python's re module.

Part 1 — Extract Emails

re.findall(pattern, string) returns all non-overlapping matches as a list.

import re
result = re.findall(r'\d+', 'There are 3 cats and 12 dogs')
print(result)  # ['3', '12']
Output:
['3', '12']

What to do

  • From text, extract all email addresses.
  • Print the list.

Watch out for

  • Raw strings (r'...') are essential for regex — without the r prefix, \ would need to be escaped as \\.
  • The pattern does not validate the TLD length — it just matches the shape, which is sufficient here.
Hints (4)
  • Email pattern: r'[\w.-]+@[\w.-]+'
  • re.findall(pattern, text)
  • \w matches word characters (letters, digits, underscore); . and - are included explicitly.
  • The + quantifier means 'one or more' — both sides of @ need at least one character.

Part 2 — Validate a Phone Number

re.match() checks at the start of the string; re.fullmatch() checks the entire string.

pattern = r'\d{3}-\d{4}'
print(bool(re.match(pattern, '123-4567')))  # True
Output:
True

What to do

  • Write a function is_valid_phone(number) that returns True if number matches the pattern: 3 digits, a hyphen, 3 digits, a hyphen, 4 digits (e.g. 555-123-4567).
  • Print is_valid_phone('555-123-4567') and is_valid_phone('5551234567').

Watch out for

  • re.match() would pass '555-123-4567extra' because it only checks the start — use re.fullmatch() or add $ anchor.
  • The hyphens are literal characters — no escaping needed.
Hints (4)
  • Pattern: r'^\d{3}-\d{3}-\d{4}$'
  • return bool(re.fullmatch(pattern, number))
  • \d{3} matches exactly 3 digits; \d{4} matches exactly 4 digits.
  • re.fullmatch() anchors both ends — the entire string must match, not just part of it.

Stretch — Named Groups & Substitution stretch

Named groups (?P<name>...) let you extract sub-parts of a match by name. re.sub() replaces matches with a new string, which can reference groups.

# Named group:
# m = re.search(r'(?P<year>\d{4})-(?P<month>\d{2})', '2024-03')
# m.group('year')  # '2024'

# re.sub with back-reference:
# re.sub(r'(\d+)', r'[\1]', 'abc 42 def')  # 'abc [42] def'

What to do

  • Given log_line = '2024-03-15 ERROR Something went wrong', parse it using a pattern with named groups: date, level, message.
  • Print each group on its own line: date, level, message.
  • Then use re.sub to redact all digits in log_line, replacing each digit with '*'.
  • Print the redacted line.

Watch out for

  • .+ is greedy and will capture everything after the level, including trailing whitespace — strip() if needed.
  • Named groups use (?P<name>...) syntax — not the same as non-capturing (?:...).
Hints (3)
  • Pattern: r'(?P<date>\d{4}-\d{2}-\d{2}) (?P<level>\w+) (?P<message>.+)'
  • m = re.search(pattern, log_line); m.group('date'), m.group('level'), m.group('message')
  • re.sub(r'\d', '*', log_line) replaces each digit with a single '*'.
