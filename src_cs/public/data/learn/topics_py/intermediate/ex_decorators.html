Build and Stack Decorators

Write decorators that add timing, logging, and validation to functions.

Part 1 — Timer Decorator

A decorator is a function that takes a function and returns a new function. Use functools.wraps to preserve the original function's metadata.

import functools

def my_decorator(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        print("Before")
        result = func(*args, **kwargs)
        print("After")
        return result
    return wrapper

@my_decorator
def say_hi():
    print("Hi!")
Output:
Before
Hi!
After

What to do

  • Write a @timer decorator that prints how long a function takes to run in seconds, formatted to 4 decimal places.
  • Apply it to a function slow_add(a, b) that sleeps 0.1 seconds then returns a + b.
  • Print the return value too.

Watch out for

  • The exact time will vary — the checker looks for 'slow_add took 0.1' as a substring.
  • Do not forget to return result from wrapper — otherwise slow_add(...) returns None.
Hints (4)
  • start = time.time() before calling func; elapsed = time.time() - start after.
  • print(f"{func.__name__} took {elapsed:.4f}s")
  • Use @functools.wraps(func) on wrapper to preserve __name__ and __doc__.
  • The return value of the decorated function must still be returned from wrapper.

Stretch — Retry Decorator stretch

A retry decorator calls the wrapped function again if it raises an exception, up to a maximum number of attempts. This is common in network and I/O code.

# Usage:
# @retry(times=3)
# def flaky(): ...

What to do

  • Write a decorator factory retry(times=3) that retries the function up to times attempts if it raises an exception.
  • On each failure, print: 'Attempt [n] failed: [error]'.
  • If all attempts fail, re-raise the last exception.
  • Test with a function that fails the first two calls (use a counter) then succeeds on the third.
  • Print the successful return value.

Watch out for

  • @retry(times=3) is a decorator factory — it is called with parentheses, unlike @timer.
  • Re-raise after all attempts so the caller can still handle the error if needed.
Hints (5)
  • retry(times) returns a decorator; the decorator returns a wrapper — so there are three levels of nesting.
  • Use a for loop with range(times) inside wrapper.
  • Catch all exceptions with except Exception as e; print the attempt number and error.
  • After the loop, raise the last exception: raise e
  • A counter can be simulated with a list: calls = [0]; inside the function calls[0] += 1; if calls[0] < 3: raise RuntimeError(...)
