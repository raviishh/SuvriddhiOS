The Basic Execution Model in Python

Understand how Python runs your code — step by step, line by line. This lesson explains what really happens when you press “Run.” Think of it as watching the engine work under the hood.

1. Programs run top to bottom

Python executes code in a simple way: it starts at the first line and moves downward, one statement at a time. There is no guessing, no skipping — unless you explicitly tell it to.

Analogy
Imagine reading a book. You start at the top of the page and move down line by line. Python does the same with your code.
print("Start")
print("Middle")
print("End")
Output:
Start Middle End

2. Variables are stored in memory

When Python sees a variable assignment, it:

  • Evaluates the right-hand side
  • Stores the result in memory
  • Associates the name with that value

x = 10
y = x + 5
print(y)

Execution steps:

  1. Store 10 in x
  2. Compute x + 5 → 15
  3. Store 15 in y
  4. Print 15

3. Expressions are evaluated before assignment

Python always evaluates expressions first, then performs assignments. 

a = 2 + 3 * 4
print(a)

Order of evaluation:

  • Multiplication happens before addition
  • 3 × 4 = 12
  • 2 + 12 = 14
  • 14 is stored in a

4. Control flow changes execution order

By default, execution is sequential. But control statements like if and loops change the flow. 

num = 3

if num > 0:
    print("Positive")

print("Done")

If the condition is true, the indented block runs. If false, Python skips it and continues downward.

Key Rule
Indentation defines execution blocks in Python. Incorrect indentation changes program behavior or causes errors.

5. Functions pause and resume execution

When Python encounters a function call:

  • It jumps into the function
  • Executes its body
  • Returns to where it was called

def greet():
    print("Hello")

print("Before")
greet()
print("After")
Output:
Before Hello After

Execution order:

  1. Print "Before"
  2. Jump into greet()
  3. Print "Hello"
  4. Return
  5. Print "After"

6. Errors stop execution

If Python encounters an error, execution immediately stops. 

print("Start")
print(10 / 0)
print("End")

Since division by zero causes an error, "End" will never print.

Important
Always read error messages carefully. They tell you exactly where execution failed.

7. Putting it all together

Python’s execution model is built on simple rules:

  • Code runs top to bottom
  • Expressions are evaluated before assignment
  • Indentation defines blocks
  • Functions temporarily change execution flow
  • Errors stop execution

Master this mental model and debugging becomes much easier. If you can trace code line by line on paper, you truly understand how it runs.

Friendly tip
When confused, simulate execution manually. Pretend you are the interpreter. Follow each line carefully and track variable values.
