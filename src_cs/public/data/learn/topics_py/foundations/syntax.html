Python Syntax — The Rules of the Language

A friendly introduction to Python syntax — clear, practical, and example-driven. Think of syntax as the grammar rules that help the computer understand exactly what you mean.

1. What is syntax?

Syntax is the set of rules that tells Python how code must be written. Just like English needs correct grammar to make sense, Python needs correct syntax to run.

Analogy
If vocabulary is the words you know, syntax is the sentence structure. “Dog bites man” works. “Bites man dog” is confusing. Python feels the same way.

The good news? Python is known for having clean, readable syntax. It avoids unnecessary symbols and tries to look like plain English.

2. Your first Python program

In Python, you don’t need a special main() function. The code runs from top to bottom. 

print("Hello, World!")

That’s it. No semicolons. No curly braces. Just a function call.

  • print() displays text on the screen.
  • Text must go inside quotation marks.

Try it: Save as hello.py, run with python hello.py.
Output:
Hello, World!

3. Indentation — Python’s most important rule

Unlike many languages, Python uses indentation (spaces at the beginning of a line) to define blocks of code. Indentation is not optional — it’s required. 

age = 18

if age >= 18:
    print("You are an adult.")
    print("You can vote.")
print("Done.")

Explanation:

  • The colon : starts a block.
  • Indented lines belong to that block.
  • Lines without indentation are outside the block.

Important
Most Python programs use 4 spaces for indentation. Be consistent — mixing tabs and spaces can cause errors.

4. Variables — simple and dynamic

Python does not require you to declare variable types. The type is decided automatically. 

name = "Aarav"
age = 15
height = 5.7
is_student = True

print(name)
print(age)

Notes:

  • No need to write int or float before variables.
  • Strings use quotes.
  • Booleans are written as True and False (capitalized).

Analogy
In Python, boxes label themselves automatically. You just put something in, and Python figures out what it is.

5. Comments — notes for humans

Comments are ignored by Python. They are used to explain code to other people (or your future self). 

# This is a single-line comment

age = 20  # This is an inline comment

"""
This is a multi-line comment.
It can span several lines.
"""

Use comments wisely. Clear code is better than too many comments.

6. Line breaks and statements

In Python, each line usually represents one statement. You don’t need semicolons. 

x = 5
y = 10
print(x + y)

If a statement is very long, you can continue it using parentheses: 

total = (10 + 20 +
         30 + 40)

7. Keywords — reserved words in Python

Python has reserved words that have special meaning. You cannot use them as variable names. 

if
else
for
while
def
return
class
import

These words form the structure of Python programs.

8. Quick recap — the core syntax rules

You now understand:

  • Python runs top to bottom.
  • Indentation defines blocks.
  • No semicolons required.
  • Variables are dynamically typed.
  • Colons start code blocks.

Friendly tip
When you see an error like IndentationError or SyntaxError, don’t panic. Read the message carefully — it usually tells you exactly what rule you broke.
