Write Tests for the Bank Account Project

Write pytest tests covering normal behaviour and edge cases.

Part 1 — Happy-Path Tests

A pytest test is a function whose name starts with test_. Use assert to check expected values.

def test_addition():
    assert 1 + 1 == 2

def test_string_upper():
    assert 'hello'.upper() == 'HELLO'

What to do

  • Write test_deposit() — deposit 100, assert balance is 100.
  • Write test_withdraw() — deposit 200, withdraw 50, assert balance is 150.
  • Write test_initial_balance() — assert default balance is 0.

Watch out for

  • Each test function must be independent — create a new BankAccount inside each one.
  • Assert the final balance, not an intermediate state — one assert per test is fine.
Hints (4)
  • account = BankAccount() creates a fresh account each test.
  • assert account.balance == expected_value
  • Each test function is independent — a fresh BankAccount() in every test prevents state from leaking.
  • pytest discovers functions starting with test_ automatically — no registration needed.

Part 2 — Edge Cases with pytest.raises

pytest.raises(ExceptionType) asserts that a block of code raises the expected exception.

def test_bad_input():
    with pytest.raises(ValueError):
        int("not a number")

What to do

  • Write test_overdraft() — assert that withdrawing more than the balance raises ValueError.

Watch out for

  • The withdrawal amount must exceed the balance — don't accidentally set the balance higher than the withdrawal.
  • pytest.raises is a context manager — the risky code must be inside the with block.
Hints (3)
  • with pytest.raises(ValueError): account.withdraw(amount_greater_than_balance)
  • You can also capture the exception info: with pytest.raises(ValueError) as exc_info: ... and then check str(exc_info.value).
  • The test passes only if the ValueError is raised — if no exception is raised, the test fails.

Stretch — Parametrised Tests stretch

@pytest.mark.parametrize runs a test function multiple times with different inputs, keeping your test suite DRY.

@pytest.mark.parametrize('a, b, expected', [
    (1, 2, 3),
    (0, 0, 0),
    (-1, 1, 0),
])
def test_add(a, b, expected):
    assert add(a, b) == expected

What to do

  • Write a parametrised test test_deposit_amounts that tests depositing 0, 50, and 1000.
  • Each case should assert the resulting balance equals the deposited amount (starting from 0).
  • Write a second parametrised test test_withdraw_valid that tests valid withdrawals from a 500-balance account: withdraw 100 (balance 400), 0 (balance 500), 500 (balance 0).
  • All 6 parametrised cases must pass.

Watch out for

  • Do not use a module-level account — create a fresh one inside each parametrised test body.
  • Depositing 0 should leave balance at 0 — test this edge case explicitly.
Hints (4)
  • @pytest.mark.parametrize('amount, expected', [(0, 0), (50, 50), (1000, 1000)])
  • For withdrawal: create BankAccount(500) inside the test to start with a known balance.
  • Each parametrised case is a separate test run — they are isolated.
  • Parametrize decorator goes directly above the test function definition.
