String Calculator Kata

TDD cycle: write a failing test (Red), make it pass with minimal code (Green), then improve the code (Refactor).

# Red
def test_add_empty_string():
    assert add("") == 0

# Green (minimal)
def add(s):
    return 0

What to do

  • Implement add(s) that: returns 0 for empty string, returns the number for a single number string ('5' → 5), returns the sum for comma-separated numbers ('1,2,3' → 6).
  • Write a test for each case before implementing it.
  • All tests must pass.

Watch out for

  • Write each test before writing the implementation code — that's the kata discipline.
  • s.split(',') on an empty string gives [''] — handle the empty string case before splitting.
Hints (4)
  • For empty string: return 0
  • For non-empty: return sum(int(n) for n in s.split(','))
  • The single-number case is automatically handled by split(',') — '5'.split(',') gives ['5'].
  • Follow the TDD cycle strictly: write test → see it fail → write code → see it pass.

Stretch — Extend the Kata: Newline Delimiter stretch

TDD encourages incrementally extending behaviour. Each new rule gets a test first.

# New rules:
# '1\n2,3' → 6  (newlines are also valid delimiters)
# '//;\n1;2;3' → 6  (custom delimiter specified in first line)

What to do

  • Extend add(s) to support newlines as an additional delimiter alongside commas.
  • Further extend it to support a custom delimiter: if s starts with '//', the next character is the delimiter and numbers follow after a newline.
  • Write a test for each new case before implementing it.
  • All tests (original 3 + at least 2 new) must pass.

Watch out for

  • The custom delimiter line ('//;') must be stripped before processing the numbers.
  • Test '1\n2,3' (should give 6) and '//;\n1;2' (should give 3) before implementing them.
Hints (4)
  • For newline support: replace newlines with commas before splitting: s.replace('\n', ',')
  • For custom delimiter: if s.startswith('//'): delimiter = s[2]; s = s[s.index('\n')+1:]
  • Then split on the delimiter and sum.
  • re.split(r'[,\n]', s) handles both delimiters in one call.
