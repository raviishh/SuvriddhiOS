While Loops in Python

A while loop repeatedly executes code as long as a condition remains True. It is the foundation of indefinite iteration — loops where the number of repetitions is not known beforehand.

1. Basic Syntax

A while loop runs as long as its condition evaluates to True. 

count = 1

while count <= 5:
    print(count)
    count += 1

Execution flow:

  • Check condition
  • If True → execute block
  • Repeat
  • If False → exit loop

Critical Rule
You must modify the loop variable. Otherwise, the condition never changes — resulting in an infinite loop.

2. Infinite Loops

An infinite loop occurs when the condition never becomes False. 

while True:
    print("This runs forever")

Infinite loops are sometimes intentional (e.g., servers, game loops), but dangerous if accidental.

Debug Tip
If your program freezes, check your loop condition first.

3. Using break

break immediately exits the loop, regardless of the condition. 

while True:
    user_input = input("Enter 'q' to quit: ")
    if user_input == "q":
        break
    print("You typed:", user_input)

The loop continues until break is triggered.

4. Using continue

continue skips the rest of the current iteration and moves to the next cycle. 

count = 0

while count < 5:
    count += 1
    if count == 3:
        continue
    print(count)

When count == 3, the print statement is skipped.

5. While with else

A while loop can include an else block. It runs only if the loop ends normally (not via break). 

count = 1

while count <= 3:
    print(count)
    count += 1
else:
    print("Loop finished normally")

If a break occurs, the else block is skipped.

6. Common Patterns

While loops are useful when:

  • Waiting for user input
  • Running until a condition changes
  • Polling or monitoring systems
  • Retrying operations until success
password = ""

while password != "secret":
    password = input("Enter password: ")

print("Access granted")

7. Common Mistakes

  • Forgetting to update the loop variable
  • Creating accidental infinite loops
  • Misplacing break or continue
  • Overusing while loops when a for loop is clearer
Quick Check
What happens here? x = 5 while x > 0: print(x) x -= 2 How many times does it print?

8. Why While Loops Matter

While loops give you control over repetition when the number of iterations isn’t predetermined.

They are powerful — but require disciplined condition management.

Practice Strategy
Write loops that terminate predictably. Then intentionally break them. Understanding failure modes builds real mastery.
