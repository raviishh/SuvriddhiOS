Pattern Matching Practice

Use Python 3.10+ match/case to handle structured data cleanly.

Part 1 — HTTP Status Classifier

match/case works like a supercharged switch. The wildcard case _ matches anything not caught above.

status = 404
match status:
    case 200:
        print("OK")
    case 404:
        print("Not Found")
    case _:
        print("Unknown")
Output:
Not Found

What to do

  • Write a match statement for a variable code.
  • 200 → print OK
  • 301 → print Moved Permanently
  • 404 → print Not Found
  • 500 → print Internal Server Error
  • Anything else → print Unknown Status
  • Test it with code = 301.

Watch out for

  • Python 3.10+ required for match/case.
  • case _ must be the last branch — if placed earlier it will catch everything and skip later cases.
Hints (4)
  • Each case: value: is a branch.
  • Put case _: last as the catch-all.
  • There is no fall-through in Python's match — only the first matching case runs.
  • The match subject (code) is evaluated once, then compared to each case in order.

Part 2 — Command Dispatcher

match can match strings and even destructure sequences (lists/tuples).

command = ["go", "north"]
match command:
    case ["go", direction]:
        print(f"Moving {direction}")
    case ["quit"]:
        print("Goodbye")
Output:
Moving north

What to do

  • Create command = ["go", "south"].
  • Match on: ["go", direction] → Moving [direction]
  • Match on: ["pick", item] → Picking up [item]
  • Match on: ["quit"] → Goodbye
  • Default → Unknown command

Watch out for

  • case ["go", direction] only matches a two-element list starting with 'go' — a three-element list would fall through to the default.
  • Literal string patterns are case-sensitive: 'Go' would not match 'go'.
Hints (4)
  • Sequence patterns unpack automatically — direction captures the second element.
  • Test a few values by changing command to verify all branches work.
  • The capture variable (direction, item) can be any valid Python name — it binds to whatever is in that position.
  • case ["quit"] matches a list with exactly one element 'quit'.

Stretch — Nested Pattern Matching stretch

match/case can destructure nested structures such as dicts and tuples, and can apply guard clauses with if.

# Guard clause example:
# case ["go", direction] if direction in ("north", "south", "east", "west"):
#     print(f"Valid move: {direction}")

What to do

  • Extend the command dispatcher to handle a dict-based command format.
  • command = {'action': 'go', 'direction': 'north'} → Moving north
  • command = {'action': 'pick', 'item': 'sword'} → Picking up sword
  • Add a guard: only accept directions in ('north', 'south', 'east', 'west'); anything else → Invalid direction.
  • Test with {'action': 'go', 'direction': 'up'} — should print Invalid direction.

Watch out for

  • Dict patterns match if the dict contains at least those keys — extra keys are ignored.
  • Guard conditions (if ...) are evaluated only if the structural pattern matches first.
Hints (3)
  • Dict patterns: case {'action': 'go', 'direction': direction}:
  • Add a guard: case {'action': 'go', 'direction': direction} if direction in ('north', 'south', 'east', 'west'):
  • Put the guarded case before a more general 'go' case that handles the invalid direction.
