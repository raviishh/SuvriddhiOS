Subprocess Pipeline Challenge

Use the subprocess module to run shell commands, capture output, and chain commands like a pipeline.

Part 1 — Running a Command and Capturing Output

subprocess.run(args, capture_output=True, text=True) runs a command and gives you a CompletedProcess with .stdout, .stderr, and .returncode.

import subprocess
result = subprocess.run(["echo", "hello"], capture_output=True, text=True)
print(result.stdout.strip())
print(result.returncode)
Output:
hello
0

What to do

  • Run the command ['python3', '-c', 'print(2 + 2)'] using subprocess.run with capture_output=True and text=True.
  • Print the stripped stdout.
  • Print the return code.

Watch out for

  • If 'python3' isn't the correct interpreter name in your environment, adjust it accordingly (e.g. 'python') — the concept is what matters.
  • Always pass args as a list, not a single shell string, unless you explicitly set shell=True (which has security implications with untrusted input).
Hints (4)
  • subprocess.run(['python3', '-c', 'print(2 + 2)'], capture_output=True, text=True)
  • result.stdout includes a trailing newline — use .strip() before printing.
  • A returncode of 0 conventionally means the command succeeded; non-zero means an error occurred.
  • text=True decodes stdout/stderr as strings instead of raw bytes.

Part 2 — Handling Failures Gracefully

check=True makes subprocess.run() raise CalledProcessError on a non-zero exit code, which you can catch just like any other exception.

import subprocess
try:
    subprocess.run(["python3", "-c", "import sys; sys.exit(1)"], check=True)
except subprocess.CalledProcessError as e:
    print(f"Failed with code {e.returncode}")
Output:
Failed with code 1

What to do

  • Write run_safely(cmd) that runs cmd with subprocess.run(cmd, capture_output=True, text=True, check=True) inside a try/except.
  • On success, return the stripped stdout.
  • On subprocess.CalledProcessError, return f'ERROR (code {e.returncode})'.
  • Call run_safely(['python3', '-c', 'print("ok")']) and run_safely(['python3', '-c', 'import sys; sys.exit(3)']), printing both results.

Watch out for

  • Catch subprocess.CalledProcessError specifically, not a bare except — you want to let unrelated errors (like a missing interpreter) surface normally.
  • The f-string format 'ERROR (code {e.returncode})' must match exactly, including the parentheses.
Hints (4)
  • check=True is what causes a non-zero exit to raise CalledProcessError — without it, you'd need to check result.returncode manually.
  • e.returncode on the caught exception gives you the exact exit code the subprocess reported.
  • Remember to .strip() the successful stdout before returning it.
  • Each call to run_safely is independent — the second call's failure does not affect the first call's already-returned result.

Stretch — Chaining Two Commands (Simulated Pipe) stretch

A shell pipeline like 'cmd1 | cmd2' feeds cmd1's stdout into cmd2's stdin. subprocess lets you do this manually by passing one process's output as the next one's input.

# Shell equivalent: echo "b\na\nc" | sort
p1 = subprocess.run(["echo", "b\na\nc"], capture_output=True, text=True)
p2 = subprocess.run(["sort"], input=p1.stdout, capture_output=True, text=True)
print(p2.stdout)

What to do

  • Write pipeline(cmd1, cmd2) that runs cmd1, captures its stdout, then feeds that stdout as the 'input' argument to cmd2, returning cmd2's stripped stdout.
  • Test it with cmd1 = ['python3', '-c', 'print("3\\n1\\n2")'] and cmd2 = ['sort'].
  • Print the result — it should show the numbers sorted: 1, 2, 3 each on their own line.

Watch out for

  • Do not use shell=True with string concatenation to build this pipeline — passing untrusted or dynamic strings to a shell is a command-injection risk; the input= approach avoids that entirely.
  • 'sort' must be available on the system running this — if unavailable, the underlying concept (feeding one process's output into another's input) is still what's being tested.
Hints (4)
  • step1 = subprocess.run(cmd1, capture_output=True, text=True); step2 = subprocess.run(cmd2, input=step1.stdout, capture_output=True, text=True)
  • The 'input' parameter feeds a string directly into the subprocess's stdin — no need to manage pipes manually.
  • return step2.stdout.strip()
  • This is functionally identical to shell piping (cmd1 | cmd2) but expressed in pure Python without shell=True.
