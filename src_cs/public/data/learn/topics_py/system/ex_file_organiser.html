Build a File Organiser Script

Simulate scanning a folder of mixed files and sorting them into subfolders by type — using an in-memory model of the filesystem.

Part 1 — Categorise by Extension

os.path.splitext(path) splits a filename into (root, extension). The extension includes the leading dot.

import os
root, ext = os.path.splitext("photo.jpg")
print(root, ext)  # photo .jpg
Output:
photo .jpg

What to do

  • Using filenames above, build a dict mapping each file extension (without the dot, lowercase) to a list of filenames with that extension.
  • Print the dict, with extensions sorted alphabetically as keys.
  • Print the total number of distinct extensions found.

Watch out for

  • Print the dict with sorted keys but original (insertion) order within each list — do not sort the filenames themselves.
  • Extensions must be lowercased even if a real filename used uppercase, e.g. '.JPG' should count as 'jpg'.
Hints (4)
  • os.path.splitext(name) returns (root, ext); strip the leading dot with ext[1:] and lowercase it.
  • Build the dict with grouped.setdefault(ext, []).append(name), iterating filenames in their original order so pdf's list keeps report.pdf before invoice.pdf.
  • To print keys sorted alphabetically while preserving insertion order for values, build a new dict: {k: grouped[k] for k in sorted(grouped)}.
  • len(grouped) gives the number of distinct extensions.

Part 2 — Simulated Move Plan

A real organiser would call shutil.move(src, dest_folder). Here we simulate that by producing a plan (a list of source→destination pairs) without touching disk.

CATEGORY_MAP = {
    "pdf": "Documents", "txt": "Documents", "md": "Documents", "csv": "Documents",
    "jpg": "Images", "png": "Images",
    "zip": "Archives"
}

What to do

  • Using CATEGORY_MAP above and filenames from the starter code, build a move plan: a list of tuples (filename, target_folder), where target_folder comes from CATEGORY_MAP based on the file's extension.
  • If an extension isn't in CATEGORY_MAP, use 'Misc' as the target folder.
  • Print each plan entry as 'filename -> target_folder/', one per line, preserving the original filenames order.
  • Print the total count of files planned for the 'Documents' folder.

Watch out for

  • Preserve the exact order from the original filenames list — do not group or sort them for this part.
  • The trailing slash in the printed target folder ('Documents/') is required and part of the exact match.
Hints (4)
  • Get the extension the same way as Part 1: os.path.splitext(name)[1][1:].lower()
  • Use CATEGORY_MAP.get(ext, 'Misc') to default unmapped extensions to 'Misc'.
  • f'{name} -> {folder}/' formats each plan line.
  • Count Documents entries with sum(1 for _, folder in plan if folder == 'Documents').

Stretch — Duplicate Detection by Simulated Content Hash stretch

Real duplicate-file finders hash file contents (e.g. with hashlib.sha256) rather than comparing names, since two differently-named files can have identical content.

import hashlib
data = b"hello world"
print(hashlib.sha256(data).hexdigest()[:8])
Output:
b94d27b9

What to do

  • Given file_contents = {'a.txt': 'hello', 'b.txt': 'world', 'c.txt': 'hello', 'd.txt': 'hello', 'e.txt': 'world'} (simulating file contents as strings instead of reading real files), compute the SHA-256 hash of each file's content (encode as utf-8 first).
  • Group filenames by identical hash.
  • Print only the groups that contain more than one file (i.e. actual duplicates), sorted by the first filename in each group, each group's filenames also sorted.
  • Expected: [['a.txt', 'c.txt', 'd.txt'], ['b.txt', 'e.txt']]

Watch out for

  • Always encode strings to bytes before hashing — hashlib functions require bytes, not str, and will raise a TypeError otherwise.
  • Sort both the outer list of groups and the filenames within each group for a deterministic, checker-friendly output.
Hints (4)
  • hashlib.sha256(content.encode('utf-8')).hexdigest() gives a stable hash string per unique content.
  • Group with a dict: hash_groups.setdefault(hash_value, []).append(filename).
  • Filter to groups where len(group) > 1, then sort each group's filenames and sort the list of groups by their first element.
  • Two files with identical content will always produce identical hashes — this is deterministic, unlike comparing by filename or size alone.
