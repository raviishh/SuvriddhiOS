Model a Real-World System in Classes

Design and implement a small class hierarchy for a library book system.

Part 1 — Book Class

__init__ sets up instance variables. Methods operate on self — the specific instance.

class Dog:
    def __init__(self, name, breed):
        self.name = name
        self.breed = breed

    def bark(self):
        return f"{self.name} says: Woof!"

dog = Dog("Rex", "Labrador")
print(dog.bark())
Output:
Rex says: Woof!

What to do

  • Create a class Book with attributes: title, author, pages.
  • Add a method summary() that returns: '[title] by [author] ([pages] pages)'.
  • Create a Book and print its summary().

Watch out for

  • Use the exact title/author/pages shown in the expected output.
  • pages is an integer in __init__ — the f-string will format it correctly as a number.
Hints (4)
  • def __init__(self, title, author, pages):
  • def summary(self): return f'{self.title} by {self.author} ({self.pages} pages)'
  • The method must return the string, not print it — the caller prints it.
  • self is the first parameter of every instance method — it refers to the specific Book object.

Part 2 — Library Class

One class can hold a collection of instances of another class — a composition pattern.

class Shelf:
    def __init__(self):
        self.books = []

    def add(self, book):
        self.books.append(book)

What to do

  • Create a Library class with an empty list of books.
  • Add an add_book(book) method that appends a Book.
  • Add a list_books() method that prints each book's summary() on its own line.
  • Add 2 books and call list_books().

Watch out for

  • Books must appear in the order they were added — do not sort them.
  • Call book.summary() to get the string — do not duplicate the formatting logic in Library.
Hints (4)
  • self.books = [] in __init__.
  • Iterate with for book in self.books: print(book.summary())
  • list_books() uses print() internally — it doesn't need to return anything.
  • Create the Library, add both Book objects, then call library.list_books().

Stretch — Borrowing System stretch

Add state to Book and behaviour to Library to model a real borrowing system. Track whether each book is checked out and who has it.

# Book gains: is_checked_out (bool), borrower (str or None)
# Library gains: checkout(title, borrower) and return_book(title)

What to do

  • Add is_checked_out = False and borrower = None to Book.__init__.
  • Add checkout(borrower) and return_book() methods to Book.
  • Add checkout(title, borrower) to Library that finds the book by title and checks it out; raise ValueError if not found or already checked out.
  • Add return_book(title) to Library similarly.
  • Add available_books() that prints only books not currently checked out.
  • Demonstrate: add two books, check out one, print available_books() — only the unchecked book should appear.

Watch out for

  • Raise ValueError, don't print an error — let the caller decide how to handle it.
  • The borrower attribute should reset to None when a book is returned.
Hints (4)
  • Book.checkout(borrower): if self.is_checked_out: raise ValueError; self.is_checked_out = True; self.borrower = borrower
  • Book.return_book(): self.is_checked_out = False; self.borrower = None
  • Library.checkout: use next((b for b in self.books if b.title == title), None) to find the book.
  • available_books: filter with if not book.is_checked_out
