Enums

Enums are used when a variable should have only a fixed set of values. They make code safer and easier to read.

1. The Problem with Normal Values

Sometimes we use strings to represent fixed choices.

status = "STARTED"

But someone might accidentally write:

status = "STRATED"   # spelling mistake

Python will not stop this mistake. This can cause bugs.

2. What is an Enum?

An Enum (short for Enumeration) defines a fixed group of constants.

We must import it first.

from enum import Enum

class Status(Enum):
    STARTED = 1
    RUNNING = 2
    FINISHED = 3

Explanation:

  • Status is an Enum class.
  • Each name has a fixed value.
  • No other values are allowed.

3. Using Enums

current_status = Status.STARTED

print(current_status)
print(current_status.name)
print(current_status.value)

Explanation:

  • Status.STARTED is a safe constant.
  • .name gives the name.
  • .value gives the number.

Now we cannot accidentally assign a wrong value.

4. Comparing Enums

Enums are easy to compare.

if current_status == Status.STARTED:
    print("The process has started")

This is safer than comparing strings.

Analogy
Think of traffic lights. Only three colors are allowed: Red, Yellow, Green. You cannot invent a new color. Enums work the same way.

5. When to Use Enums

Use Enums when:

  • There is a fixed set of choices
  • You want to prevent spelling mistakes
  • You want clearer and safer code

Examples:

  • Days of the week
  • Game difficulty levels
  • Order status
  • Traffic signals

6. Small Exercise

Create an Enum called Difficulty.

  • Add EASY, MEDIUM, and HARD.
  • Assign them values 1, 2, and 3.
  • Create a variable and compare it inside an if statement.

7. What You Learned

You now understand:

  • Enums define fixed sets of values.
  • They prevent mistakes caused by wrong strings.
  • They make comparisons clear and safe.
Friendly Tip
If a variable should only have a few allowed values, an Enum is usually better than a string. It keeps your programs strong and reliable.
