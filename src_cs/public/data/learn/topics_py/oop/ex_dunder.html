Dunder Method Implementation Drills

Make custom classes behave like built-in types by implementing __str__, __len__, __add__, and __eq__.

Part 1 — __str__ and __repr__

__str__ controls what print() shows. __repr__ is for developers (shown in the REPL and repr()).

class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __str__(self):
        return f"Point({self.x}, {self.y})"

print(Point(3, 4))  # Point(3, 4)
Output:
Point(3, 4)

What to do

  • Add __str__ to Vector so print(Vector(1, 2)) outputs: Vector(1, 2)
  • Add __repr__ so repr(Vector(1, 2)) outputs: Vector(x=1, y=2)

Watch out for

  • The formats differ: __str__ uses Vector(1, 2) and __repr__ uses Vector(x=1, y=2) — do not mix them up.
  • print(v) calls __str__; print(repr(v)) calls __repr__ then prints the resulting string.
Hints (4)
  • def __str__(self): return f'Vector({self.x}, {self.y})'
  • def __repr__(self): return f'Vector(x={self.x}, y={self.y})'
  • If __str__ is not defined, Python falls back to __repr__ — define both explicitly here.
  • print() calls __str__; repr() calls __repr__.

Part 2 — __add__ and __eq__

__add__ lets you use + between instances. __eq__ lets you use ==.

class Money:
    def __add__(self, other):
        return Money(self.amount + other.amount)

    def __eq__(self, other):
        return self.amount == other.amount

What to do

  • Add __add__ to Vector so Vector(1,2) + Vector(3,4) returns Vector(4,6).
  • Add __eq__ so Vector(1,2) == Vector(1,2) returns True.
  • Print Vector(1,2) + Vector(3,4) and print(Vector(1,2) == Vector(1,2)).

Watch out for

  • Vector(4, 6) is printed using __str__ — make sure Part 1's __str__ is still present.
  • __eq__ should return True or False — do not return 1 or 0.
Hints (4)
  • def __add__(self, other): return Vector(self.x + other.x, self.y + other.y)
  • def __eq__(self, other): return self.x == other.x and self.y == other.y
  • __add__ must return a new Vector — do not modify self in place.
  • print(Vector(1,2) + Vector(3,4)) calls __add__ then __str__ on the result.

Stretch — Full Vector Arithmetic stretch

A complete Vector class also supports subtraction, scalar multiplication, magnitude, and hashing. Implementing __hash__ alongside __eq__ makes Vectors usable in sets and dict keys.

import math
# magnitude = math.sqrt(x**2 + y**2)
# __mul__ for scalar: Vector(1,2) * 3 → Vector(3,6)
# __rmul__ for reverse: 3 * Vector(1,2) → same result

What to do

  • Add __sub__: Vector(5,6) - Vector(1,2) → Vector(4,4).
  • Add __mul__(self, scalar) and __rmul__(self, scalar) for scalar multiplication.
  • Add magnitude(self) that returns math.sqrt(x**2 + y**2), rounded to 4 decimal places.
  • Add __hash__ so Vectors can be used in sets: return hash((self.x, self.y)).
  • Demonstrate: print subtraction, scalar multiply, magnitude, and that {Vector(1,2), Vector(1,2)} has length 1.

Watch out for

  • If __eq__ is defined without __hash__, Python sets __hash__ to None — making the object unhashable. Always define both together.
  • magnitude of Vector(1,2) = sqrt(5) ≈ 2.2361 — use round(..., 4).
Hints (5)
  • __sub__: return Vector(self.x - other.x, self.y - other.y)
  • __mul__(self, scalar): return Vector(self.x * scalar, self.y * scalar)
  • __rmul__ = __mul__ is valid — assign it directly on the class.
  • __hash__: return hash((self.x, self.y)) — tuples are hashable.
  • len({Vector(1,2), Vector(1,2)}) relies on both __eq__ and __hash__ being consistent.
