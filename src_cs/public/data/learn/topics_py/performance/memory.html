Memory Management Basics in Python

When your program runs, it uses memory (RAM). In this lesson, you will learn where your data lives and how Python handles it.

1. What is memory?

Memory (RAM) is the space where your program stores data while it runs.

Analogy
Think of memory as a desk. When you work, you place books and papers on the desk. If the desk becomes full, it becomes messy and slow to use.

Every number, string, list, and object in Python takes space in memory.

2. Variables are labels, not boxes

In Python, variables do not directly store values. They are labels that point to objects in memory. 

a = 10
b = a

Here:

  • An integer object 10 is created in memory.
  • a points to that object.
  • b also points to the same object.

You can check memory identity using: 

print(id(a))
print(id(b))

If both IDs are the same, they point to the same object.

3. Mutable vs Immutable Objects

This is very important for memory.

Immutable (cannot change)

  • int
  • float
  • string
  • tuple
x = 5
x = x + 1

Python does not change the old 5. It creates a new object 6 and makes x point to it.

Mutable (can change)

  • list
  • dict
  • set
numbers = [1, 2, 3]
numbers.append(4)

The same list object is modified. No new list is created.

Key Difference
Immutable objects create new memory when changed. Mutable objects modify existing memory.

4. References and shared objects

Because variables are labels, two variables can point to the same object. 

a = [1, 2]
b = a

b.append(3)
print(a)

Output:

[1, 2, 3]

Why? Because a and b point to the same list.

If you want a separate copy: 

b = a.copy()

5. Common memory mistakes

  • Creating large lists again and again
  • Copying data when not needed
  • Keeping unused variables for a long time
# Memory heavy
big_list = list(range(1000000))
another_list = big_list.copy()

This doubles the memory usage.

Think Carefully
Do you really need a new copy? Or can you reuse the existing data?
