Time Complexity in Python

Learn why some programs run fast and others become slow. In this lesson, we focus on growth — how your code behaves when the input becomes bigger.

1. What does “fast” really mean?

A program can feel fast when the input is small. But what happens when the input becomes very large?

Analogy
Imagine washing 5 plates. Easy. Now imagine washing 5,000 plates. The work grows with the number of plates.

In programming, we care about how the running time grows when the input size grows. This idea is called time complexity.

Input size can mean:

  • The number of items in a list
  • The number of characters in a string
  • The number of rows in a file

2. Big-O notation (simple version)

Big-O notation describes how runtime grows. Do not worry about math. Just focus on the pattern.

O(1) — Constant Time

The time does not change, even if input grows.

numbers = [10, 20, 30]
print(numbers[0])

Accessing by index is constant time. It takes the same time whether the list has 3 items or 3 million.

O(n) — Linear Time

The time grows with the number of items.

for item in numbers:
    print(item)

If the list doubles in size, the work also doubles.

O(n²) — Quadratic Time

The time grows very fast. Often caused by nested loops.

for i in numbers:
    for j in numbers:
        print(i, j)

If the list doubles, the work becomes about four times bigger.

Important Idea
As input grows, O(n²) becomes slow much faster than O(n).

3. Common Python operations and their cost

Some operations in Python are fast. Others are slower.

  • List indexing: fast (O(1))
  • Dictionary lookup: fast (O(1))
  • Loop through list: linear (O(n))
  • Nested loops: quadratic (O(n²))
  • Checking "x in list": linear (O(n))
  • Checking "x in set": fast (O(1))
# Slow for large data
if 1000 in large_list:
    print("Found")

# Faster
large_set = set(large_list)
if 1000 in large_set:
    print("Found")

Sets use a different internal structure that makes lookups faster.

4. Why nested loops are dangerous

Nested loops multiply the work. 

for i in range(n):
    for j in range(n):
        pass

If n = 10 → 100 operations. If n = 1,000 → 1,000,000 operations.

That growth becomes huge very quickly.

Think Before You Loop
Always ask: Can I solve this with one loop instead of two?

5. A simple optimization example

Suppose we want to count duplicates. 

# Slow version
duplicates = []
for x in numbers:
    if numbers.count(x) > 1:
        duplicates.append(x)

count() loops every time. That creates O(n²).

Better version: 

# Faster version
from collections import Counter

counts = Counter(numbers)
duplicates = [x for x in numbers if counts[x] > 1]

This avoids repeated scanning.

Answer these questions:

  • Is a single loop O(n) or O(n²)?
  • Why is searching in a set faster than in a list?
  • What happens if you use three nested loops?
Friendly Tip
Do not try to memorize everything. Just remember this rule: The way your code grows is more important than how fast it feels right now.
