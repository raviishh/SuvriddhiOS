Garbage Collection in Python

When your program creates objects, they use memory. But what happens when those objects are no longer needed? Python cleans them automatically. This process is called garbage collection.

1. What is “garbage” in programming?

Garbage means objects that are no longer used by your program.

Analogy
Imagine your desk again. When you finish using a paper, you throw it away. If you never clean your desk, it becomes full.

In Python, unused objects are cleaned so memory does not stay full.

2. Reference counting

Python mainly uses something called reference counting.

Every object keeps track of how many variables point to it. 

a = [1, 2, 3]   # reference count = 1
b = a           # reference count = 2

Now the list has two references: a and b. 

del a           # reference count = 1
del b           # reference count = 0

When the reference count becomes zero, Python removes the object from memory.

3. The problem of circular references

Sometimes objects refer to each other. 

a = []
b = []

a.append(b)
b.append(a)

Now:

  • a points to b
  • b points to a

Even if you delete a and b, they still reference each other. The reference count never becomes zero.

This is called a circular reference.

4. Python’s garbage collector

To fix circular references, Python has a special garbage collector.

It:

  • Looks for groups of objects that reference each other
  • Checks if they are still reachable from the program
  • Removes them if they are not used anymore

This runs automatically. Most of the time, you do not need to control it.

5. When should you care?

Usually, you do not need to think about garbage collection.

But you should care when:

  • Your program runs for a very long time
  • You handle very large data
  • Memory usage keeps increasing

You can manually run garbage collection: 

import gc
gc.collect()

But in most cases, this is not necessary.

Important Rule
If memory keeps growing, check your references. Something may still be pointing to objects.

6. Practice

Think about this: 

a = [1, 2, 3]
b = a
a = None
print(b)

Questions:

  • Does the list still exist in memory?
  • Why?

Friendly Tip
Python helps you manage memory automatically. But understanding how it works makes you a stronger programmer.
