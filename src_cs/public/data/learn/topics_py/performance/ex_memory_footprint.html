Memory Footprint Comparison

Compare memory usage of lists vs generators using sys.getsizeof.

List vs Generator Memory

A list materialises all values in memory. A generator produces values lazily — it only holds state, not data. For large sequences the difference is dramatic.

import sys

my_list = [x for x in range(1000)]
my_gen = (x for x in range(1000))

print(sys.getsizeof(my_list))  # ~8856
print(sys.getsizeof(my_gen))   # ~104
Output:
8856
104

What to do

  • Create a list comprehension for range(1_000_000) and a generator expression for the same.
  • Print the size of each using sys.getsizeof.
  • Print which one is larger: 'List is larger' or 'Generator is larger'.

Watch out for

  • The generator object is tiny (around 104–120 bytes) regardless of range size — it stores only a reference to the iterator state.
  • Do not consume the generator before measuring — iterating it changes its state.
Hints (4)
  • my_list = [x for x in range(1_000_000)]
  • my_gen = (x for x in range(1_000_000))
  • Compare sys.getsizeof(my_list) > sys.getsizeof(my_gen)
  • sys.getsizeof only measures the container itself, not the objects inside — for a list of ints the container alone is already ~8MB.

Stretch — Chunked Processing stretch

When data is too large to fit in memory, process it in chunks using a generator. This keeps memory usage constant regardless of total data size.

def chunked(iterable, size):
    chunk = []
    for item in iterable:
        chunk.append(item)
        if len(chunk) == size:
            yield chunk
            chunk = []
    if chunk:
        yield chunk

What to do

  • Implement chunked(iterable, size) as shown above.
  • Use it to process range(1, 16) in chunks of 4, printing each chunk.
  • Then compute the sum of each chunk and print a running total after each chunk.
  • Final running total should be 120.

Watch out for

  • range(1, 16) includes 15 — you should see a final chunk of [13, 14, 15].
  • Print each chunk before adding to the running total — order matters.
Hints (3)
  • The last chunk may be smaller than size — the if chunk: yield chunk handles it.
  • running_total = 0; for chunk in chunked(range(1,16), 4): print(chunk); running_total += sum(chunk)
  • After the loop: print(running_total)
