Profile and Optimize a Slow Script

Identify bottlenecks with timeit and replace O(n²) logic with O(n) alternatives.

Part 1 — Faster Duplicate Check

Checking membership in a set is O(1). Building the set is O(n). That makes a set-based duplicate check O(n) — much better than the O(n²) nested loop.

def has_duplicates_fast(lst):
    return len(lst) != len(set(lst))

What to do

  • Write has_duplicates_fast(lst) using a set.
  • Assert it gives the same results as the slow version for [1,2,3], [1,2,1], and [].
  • Print True and False to confirm.

Watch out for

  • Print True first (for [1,2,1]) and False second (for [1,2,3]) — order matters.
  • The fast version must return a bool, not a set or length.
Hints (4)
  • len(lst) != len(set(lst)) is True only when there are duplicates.
  • Test with [1, 2, 1] (True) and [1, 2, 3] (False).
  • An empty list has no duplicates — len([]) == len(set([])) == 0, so False is correct.
  • set() discards duplicates — if the lengths differ, at least one duplicate existed.

Stretch — Benchmark with timeit stretch

timeit.timeit(stmt, number) runs stmt n times and returns total seconds. Comparing both versions on a large input quantifies the speedup.

import timeit
t = timeit.timeit('has_duplicates_slow(lst)', globals=globals(), number=100)

What to do

  • Create test_list = list(range(1000)) (no duplicates — worst case for the slow version).
  • Use timeit.timeit to time has_duplicates_slow(test_list) and has_duplicates_fast(test_list), each for 100 repetitions.
  • Print the speedup factor: slow_time / fast_time, rounded to 1 decimal place.
  • Print: 'Fast is Nx faster' where N is the rounded speedup.

Watch out for

  • number=100 is enough — using 10000 on the slow version will take a long time.
  • The exact speedup depends on your machine — the checker only looks for 'Fast is' as a prefix.
Hints (4)
  • timeit.timeit(lambda: has_duplicates_slow(test_list), number=100) is cleaner than using globals().
  • speedup = slow_time / fast_time
  • f'Fast is {round(speedup, 1)}x faster'
  • On most machines the fast version is 50–200x faster for n=1000.
