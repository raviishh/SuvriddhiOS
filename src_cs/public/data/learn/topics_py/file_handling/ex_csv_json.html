CSV & JSON Read/Write Drills

Practice round-tripping structured data through the csv and json modules.

Part 1 — Writing and Reading CSV

csv.writer writes rows as lists. csv.DictWriter writes rows as dicts, using a header row of fieldnames.

import csv, io
buf = io.StringIO()
writer = csv.writer(buf)
writer.writerow(["name", "age"])
writer.writerow(["Alice", 30])
print(buf.getvalue())
Output:
name,age
Alice,30

What to do

  • Using io.StringIO() as an in-memory file, write a CSV with header ['name', 'score'] and rows [('Alice', 85), ('Bob', 92), ('Carol', 78)] using csv.writer.
  • Then read it back using csv.reader, skipping the header.
  • Print each data row as a tuple: ('Alice', '85') etc.
  • Print the sum of all scores converted to int: 255.

Watch out for

  • Remember to call buf.seek(0) after writing and before reading — the cursor is at the end after writing.
  • Values read from CSV are always strings, even numbers like '85' — sum them only after converting to int.
Hints (4)
  • Use io.StringIO() to avoid touching the real filesystem, then buf.seek(0) before reading it back.
  • csv.reader yields lists — convert with tuple(row) to match the expected format.
  • next(reader) consumes and discards the header row.
  • CSV always returns strings — convert with int(row[1]) before summing.

Part 2 — JSON Serialization Round-Trip

json.dumps() converts a Python object to a JSON string; json.loads() converts it back. Only JSON-compatible types (dict, list, str, int, float, bool, None) can be serialised directly.

import json
data = {"name": "Alice", "active": True}
s = json.dumps(data)
print(s)  # '{"name": "Alice", "active": true}'
back = json.loads(s)
print(back == data)  # True
Output:
{"name": "Alice", "active": true}
True

What to do

  • Create a dict: student = {'name': 'Dana', 'grades': [88, 92, 79], 'passed': True}.
  • Serialise it to a JSON string with json.dumps(), using indent=2.
  • Print the JSON string.
  • Deserialise it back and print the average of grades, rounded to 2 decimal places.

Watch out for

  • json.dumps() converts True/False to lowercase true/false in the JSON text, but json.loads() converts them back to Python's True/False.
  • Don't forget to round the average — the raw float will have many more decimal digits.
Hints (4)
  • json.dumps(student, indent=2) produces pretty-printed, human-readable JSON.
  • json.loads(json_string) reconstructs the original dict (with the same structure).
  • average = round(sum(grades) / len(grades), 2)
  • (88 + 92 + 79) / 3 = 259 / 3 ≈ 86.33

Stretch — CSV to Nested JSON Pipeline stretch

A common real-world task is converting flat tabular data (CSV) into a nested structure (JSON) grouped by a key — for example, grouping purchases by customer.

# CSV rows: customer,item,price
# alice,book,12.99
# alice,pen,1.50
# bob,book,12.99
# Target JSON:
# {"alice": [{"item": "book", "price": 12.99}, {"item": "pen", "price": 1.5}], "bob": [...]}

What to do

  • Using csv.DictReader on the in-memory CSV text: 'customer,item,price\nalice,book,12.99\nalice,pen,1.50\nbob,book,12.99\nbob,pen,1.50\nbob,pencil,0.50\n', group rows by customer into a dict of lists.
  • Each list entry should be a dict with keys 'item' and 'price' (price converted to float).
  • Serialise the resulting dict to a JSON string with json.dumps(sort_keys=True).
  • Print the total spent by bob, rounded to 2 decimal places: 14.99.

Watch out for

  • Prices arrive as strings from CSV — always convert to float before summing or storing.
  • 12.99 + 1.50 + 0.50 = 14.99 — verify your rounding doesn't introduce floating-point artefacts like 14.989999999999998; round() fixes this for display.
Hints (4)
  • csv.DictReader(io.StringIO(csv_text)) gives you an iterable of dicts with keys matching the header row.
  • Use grouped.setdefault(row['customer'], []).append({'item': row['item'], 'price': float(row['price'])}).
  • Sum bob's prices with sum(entry['price'] for entry in grouped['bob']).
  • json.dumps(grouped, sort_keys=True) is only needed to demonstrate serialisation — the final printed value is the numeric total.
