File Parser Challenges

Read, parse, and extract structured information from plain text files.

Part 1 — Line Counting & Filtering

open(path) returns a file object you can iterate line by line. Always use a 'with' block so the file closes automatically.

with open("notes.txt") as f:
    lines = f.readlines()
print(len(lines))
Output:
5

What to do

  • Given the multi-line string data = "apple\nbanana\n\ncherry\ndate\n\n\n", split it into lines using .splitlines().
  • Print the total number of non-empty lines.
  • Print only the non-empty lines, one per line.

Watch out for

  • This task works on an in-memory string, not an actual file — no need to open() anything here.
  • Blank lines with just whitespace (e.g. ' ') should also be treated as empty.
Hints (4)
  • data.splitlines() avoids trailing newline characters, unlike readlines().
  • Filter with: [line for line in lines if line.strip()]
  • len() of the filtered list gives the non-empty count.
  • An empty string is falsy, so 'if line.strip()' skips blank lines including ones with only whitespace.

Part 2 — Structured Log Extraction

Real log files have a predictable shape per line. Splitting on whitespace or a delimiter lets you pull out fields.

line = "2024-01-15 ERROR Disk full"
parts = line.split(" ", 2)
print(parts)  # ['2024-01-15', 'ERROR', 'Disk full']
Output:
['2024-01-15', 'ERROR', 'Disk full']

What to do

  • Given log_data = "2024-01-15 INFO Server started\n2024-01-15 ERROR Disk full\n2024-01-16 INFO Backup complete\n2024-01-16 ERROR Connection lost\n2024-01-16 ERROR Timeout", split it into lines.
  • Count how many lines contain the level ERROR.
  • Print: 'ERROR count: [n]'
  • Print each ERROR line's message only (the text after the level), one per line.

Watch out for

  • Without maxsplit, a message containing spaces (like 'Disk full') would be split into extra parts.
  • Print the count line first, then the messages in their original order.
Hints (4)
  • Use line.split(" ", 2) to split into exactly 3 parts: date, level, message.
  • Check parts[1] == 'ERROR' to filter.
  • The maxsplit argument (the '2' in split(' ', 2)) prevents the message itself from being split further.
  • Build a list of ERROR messages first, then print the count followed by each message.

Stretch — Word Frequency Across a Multi-Line Document stretch

Parsing real documents usually means normalising case and punctuation before counting, otherwise 'Word' and 'word,' are treated as different tokens.

import string
text = "The cat. The dog! The cat?"
clean = text.translate(str.maketrans('', '', string.punctuation)).lower()
print(clean.split())
Output:
['the', 'cat', 'the', 'dog', 'the', 'cat']

What to do

  • Given document = "Python is great. Python is fun! Is Python easy? Python, yes.", normalise it by removing punctuation and lowercasing.
  • Count the frequency of each word.
  • Print the top 3 most frequent words with their counts, sorted by count descending then alphabetically for ties, one per line as 'word: count'.
  • Expected: python: 4, is: 3, great: 1 (or another word tied at 1, resolved alphabetically).

Watch out for

  • Do not use regex here — string.punctuation and str.translate are sufficient and simpler.
  • Ties among lower-frequency words should break alphabetically, matching Python's stable sort behaviour when you sort by (-count, word).
Hints (4)
  • Use str.maketrans('', '', string.punctuation) with .translate() to strip punctuation.
  • Sort with sorted(freq.items(), key=lambda x: (-x[1], x[0])).
  • Slice the sorted list to get the top 3.
  • Remember to .lower() before splitting so 'Python' and 'python' count as the same word.
