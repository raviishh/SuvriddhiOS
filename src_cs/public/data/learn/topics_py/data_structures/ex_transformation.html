Data Structure Transformation Challenges

Convert between lists, tuples, sets, and dicts to solve real-shaped problems.

Part 1 — Group Scores by Name

Dicts with list values are a common pattern for grouping. Check if the key exists first, or use dict.setdefault().

grouped = {}
for name, score in data:
    grouped.setdefault(name, []).append(score)
print(grouped)
Output:
{'Alice': [85, 78], 'Bob': [92, 88], 'Carol': [95]}

What to do

  • Using data above, build a dict mapping each name to a list of their scores.
  • Print the dict.

Watch out for

  • The order of names in the output dict follows insertion order (Python 3.7+) — Alice appears first because she appears first in data.
  • Do not sort the output — match the expected order exactly.
Hints (4)
  • Use grouped.setdefault(name, []).append(score).
  • Iterate over the list of tuples with for name, score in data.
  • setdefault(key, default) returns the existing value if the key exists, otherwise inserts default and returns it.
  • An alternative: if name not in grouped: grouped[name] = []; grouped[name].append(score)

Part 2 — Average per Person

Once you have grouped data, computing aggregates (sum, average) per group is straightforward.

scores = [85, 78]
avg = sum(scores) / len(scores)
print(avg)  # 81.5
Output:
81.5

What to do

  • From the grouped dict, create a new dict mapping each name to their average score (rounded to 1 decimal place).
  • Print it.

Watch out for

  • Carol's average is 95.0, not 95 — round() preserves the float.
  • Build grouped first (from Part 1) before computing averages.
Hints (4)
  • Use a dict comprehension: {name: round(sum(scores)/len(scores), 1) for name, scores in grouped.items()}
  • round(value, 1) rounds to one decimal place.
  • sum([95]) / len([95]) = 95.0 — a single-element list still works correctly.
  • .items() returns (key, value) pairs — unpack them in the comprehension.

Stretch — Ranking & Sorting stretch

sorted() with a key function lets you sort by any attribute. Combined with enumerate(), you can produce ranked output.

# Sort a dict by value descending:
# sorted(d.items(), key=lambda x: x[1], reverse=True)

What to do

  • Using the averages dict from Part 2, produce a ranked leaderboard.
  • Sort names by average score descending.
  • Print each entry as: #[rank] [name]: [average]
  • Expected: #1 Carol: 95.0 #2 Bob: 90.0 #3 Alice: 81.5

Watch out for

  • rank starts at 1, not 0 — use enumerate(..., start=1).
  • Ties are not present in this data, but in general sorted() is stable — equal items keep their original order.
Hints (3)
  • sorted(averages.items(), key=lambda x: x[1], reverse=True) gives [(name, avg), ...] in descending order.
  • Use enumerate(sorted_items, start=1) to get 1-based ranks.
  • f"#{rank} {name}: {avg}" formats the output.
