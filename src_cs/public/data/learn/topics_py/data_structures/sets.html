Sets in Python

Sets are unordered collections of unique elements. They are optimized for membership testing, deduplication, and mathematical set operations.

1. What Is a Set?

A set is a collection of distinct values enclosed in curly braces. Duplicate values are automatically removed. 

numbers = {1, 2, 3, 3}
print(numbers)   # {1, 2, 3}
Key Properties
• Unordered
• Mutable
• No duplicate elements
• Fast membership testing

2. Creating Sets

Use curly braces or the set() constructor.

a = {1, 2, 3}
b = set([4, 5, 6])
empty = set()   # Not {}

Note: {} creates an empty dictionary, not a set.

3. Adding and Removing Elements

fruits = {"apple", "banana"}

fruits.add("cherry")
fruits.remove("banana")
fruits.discard("mango")   # No error if not found
fruits.pop()              # Removes arbitrary element

print(fruits)

Since sets are unordered, pop() removes a random element.

4. Set Operations

Sets support mathematical operations.

a = {1, 2, 3}
b = {3, 4, 5}

print(a | b)   # Union
print(a & b)   # Intersection
print(a - b)   # Difference
print(a ^ b)   # Symmetric difference

Equivalent methods:

a.union(b)
a.intersection(b)
a.difference(b)
a.symmetric_difference(b)

5. Membership and Performance

Sets use hash tables internally. Membership testing is typically O(1). 

numbers = {1, 2, 3, 4}
print(3 in numbers)   # True
Performance Insight
Use sets instead of lists when frequent membership checks are required.

6. Immutable Sets (frozenset)

A frozenset is an immutable version of a set.

fs = frozenset([1, 2, 3])
# fs.add(4)  # AttributeError

Useful when sets need to be dictionary keys.

7. Common Mistakes

  • Expecting sets to preserve order
  • Using {} to create an empty set
  • Trying to add mutable objects (like lists)
  • Assuming pop() removes the “last” element
Quick Check
What is the result? s = {1, 2, 2, 3} s.add(3) print(len(s))

8. Why Sets Matter

Sets provide efficient tools for uniqueness and mathematical comparisons. They are critical in data cleaning, filtering, and optimization scenarios.

Mastery of sets enables better algorithm design, particularly when working with large datasets.

Practice Strategy
