Frequency Counter & Deduplication

Count occurrences and remove duplicates — two of the most common real-world data tasks.

Part 1 — Word Frequency

A frequency counter is a dict where keys are items and values are counts. dict.get(key, 0) returns 0 if the key is missing.

freq = {}
for item in items:
    freq[item] = freq.get(item, 0) + 1

What to do

  • Count how many times each word appears in words.
  • Print the frequency dict.
  • Print the most common word and its count.

Watch out for

  • The dict must be printed before the most common word — order matters.
  • If there were a tie, max() would return one arbitrarily — no tie-breaking needed here.
Hints (4)
  • freq[word] = freq.get(word, 0) + 1
  • max(freq, key=freq.get) gives the key with the highest value.
  • collections.Counter(words) does this in one line — implement it manually first, then compare.
  • f"{most_common}: {freq[most_common]}" formats the second output line.

Part 2 — Deduplication with Order Preserved

set() removes duplicates but loses order. To deduplicate while preserving insertion order, use a dict (Python 3.7+ maintains insertion order).

seen = {}
for item in items:
    seen[item] = True
deduped = list(seen.keys())

What to do

  • Produce a deduplicated version of words preserving original order.
  • Print the result.

Watch out for

  • Do not use set() — it does not preserve order.
  • The output must be a list, not a dict or a dict_keys view.
Hints (4)
  • dict.fromkeys(words) creates a dict with each word as a key (values are None) — then list() it.
  • Or iterate and track seen items in a regular dict.
  • list(dict.fromkeys(words)) is idiomatic Python for order-preserving deduplication.
  • Verify: len(deduped) should be 4.

Stretch — Top-N with Ties stretch

collections.Counter has a most_common(n) method, but implementing it yourself teaches sorting and tie-handling.

# Target: top 2 most frequent words, ties broken alphabetically
# apple: 3, banana: 2 → apple and banana

What to do

  • Using the frequency dict from Part 1, write top_n(freq, n) that returns the top-n words by frequency.
  • Break ties alphabetically (ascending).
  • Print top_n(freq, 2).
  • Expected: ['apple', 'banana']

Watch out for

  • Negating the count (-x[1]) sorts descending by count while keeping the sort ascending for the word.
  • cherry and date both have count 1 — they are tied; alphabetically cherry < date, so cherry ranks higher among the tail.
Hints (4)
  • Sort by (-count, word) to get highest count first and alphabetical order for ties.
  • sorted(freq.items(), key=lambda x: (-x[1], x[0]))
  • Slice the sorted list: [:n] to get the top n.
  • Return only the words, not the counts: [word for word, _ in sorted_items[:n]]
