Type Coercion Puzzles

Predict and verify what Python does when types meet each other.

Part 1 — Explicit Conversion

int(), float(), and str() convert values between types. int(3.9) truncates (gives 3), not rounds.

print(int(3.9))   # 3
print(float(7))   # 7.0
print(str(42))    # '42'
Output:
3
7.0
42

What to do

  • Convert the float 9.99 to an int and print it.
  • Convert the integer 100 to a float and print it.
  • Convert the integer 7 to a string, concatenate " wonders", and print it.

Watch out for

  • int() truncates, it does not round — int(9.99) is 9, not 10.
  • int(-3.9) is -3, not -4 — truncation always moves toward zero.
Hints (4)
  • int(9.99) truncates toward zero.
  • String concatenation requires both sides to be strings: str(7) + " wonders"
  • float(100) adds a decimal point: 100.0
  • You cannot concatenate an int and a string directly — Python will raise a TypeError.

Part 2 — Mixed-Type Arithmetic

Python automatically promotes int to float when you mix them in arithmetic. Adding a bool to an int also works — True is 1, False is 0.

print(3 + 1.0)      # 4.0
print(True + 5)     # 6
print(False + 10)   # 10
Output:
4.0
6
10

What to do

  • Print the result of 5 + 2.0
  • Print the result of True + True + False
  • Print the type of (5 + 2.0) using type()

Watch out for

  • Print the type object itself, not a string description like 'float'.
  • type(5 + 2.0) gives <class 'float'>, not <class 'int'> — the int is promoted.
Hints (4)
  • int + float → float automatically.
  • type() returns the type object; print it directly.
  • True == 1 and False == 0 in arithmetic contexts — this is guaranteed by the Python spec.
  • Wrap the expression in type(): type(5 + 2.0)

Stretch — Safe String-to-Number Pipeline stretch

int() and float() raise ValueError for invalid strings. A robust pipeline checks the string first or catches the exception.

def safe_int(s):
    try:
        return int(s)
    except ValueError:
        return None

What to do

  • Write a function coerce_number(s) that tries to convert string s to int first, then float, and returns None if neither works.
  • coerce_number('42') → 42 (int)
  • coerce_number('3.14') → 3.14 (float)
  • coerce_number('hello') → None
  • Print the type and value for each of the three test cases.
  • Expected output (three lines): <class 'int'> 42 / <class 'float'> 3.14 / <class 'NoneType'> None

Watch out for

  • int('3.14') raises ValueError — you cannot skip the float step.
  • Return the converted value, not a string representation of it.
Hints (3)
  • Nest two try/except blocks: try int() first, then float() in the except, then return None in the second except.
  • type(None) is <class 'NoneType'>.
  • print(type(result), result) produces the correct format in one call.
