Spot the Mutation Bug

Identify and fix bugs caused by unexpected mutation and shared references.

Part 1 — Fix the Shared Reference

Assigning a list to a new variable does NOT copy it — both names point to the same object. Use list() or [:] to make a real copy.

a = [1, 2, 3]
b = a          # same object!
b.append(4)
print(a)       # [1, 2, 3, 4]  ← surprise!
Output:
[1, 2, 3, 4]

What to do

  • Fix the starter code so that appending 4 to copy does NOT affect original.
  • After your fix, original should still print [1, 2, 3] and copy should print [1, 2, 3, 4].

Watch out for

  • Both lines must be printed in the exact format shown.
  • copy = original[:] creates a shallow copy — nested lists inside would still be shared. For this exercise the list is flat, so [:] is sufficient.
Hints (4)
  • Use copy = original[:] or copy = list(original).
  • Do not use copy = original — that just creates another name for the same list.
  • You can verify they are different objects with: print(original is copy) — it should print False after the fix.
  • copy.copy() from the copy module also works: import copy; c = copy.copy(original)

Part 2 — Immutable Strings

Strings are immutable — methods like .replace() return a new string rather than modifying the original.

word = "hello"
word.upper()     # does nothing to word!
print(word)      # still 'hello'
Output:
hello

What to do

  • Start with greeting = "hello world".
  • Convert it to upper case and store the result in shouted.
  • Print greeting (unchanged) and then print shouted.

Watch out for

  • greeting must remain lowercase in the output — do not reassign it.
  • Print greeting first, then shouted — order matters for the checker.
Hints (4)
  • shouted = greeting.upper() — remember to capture the return value.
  • Do not reassign greeting.
  • If you just write greeting.upper() without assigning it, the result is discarded.
  • This pattern — capturing the return value of a method — is the correct way to work with all immutable types.

Stretch — Deep Copy Trap stretch

A shallow copy duplicates the outer list but not any nested lists inside it. Use copy.deepcopy() when you need full independence at every level.

import copy
nested = [[1, 2], [3, 4]]
shallow = nested[:]
shallow[0].append(99)
print(nested)   # [[1, 2, 99], [3, 4]] ← the inner list is still shared!
Output:
[[1, 2, 99], [3, 4]]

What to do

  • Start with matrix = [[1, 2], [3, 4]].
  • Create a shallow copy called shallow using [:].
  • Create a deep copy called deep using copy.deepcopy().
  • Append 99 to shallow[0] and append 88 to deep[1].
  • Print matrix, shallow, and deep — one per line.
  • Expected: matrix is [[1,2,99],[3,4]], shallow is [[1,2,99],[3,4]], deep is [[1,2],[3,4,88]].

Watch out for

  • shallow = matrix[:] copies the outer list only — inner list objects are still the same.
  • Print each list on its own line — the checker matches line by line.
Hints (4)
  • import copy at the top.
  • deep = copy.deepcopy(matrix) creates a fully independent copy.
  • Appending to shallow[0] modifies the inner list, which matrix still references.
  • Appending to deep[1] only affects deep — its inner lists are independent copies.
