Data Analysis on a Sample Dataset

Use pandas-style thinking (and pandas itself, if available) to explore, clean, and summarise tabular data.

Part 1 — Group-By Aggregation

pandas.DataFrame(data).groupby('dept')['salary'].mean() computes the average salary per department. The same result can be built manually with a dict of running sums and counts.

import pandas as pd
df = pd.DataFrame(data)
print(df.groupby("dept")["salary"].mean())
Output:
dept
Engineering    99333.333333
Sales          60000.000000
Name: salary, dtype: float64

What to do

  • Using data above, compute the average salary per department.
  • Print each department and its average salary rounded to 2 decimal places, sorted alphabetically by department name, one per line as 'dept: average'.
  • Print the department with the highest average salary.

Watch out for

  • 60000.0 prints with a trailing .0 since it's a float average — don't convert it to an int.
  • Sort departments alphabetically for the printed lines, but the final 'highest average' line is not sorted — it's just the single winning department name.
Hints (4)
  • If using pandas: df.groupby('dept')['salary'].mean().round(2)
  • Without pandas: build a dict of {dept: [salaries]} using setdefault, then compute averages manually.
  • (95000 + 105000 + 98000) / 3 = 99333.333... which rounds to 99333.33.
  • Compare the computed averages to find the department with the highest mean.

Part 2 — Handling Missing and Outlier Values

Real datasets have gaps. A common cleaning step is dropping or imputing missing values before aggregating, so they don't silently skew results.

raw = [10, None, 20, 15, None, 1000]
cleaned = [x for x in raw if x is not None]
print(cleaned)  # [10, 20, 15, 1000]
Output:
[10, 20, 15, 1000]

What to do

  • Given scores = [88, 92, None, 79, 45, None, 91, 500], first remove all None values.
  • Then remove outliers: any value more than 2x the median of the cleaned list (this correctly flags 500 as an outlier given the other values).
  • Print the final cleaned list (order preserved).
  • Print the mean of the final cleaned list, rounded to 2 decimal places.

Watch out for

  • Compute the median BEFORE removing outliers (using the None-free list), not after — the outlier threshold must be based on the original spread of legitimate values.
  • The final list must preserve the ORIGINAL order from scores (minus the removed None and the outlier), not the sorted order used internally to compute the median.
Hints (4)
  • First pass: no_none = [x for x in scores if x is not None] → [88, 92, 79, 45, 91, 500]
  • Compute the median of no_none using sorted values and picking the middle (or averaging the two middle values for an even-length list).
  • Filter: [x for x in no_none if x <= 2 * median]
  • sorted([88, 92, 79, 45, 91, 500]) = [45, 79, 88, 91, 92, 500], median of 6 items = (79+91)/2? No — length 6, so median = average of the 3rd and 4th sorted values: (88+91)/2 = 89.5, and 2*89.5 = 179, correctly excluding 500.

Stretch — Pivot-Style Cross-Tabulation stretch

A pivot table cross-tabulates two categorical dimensions with an aggregated value — for example, total sales by region AND product category simultaneously.

# pandas: df.pivot_table(index='region', columns='category', values='amount', aggfunc='sum')

What to do

  • Given sales = [{'region': 'East', 'category': 'A', 'amount': 100}, {'region': 'East', 'category': 'B', 'amount': 50}, {'region': 'West', 'category': 'A', 'amount': 200}, {'region': 'West', 'category': 'B', 'amount': 75}, {'region': 'East', 'category': 'A', 'amount': 30}], build a nested dict pivot[region][category] = total_amount.
  • Print the pivot dict with regions and categories both sorted alphabetically at each level.
  • Print the grand total across all entries.

Watch out for

  • Build the sorted-key version only for printing — don't let sorting logic interfere with the accumulation step, which should happen before any sorting.
  • The grand total must equal the sum of ALL original 'amount' values, which serves as a good sanity check against your pivot's own totals.
Hints (4)
  • pivot = {}; for row in sales: pivot.setdefault(row['region'], {}); pivot[row['region']][row['category']] = pivot[row['region']].get(row['category'], 0) + row['amount']
  • To print with sorted keys at both levels: {r: {c: pivot[r][c] for c in sorted(pivot[r])} for r in sorted(pivot)}
  • East's category A total: 100 + 30 = 130.
  • Grand total: sum of every 'amount' value across all rows in sales = 100+50+200+75+30 = 455.
