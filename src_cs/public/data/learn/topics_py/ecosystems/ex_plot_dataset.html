Part 1 — Summary Statistics for a Line Chart

Before plotting a trend line, you typically compute the min, max, and overall change to understand the shape of the data. matplotlib.pyplot.plot(x, y) would draw this as a line, but the numbers behind it are what matter first.

data = [10, 15, 12, 20]
print(max(data), min(data), data[-1] - data[0])  # 20 10 10
Output:
20 10 10

What to do

  • Using monthly_sales (12 values, one per month, January first), find the month with the highest sales (1-indexed) and the value.
  • Find the month with the lowest sales (1-indexed) and the value.
  • Print: 'Peak: month [n] ([value])'
  • Print: 'Trough: month [n] ([value])'
  • Print the net change from January to December: 'Net change: [value]'

Watch out for

  • Months are 1-indexed in the output (January = month 1), but list indices are 0-indexed — remember the +1 adjustment.
  • index() returns the FIRST matching index — fine here since there are no duplicate max/min values in this dataset.

  • Similarly for the trough using min().
  • Net change: monthly_sales[-1] - monthly_sales[0]
  • This is exactly the data a line chart's y-axis annotations would highlight.

Part 2 — Binning for a Histogram

A histogram groups continuous data into bins and counts how many values fall in each. matplotlib.pyplot.hist(data, bins=n) does this automatically, but understanding the binning logic helps you interpret (or debug) the resulting chart.

values = [5, 12, 15, 22, 28, 33, 8]
bin_size = 10
bins = {}
for v in values:
    b = (v // bin_size) * bin_size
    bins[b] = bins.get(b, 0) + 1
print(sorted(bins.items()))
Output:
[(0, 2), (10, 2), (20, 2), (30, 1)]

What to do

  • Using monthly_sales, create bins of width 25 starting at 100 (i.e. bins are [100,125), [125,150), [150,175), [175,200), [200,225)).
  • Count how many months fall into each bin.
  • Print the bin counts as a sorted list of tuples (bin_start, count), including any empty bins in the range with count 0.

Watch out for

  • Without pre-initialising all expected bins, an empty bin would simply be missing from the dict rather than showing a 0 count — make sure to seed them explicitly.
  • Double-check boundary values like 150 and 175 fall into the correct bin ([150,175) not [125,150), and [175,200) not [150,175)).
Hints (4)
  • bin_start = 100 + ((v - 100) // 25) * 25 assigns each value to the correct bin floor.
  • Initialise all 5 expected bins to 0 first: bins = {100: 0, 125: 0, 150: 0, 175: 0, 200: 0}, then increment as you iterate.
  • This ensures empty bins still appear in the final sorted output, matching what a real histogram would show as a zero-height bar.
  • 200 falls into the [200, 225) bin on its own since it's the maximum value in the data.

Stretch — Moving Average Smoothing stretch

A moving average smooths a noisy line chart by averaging each point with its neighbours, making the underlying trend easier to see — a very common technique before plotting time-series data.

# 3-point moving average for [1, 2, 3, 4, 5]:
# point 2 (index 1): avg(1,2,3) = 2.0
# point 3 (index 2): avg(2,3,4) = 3.0
# edges (index 0 and 4) have fewer neighbours and are often left out or handled specially

What to do

  • Write moving_average(data, window) that computes a centered moving average, only for indices that have a FULL window available on both sides (i.e. skip the edges rather than padding them).
  • For window=3, the result for monthly_sales should have exactly 10 values (skipping the first and last month).
  • Print the moving average list, with each value rounded to 2 decimal places.
  • Print how much smoother it is: the difference between max(monthly_sales) - min(monthly_sales) and max(smoothed) - min(smoothed), rounded to 2 decimal places (should be positive, showing reduced range).

Watch out for

  • A centered window requires an odd window size to have equal neighbours on both sides — window=3 is used here specifically because it divides evenly around each center point.
  • Don't pad the edges with partial windows for this task — the instructions specifically ask you to skip indices without a full window, which is why the result has exactly 10 (not 12) values.

  • len(monthly_sales) - 2 = 10 valid centered points for a window of 3.
  • range_original = max(monthly_sales) - min(monthly_sales); range_smoothed = max(smoothed) - min(smoothed); the difference should be positive since smoothing narrows the range.
